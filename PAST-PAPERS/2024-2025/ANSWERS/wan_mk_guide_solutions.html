B2 — Steps when conducting a WAN site visit

Step No.Step
1Plan and schedule the visit with relevant stakeholders.
2Identify and survey the physical location of network devices.
3Record existing network topology and cabling.
4Check for availability of power, cooling, and rack space.
5Verify WAN connectivity options (leased line, fiber, DSL, wireless).
6Assess security and environmental risks.
7Document findings and report back to the network team.

B3 — Four WAN technologies & applicability

WAN TechnologyWhere applicable / Typical use
Leased Line (Point-to-Point)Dedicated site-to-site links for enterprises requiring stable, guaranteed bandwidth.
MPLS (Multiprotocol Label Switching)Large multi-site networks needing scalable VPNs and traffic engineering (enterprise/VPN services).
DSL (Digital Subscriber Line)Cost-effective Internet for small offices or branch sites using existing telephone copper lines.
Fiber Optic WANHigh-capacity, long-distance links for ISPs, data centers and large organizations requiring high throughput.

Section C — Modulation & Switch Configuration

C1.A — What is modulation and demodulation

TermDefinition
ModulationProcess of varying a carrier signal’s property (amplitude, frequency or phase) to encode and transmit information.
DemodulationReverse process of extracting the original information signal from the modulated carrier.

C1.B — Three types of analog modulation

No.TypeShort description
1Amplitude Modulation (AM)Information is encoded by varying the amplitude of the carrier wave.
2Frequency Modulation (FM)Information is encoded by varying the frequency of the carrier wave.
3Phase Modulation (PM)Information is encoded by varying the phase of the carrier wave.

C2 — Basic switch & VLAN configuration (concise commands)

Example commands for switch S1 (adapt for S3 as needed):

enable
configure terminal
hostname S1

!-- EXEC (enable) password
S1(config)# enable password class

!-- Console password
S1(config)# line console 0
S1(config-line)# password cisco
S1(config-line)# login

!-- VTY password
S1(config)# line vty 0 4
S1(config-line)# password cisco
S1(config-line)# login

!-- Disable DNS lookup
S1(config)# no ip domain-lookup

!-- Create VLANs
S1(config)# vlan 10
S1(config-vlan)# name HR
S1(config)# vlan 20
S1(config-vlan)# name Sales
S1(config)# vlan 30
S1(config-vlan)# name Manager

!-- Assign ports to VLANs
S1(config)# interface range fa0/1-5
S1(config-if-range)# switchport mode access
S1(config-if-range)# switchport access vlan 10

!-- Configure trunking
S1(config)# interface fa0/24
S1(config-if)# switchport mode trunk

!-- Verify
S1# show vlan brief
