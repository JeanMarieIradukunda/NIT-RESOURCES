✅ SECTION A (40 Marks)

8. TRUE/FALSE (5 marks)

  • A. True – sudo apt install package_name installs software.
  • B. True – sudo apt update refreshes package lists.
  • C. False – sudo apt remove removes the package but keeps config files; use sudo apt purge to remove configs.
  • D. False – correct command is sudo apt upgrade (no hyphen).
  • E. False – apt-cache search is used to search, not remove software.

9. Copy, move, rename to final_report.txt (3 marks)

Assuming starting file: /home/user/documents/report.txt 

                                    
                                        cp /home/user/documents/report.txt /home/user/reports/

mv /home/user/reports/report.txt /home/user/reports/final_report.txt

10. chmod permissions (2 marks)

Owner: execute, Group: read, Others: no access

chmod 740 deploy.sh     # Numeric notation
chmod u=rwx,g=r--,o=--- deploy.sh   # Symbolic notation

11. TRUE/FALSE (4 marks)

  • A. True – rm deletes permanently.
  • B. True – rm -r removes directories recursively.
  • C. False – rm -rf / would destroy the system.
  • D. False – rmdir only removes empty directories.

✅ SECTION B (30 Marks)

18. File permissions question

a) Permissions (3 marks)

-rw-r-----

  • Owner (alice): rw- (read, write)
  • Group (managers): r-- (read)
  • Others: --- (no access)

b) Removing write from owner (2 marks)

Owner will not be able to edit or save changes to the file.

c) Give bob read-only access (2 marks)

                                    chmod o+r report.txt

d) Security advantage & risk (3 marks)

Advantage: Limits access to authorized users only.
Risk: May block legitimate users and reduce collaboration.

19. Prepare computer for Ubuntu installation (10 marks)

  1. Download Ubuntu ISO from ubuntu.com
  2. Insert a USB drive (≥4GB)
  3. Use Rufus (Windows) or balenaEtcher (cross-platform) to write the ISO
  4. Backup important data
  5. Reboot and enter BIOS/UEFI
  6. Enable USB boot and set USB as first boot device
  7. Save changes and reboot to the installer
  8. Follow on-screen installation steps

20. Fix “No bootable device found” (10 marks)

  • Check BIOS/UEFI boot order
  • Ensure correct disk mode (UEFI vs Legacy)
  • Repair GRUB from live USB: 
                                                
                                                sudo grub-install /dev/sda
sudo update-grub
  • Verify the disk is detected and Ubuntu installed
  • Use Boot Repair tool if necessary

21. Permission breakdown (7 marks)

a) Owner / Group / Others (2 marks)

Owner: sam → rwx
Group: devteam → r--
Others: r--

b) Explain -rwxr--r-- (3 marks)

`-` = regular file; `rwx` owner; `r--` group; `r--` others.

c) Can john execute the file? (2 marks)

No — john is in devteam and group perms are r--, so no execute.

d) Security implications of chmod a+x deploy.sh (3 marks)

Allows anyone to run the script — increased risk if the script is sensitive or destructive.

✅ SECTION C (15 Marks)

23. DNS Server Deployment

a) Hostname Configuration (3 marks)

                                    
                                        sudo hostnamectl set-hostname dns.training.local

# /etc/hostname
# dns.training.local

# /etc/hosts
127.0.0.1   dns.training.local

# verify
hostnamectl

b) BIND9 Installation & Setup (3 marks)

                                    
                                        sudo apt update
sudo apt install bind9 bind9utils bind9-doc

# in /etc/bind/named.conf.options add:
forwarders {
  8.8.8.8;
  8.8.4.4;
};

c) DNS Zone Configuration (4 marks)

named.conf.local 

                                    
                                        zone "training.local" {
  type master;
  file "/etc/bind/training.local.db";
};

zone "50.168.192.in-addr.arpa" {
  type master;
  file "/etc/bind/rev.50.168.192.db";
};

Forward zone file (sample) 

                                    
                                        $TTL 604800
@ IN SOA dns.training.local. admin.training.local. (
  2 ; Serial
  604800 ; Refresh
  86400 ; Retry
  2419200 ; Expire
  604800 ) ; Negative Cache TTL

@ IN NS dns.training.local.
dns IN A 192.168.50.10

Reverse zone (sample PTR) 

                                    10 IN PTR dns.training.local.

d) Validation and Testing (3 marks)

                                    
                                        sudo named-checkconf
sudo named-checkzone training.local /etc/bind/training.local.db
sudo systemctl restart bind9

# test
dig dns.training.local
dig -x 192.168.50.10

e) Documentation summary (2 marks)

Correct hostname, accurate zone files, and thorough testing ensure reliable name resolution, avoid client failures, and make maintenance predictable — essential before handing the server to the IT manager.

24. User/Group/Permission Configuration (10 marks)

a) Create users & group (3 marks)

                                    
                                        sudo useradd maria
sudo useradd kevin
sudo useradd josh

sudo groupadd alpha

sudo usermod -aG alpha maria
sudo usermod -aG alpha kevin
sudo usermod -aG alpha josh

b) Directory setup (3 marks)

                                    
                                        sudo mkdir -p /projects/alpha/
sudo chgrp alpha /projects/alpha/
sudo chmod 770 /projects/alpha/

c) Permissions (3 marks)

I. Directory (symbolic):

                                    chmod ug+rwx,o--- /projects/alpha/

II. Script files (octal):

                                    chmod 740 script.sh

III. Meaning: Read = view, Write = modify/create/delete, Execute = run file/enter directory.

d) Testing configuration (4 marks)

  1. Users run: cd /projects/alpha/ && touch testfile
  2. Check: ls -l /projects/alpha/
  3. Non-member test: sudo -u outsider ls /projects/alpha/ (should fail)
  4. Script exec test: owner can run ./script.sh ; group/others cannot.
