A growing organization is using Windows Server with Active Directory to improve security and manage users more easily. At the moment, users and computers are not well organized, and security settings are not consistent. The IT team wants to create a simple and secure system to manage users, computers, and access resources from one central place.

Based on the scenario above, explain clearly how each of the following Active Directory concepts can be applied to design an efficient and secure directory structure:
a) Domain
b) Domain Controller (DC)
c) Organizational Units (OUs)
d) Group Policy Object (GPO)
e) Security and Distribution Groups

Official Grading Solution Matrix:
  • a) Domain (2 Marks): Acts as the core logical administrative boundary. It groups all corporate nodes under a single DNS namespace (e.g., corp.local), providing centralized authentication databases and a uniform security scope.
  • b) Domain Controller (2 Marks): The physical or virtual server executing the Active Directory Domain Services (AD DS) role. It runs the KDC (Kerberos Key Distribution Center) engine to validate user identities and replicates the NTDS.dit database cluster-wide.
  • c) Organizational Units (2 Marks): Sub-containers within the domain used to mirror the physical departmental layout (e.g., HR, Finance). OUs allow administrators to delegate explicit control rights to localized tier engineers without assigning complete domain admin privileges, and act as targets for GPOs.
  • d) Group Policy Object (2 Marks): Configuration architectures used to enforce computational baselines and programmatic security constraints (such as password minimum profiles, account lockouts, or automated endpoint firewalls) across OUs uniformly.
  • e) Security and Distribution Groups (2 Marks):
    • Security Groups: System tokens used to assign access permissions across Shared folders, print resources, and network drives.
    • Distribution Groups: Collections reserved exclusively for mass email communication via mail environments; they hold no structural access control properties.

A network administrator is setting up a DHCP service to automatically assign IP addresses to computers in a new office network. To ensure the system works correctly, all configuration steps must be properly followed. Explain each of the following steps involved in setting up a DHCP service. DHCP Deployment Infrastructure Execution Stages (10 Marks):
  1. Role Installation via Server Manager: Provisioning the DHCP server architecture components into memory. This action constructs the core runtime daemons designed to listen for remote Layer 2 broad-broadcast DHCP request queries.
  2. DHCP Authorization in Active Directory: A security mechanism that registers the DHCP server's engine identity inside the AD configuration partition. This prevents rogue, un-authorized servers from handing out invalid IP sequences.
  3. Scope Creation and Address Pool Definition: Configuring the contiguous blocks of subnet IP strings allocated for automated end-node dynamic distribution (e.g., 192.168.10.100 to 192.168.10.200).
  4. Exclusion and Reservation Mapping:
    • Exclusions: Blocks out ranges kept safe from automated allocation (e.g., routers, structural domain components).
    • Reservations: Locks a single IP to a matching machine hardware MAC address, ensuring systems like local network printers maintain structural address stability.
  5. Configuration of Core DHCP Scope Options: Attaches necessary parameters to IP leases, including the Default Gateway (router address), DNS servers, and your corporate connection suffixes.
  6. Lease Duration Parameters: Controls how long a device owns an assigned address package before it must run renewal operations with the host server. Short leases recycle addresses quickly in high-turnover environments; long leases keep local networks stable.

Question 23
15 Marks

A medium-sized organization recently experienced a system failure due to a cyberattack, resulting in the loss of important business data, including customer records and financial information. The IT team discovered that backups were either outdated or not properly maintained, making it difficult to recover critical data quickly.

As a result, the organization has decided to improve its data protection strategy by implementing proper backup methods and a reliable recovery plan to prevent future data loss and ensure business continuity.

Based on the scenario,
a) Define the term backup in the context of server administration.
b) Explain five (5) types of backups commonly used in server environments.
c) Design a suitable backup and recovery plan for the organization. Your answer should clearly describe the steps that will ensure data can be restored efficiently after a failure.

Data Resiliency Framework Architecture Plan (15 Marks):

a) Backup Definition (2 Marks): The administrative practice of replicating data configurations and database stores from operational servers to separate physical or virtual media, creating an immutable fallback record if production nodes crash.

b) Five Common Server Backup Classifications (5 Marks):

  • Full Backup: Copies every single block of selected data within the environment, establishing a definitive baseline dataset. Highly comprehensive, but takes the longest time and uses the most storage.
  • Incremental Backup: Records only the data segments altered since the most recent backup of any kind. Very fast to run, but requires the complete chain of incremental files during a recovery process.
  • Differential Backup: Captures all data modifications made since the last Full backup baseline. Growth builds systematically, but recovery only requires the base Full file and the final differential block.
  • Synthetic Full Backup: Merges a baseline full backup with subsequent incremental files on the backup server to create a fresh full image, saving precious network bandwidth.
  • Bare-Metal Backup (BMB): Captures both storage records and operational state data (including OS layouts, system registry states, and boot setups), enabling complete hardware rebuilds from scratch.

c) Enterprise Backup & Recovery Plan Blueprint (8 Marks):

  1. Implementation of 3-2-1 Strategy: Keep 3 independent copies of company data across 2 different media types (e.g., local high-speed SSD SAN arrays and on-premise magnetic tape units), with 1 copy shipped completely offsite to an immutable cloud repository.
  2. Schedule Automation: Run a Full System image snapshot every Sunday during off-peak hours, with incremental updates processing automatically every 4 hours during standard weekdays.
  3. Encryption Enforcement: Ensure all data blocks are secured using AES-256 bit encryption algorithms both while traveling across internal network links and when sitting at rest on backup targets.
  4. RPO / RTO Parameters: Enforce strict business targets: a Recovery Point Objective (RPO) maximum limit of 4 hours of data variation, and a Recovery Time Objective (RTO) maximum cap of 2 hours for restoring critical line-of-business applications.
  5. Routine Drills: Run automated fire-drill recovery processes inside isolated network test environments every month to ensure system recovery scripts perform cleanly under stress.

Virtualization Infrastructure Blueprint Guide (15 Marks):

a) Virtual Machine Definition (2 Marks): A completely software-driven computer environment built by abstracting the physical hardware resources of a host computer via a hypervisor layer, operating independently as an isolated instance.

b) Hypervisor Application Options (2 Marks):

  • Oracle VM VirtualBox: A flexible, open-source Type-2 hypervisor widely used in desktop learning environments.
  • VMware Workstation Pro / Player: A highly optimized, professional-grade Type-2 virtualization application layer.
  • Microsoft Hyper-V: A native, Type-1/Type-2 bare-metal capable virtualization platform built directly into Windows operating systems.

c) Four Crucial Steps for VM Creation and Deployment (11 Marks):

  1. Host System Preparation and Feature Verification: Enter the physical machine's BIOS/UEFI firmware configuration screen. Enable hardware virtualization parameters explicitly (such as Intel VT-x or AMD-V extensions). Ensure the parent host machine has enough free unallocated RAM and CPU cores before provisioning new nodes. Download clean, official operating system installation media in standard `.iso` formats.
  2. Virtual Resource Provisioning & Profile Blueprinting: Launch your chosen hypervisor application and start the New Virtual Machine wizard. Define your target guest platform profile (e.g., Windows Server 2019). Allocate a specific chunk of system RAM (e.g., 4096MB) and assign virtual CPU processing units so you do not overload the parent host.
  3. Virtual Storage and Network Link Provisioning: Create a dynamically expanding virtual hard disk file (such as a `.vmdk` or `.vhdx` container) on the local drive with enough space allocated for guest system operations. Configure the guest machine's virtual network card properties—using **NAT mode** for standard outward internet routing paths, or **Bridged mode** to map the VM directly into the physical lab switch network.
  4. OS Installation and Integrated Tools Deployment: Mount the pre-downloaded `.iso` image to the virtual optical drive. Boot up the virtual machine to initiate standard operating system installation processes (such as drive formatting, timezone settings, and administrator credential creation). Once fully booted into the desktop, run the hypervisor integration tools utility package (e.g., VMware Tools or VirtualBox Guest Additions) to deploy high-performance video drivers and optimize shared mouse interactions.
