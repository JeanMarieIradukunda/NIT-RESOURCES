Question 15 • Protocol Sequences Section B • 10 Marks
The DHCP DORA Process & Transaction Sequence

Draw a clear sequence diagram illustrating the DHCP DORA process and describe each architectural phase.

DHCP Message Transaction Sequence:
Client Endpoint Container DHCP Server Instance | | | ----- 1. DHCPDISCOVER (Broadcast Link) --------> | | <---- 2. DHCPOFFER (Unicast/Broadcast Offer) -- | | ----- 3. DHCPREQUEST (Broadcast Selection) ----> | | <---- 4. DHCPACK (Finalized Lease Confirmation) | | |
1. DHCPDISCOVER: A new or unconfigured client broadcasts an unaddressed frame to locate available DHCP servers on its local segment.
2. DHCPOFFER: Servers intercepting the broadcast locate an unallocated address within their scope and offer it to the client, along with lease times, options, and masks.
3. DHCPREQUEST: The client broadcasts its choice to use that specific server's IP address. This tells all other servers to release their reserved offers back into their pools.
4. DHCPACK: The selected server commits the binding to its active lease database and sends a confirmation packet with parameters back to the client.
Question 16 • Directory Verification Section B • 10 Marks
Verifying Domain Reachability

Explain why verifying domain reachability from a client computer is an essential step before attempting a domain join. What problems arise if this is skipped?

Architectural Importance Matrix:

Verifying domain reachability ensures that underlying networking layers, routing tables, and DNS components can query and locate domain controller services. Without this verification, domain join workflows will fail because the client cannot find the required SRV records.

Potential Failures If Skipped:
  • Authentication Failures: User machines cannot validate credentials against active kerberos directory listings.
  • Group Policy Objects (GPO) Isolation: Workstations cannot retrieve security rules, system scripts, or corporate deployment policies.
  • Secure Channel Disruption: Mismatched machine account password sync windows can break the secure channel trust relationship completely.
Question 17 • Active Directory Join Section B • 10 Marks
Steps to Join a Client Computer to a Domain

Outline the core procedural workflow required to join a local computer container to an Active Directory domain.

Standard Active Directory Provisioning Process:
  1. IP and Name Resolution Alignment: Configure the client network adapter's primary DNS field to point directly to the domain controller's DNS server interface to enable proper domain SRV record lookups.
  2. Initiating Join Command: Access system properties, select corporate naming attributes, change the group assignment from standard Workgroup to Domain, and input the targeted FQDN (e.g., enterprise.local).
  3. Authentication Challenge: Provide administrative credentials or a delegated user profile authorized to add computer accounts to the domain tree.
  4. Object Provisioning & SID Exchange: Active Directory builds a new Computer Object instance, changes its Security Identifier (SID), and shares a unique machine account password with the endpoint.
  5. Host Reboot Execution: Restart the operating system to apply new security group objects and enable users to log in using domain credentials.
Question 18 • Cloud Provisioning Section B • 10 Marks
Cloud Service Model Evaluation Strategy

Evaluate the cloud service model appropriate for a business that wants to eliminate physical hardware management while retaining complete control over operating systems, databases, and applications.

Recommended Model Deployment: Infrastructure as a Service (IaaS)

Justification and Bounding Strategy:

  • Elimination of Physical Constraints: The upstream cloud provider abstracts and handles all physical operations, power delivery systems, hardware refreshes, and underlying hypervisor management layers.
  • Control Maintenance: Unlike PaaS or SaaS models, IaaS gives corporate sysadmins root and administrative control over virtual compute instances. The business manages operating system patches, custom database layouts, and network firewall policies.

Question 21 • Enterprise Engineering Implementation Section C • 15 Marks
Active Directory Domain Services (ADDS) Deployment Architecture Blueprint

Detail the precise installation procedure using Windows Server Manager to add the ADDS role, along with the corresponding post-installation configuration steps required to promote the server to an operational Domain Controller.

Part A: ADDS Installation Procedure (Server Manager Interface Pathway)
  • Prerequisites Verification Phase: Assign a static IP address to the system interface, verify local administrator credentials, and set an appropriate computer name.
  • Navigate to Server Manager, open the management drop-down menu on the dashboard, and select Add Roles and Features.
  • Set installation type parameters to Role-based or feature-based installation, then pick the targeted server container from the active pool list.
  • Locate the Server Roles checklist and check the box next to Active Directory Domain Services. Accept the automated popup prompt to add dependent management tools and snap-in components.
  • Progress through the subsequent Features and Information windows using the default settings, check the box to authorize automatic restarts if necessary, and click Install to initialize role installation.
Part B: Domain Controller Promotion & Post-Installation Plan

Promotion Lifecycle Steps:

  1. Click the alerting notification flag in Server Manager and select Promote this server to a domain controller.
  2. In Deployment Configuration settings, choose the option to Add a new forest and input the root domain name (e.g., school.internal or hq.local).
  3. Leave forest and domain functional levels at default values, ensure the Domain Name System (DNS) server box is checked, and create a strong Directory Services Restore Mode (DSRM) password.
  4. Verify NetBIOS naming matches, accept default directory path mappings for the Database, Log files, and SYSVOL file structures, pass the final requirements validation scan, and click Install.
