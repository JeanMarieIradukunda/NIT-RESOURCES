Question 01
3 Marks

A technician identifies the correct WLAN standard to solve a network setup problem and ensure proper connectivity. Read carefully the given questions and provide answers:

I. Which of the following IEEE standards defines wireless local area networks (WLAN)?

A. 802.3
B. 802.11
C. 802.15.1
D. 802.16

II. Which of the following device acts as a bridge between a home network and an Internet Service Provider (ISP)?

A. Access Point
B. Wireless Controller
C. Modem
D. Antenna

III. Which of the following options represents the typical signal range of a Wireless Personal-Area Network (WPAN)?

A. 100 meters
B. Up to 10 meters
C. 50 kilometers
D. Several kilometers
Verified Answer Key
  • I: B (IEEE 802.11 is the global standard baseline architecture for Wi-Fi implementation).
  • II: C (The Modulator-Demodulator/Modem translates digital network signals into ISP infrastructure frequencies).
  • III: B (WPAN environments like Bluetooth are restricted to low-energy personal spatial footprints, typically under 10 meters).

Question 07
3 Marks

A technician installing a wireless repeater explains correct placement, configuration via a web browser, and common default IP addresses to a trainee. State whether the following statements are True or False:

  • A. The best location for a wireless repeater is in a "dead zone" where there is no signal at all. FALSE
  • B. To configure a repeater, you usually need to access its configuration page via a web browser. TRUE
  • C. 192.168.0.1 and 192.168.1.1 are common default IP addresses used to access networking devices. TRUE

Rationale for A: A wireless repeater requires placement where it can capture a strong signal from the main source router (~50% footprint) to extend it efficiently. Placing it in a absolute dead zone means it has no signal to boost.

Question 18
10 Marks

What are the main differences between access points and wireless routers in terms of functionality, Routing, Connection, Management and Security?

Metric Parameter Wireless Access Point (AP) Wireless Router
Core Functionality Extends an existing wired Ethernet architecture into an RF wireless medium. Establishes, maintains, and manages local logical IP networks.
Routing Capabilities Operates primarily at Data Link Layer 2. No inter-subnet routing. Operates at Network Layer 3. Manages traffic pathways between LAN and WAN interfaces.
Connection Topology Requires an upstream physical router/switch backplane interface to operate. Connects directly to external ISP modems or fiber terminations.
Management Framework Often centralized via corporate software controller clusters or hardware units. Individually managed through localized embedded GUI/web configuration panels.
Embedded Security Handles pass-through client encryption schemes (e.g., WPA3-Enterprise). Features complex integrated SPI Firewalls, NAT, Port Forwarding, and Access Control Lists (ACLs).
Question 19
10 Marks

A modern smart hospital uses different wireless technologies to improve its operations. Patients wear sensors that continuously monitor and send health data to doctors in real time. Staff also use Bluetooth devices for nearby communication and Wi-Fi for internet access across the hospital...

A. Bluetooth vs. Wi-Fi: Bluetooth operates inside a WPAN footprint (<10m range) optimize for low-power point-to-point hardware syncs. Wi-Fi scales across full enterprise WLAN metrics (>50-100m zones) providing substantial bandwidth backplanes for concurrent database systems access.

B. Patient Monitoring Suitability: Medical telemetry thrives on real-time continuous vital tracking. Wireless systems allow patients to move around freely while tracking metrics via low-latency networks. This gives staff immediate warning alerts if a patient's vitals drop dangerously, without keeping them tied to a bed.

C. City vs. National Networks: City-wide systems (MAN/WiMAX/localized mesh nodes) focus on municipal coverage using compact pole-mounted repeaters. Nationwide telecom backplanes (4G/5G/LTE architectures) utilize large-scale tower cell sites, multi-tier core switching networks, and complex licensed frequency spectra to support country-wide roaming.

D. Wireless Connection Mechanics: Transmitting nodes convert logical application data bits into high-frequency RF oscillating currents using internal modulators. These currents are broadcast through an antenna as electromagnetic waves. Receiving antennas then pick up these waves and decode them back into binary code via a wireless NIC.

Question 20
10 Marks

A company in Kigali is facing several issues with its wireless network. A former employee still knows the Wi-Fi password, creating a security risk for unauthorized access. At the same time, the organization is using outdated operating systems, old router firmware, and a wireless monitoring system that cannot properly detect faults or support new devices...

A. Access Credentials and Preventive Maintenance: Regularly changing network passwords removes stale permissions, stops former employees from logging in, and protects the system from brute-force password tracking. This proactive security approach is essential to keep the network closed to unauthorized devices.

B. Software vs. Firmware Updates: Software patches update the host operating applications running on local user devices. Firmware updates rewrite the base microcode on the network router hardware. This improves low-level signal processing, patches system vulnerabilities, and adds support for newer connection standards.

C. Modern Monitoring Software Advantages: Updating to modern SNMPv3 network monitoring software gives the IT team real-time visibility into the network. It tracks live metrics, identifies signal loss points, and alerts technicians to configuration bugs or system overloads before they crash the entire network.

Question 21
10 Marks

An enterprise installation requires assessing default credentials risks, evaluating WPA2 versus WPA3 protocol enhancements, managing SSID visibility, analyzing faulty DHCP impacts, and validating baseline firmware steps.

A. Default Credentials: Keeping factory settings (like admin/admin) leaves the network open to easy takeover. Anyone can look up these defaults online, log into the console, and change system settings or intercept user data traffic.

B. WPA2 vs WPA3: WPA2 relies on a 4-way handshake that is vulnerable to offline dictionary attacks and tools like KRACK. WPA3 replaces this with Simultaneous Authentication of Equals (SAE). This provides forward secrecy, meaning it protects past traffic even if the network password is leaked, and blocks offline password-guessing tools.

C. SSID Broadcast Control: Turning off SSID broadcasting stops the router from announcing its name to nearby devices. While this hides the network from casual users, it does not stop hackers who can easily capture the network name using packet analyzers like Wireshark during device connections.

D. DHCP Fault Impact: A broken DHCP configuration can cause IP address conflicts or lock out users by failing to assign them network addresses. This cuts off communication between devices, stops internet access, and can take down network access across the entire office.

E. Pre-deployment Firmware Validation: Updating firmware before installing devices fixes known software bugs, closes security holes discovered after production, and ensures the new hardware works reliably with the rest of the company's network setup.

Question 22
10 Marks

A large university campus with multiple buildings is planning to install a wireless network to support thousands of students and staff. Decide between using a star topology or a mesh topology to ensure reliable coverage across classrooms, libraries, and outdoor areas. Justify your answer.

Recommended Solution: Hybrid Wireless Mesh Topology with Extended Star Core Base

Justification Matrix: For a university campus spanning extensive acreage and multiple physical masonry boundaries, a Wireless Mesh Architecture combined with individual building Star topology clusters provides maximum resilience. Mesh topologies offer self-healing routing mechanisms: if one outdoor intermediate node fails, alternative path nodes dynamic reroute traffic. This reduces expensive trench work for physical fiber runs across large outdoor common spaces while maintaining a seamless network connection for students moving across campus.

Question 23
15 Marks

Propose a complete wireless deployment workflow showing all key stages from planning to maintenance, and explain the main activities carried out in each stage.

Stage 1: Assessment & Planning
Gather business requirements, identify user density spikes, analyze floor plans, and choose suitable RF spectrum parameters (2.4GHz, 5GHz, or 6GHz bands).
Stage 2: RF Site Surveying
Run predictive software simulation models followed by on-site physical walk-throughs using spectrum analyzers. This identifies signal attenuators like concrete, brick walls, and elevator shafts, ensuring optimal placement for your access points.
Stage 3: Installation & Hardware Deployment
Mount the access points securely and run structured Cat6A Ethernet cables back to PoE+ network switches. Make sure all hardware is installed safely and complies with fire codes.
Stage 4: Logical Configuration & Security
Configure independent VLAN paths for staff and guests, set up corporate 802.1X enterprise authentication, customize DHCP scopes, and optimize channel widths to prevent interference.
Stage 5: Validation, Documentation & Maintenance
Test network performance by running active packet loss and throughput checks. Save the final topology designs and channel layouts, and schedule automated off-peak updates for router and AP firmware.
Question 24
15 Marks

Construct a troubleshooting plan for weak Wi-Fi signals in a secondary school environment experiencing frequent disconnections.

Official Technical Support Checklist Structure:
Phase I: Information Gathering & Baseline Metrics

Interview affected users in the library/classrooms. Use an RF analyzer app to check Received Signal Strength Indicators (RSSI). A reading below -75 dBm explains why devices are experiencing dropped connections.

Phase II: Diagnostic Isolation Procedures

Check the network console to see if too many devices are crowding a single access point. Run channel spectrum scans to check for interference from electronics like microwaves or nearby outside networks.

Phase III: Remediation Execution Steps

Adjust access point positions to clear physical obstacles. Turn down transmission power on overlapping units to fix sticky-client issues, and switch devices to clean, non-overlapping channels (like 1, 6, or 11 on the 2.4 GHz band).

Phase IV: Validation & Verification Signing

Run continuous ping tests to check for data loss, confirm devices can connect smoothly throughout classrooms, and monitor the update logs over the next week to ensure the fixes hold.
