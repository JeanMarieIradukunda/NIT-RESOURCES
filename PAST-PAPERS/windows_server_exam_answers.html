D. OPEN QUESTIONS (35 Marks)

1. AD Key Terms (5 Marks)

  • AD (Active Directory): A Microsoft directory service that stores information about users, computers, groups, and other resources, and helps manage network authentication and authorization.
  • DC (Domain Controller): A server that hosts Active Directory Domain Services (ADDS), authenticates users, enforces security policies, and manages access to network resources.
  • OU (Organizational Unit): A container within Active Directory used to organize users, computers, groups, and other objects, allowing administrators to apply policies and delegate control.
  • GPO (Group Policy Object): A set of rules and configurations used to control and manage user and computer settings across the domain—such as password policies, desktop configuration, and software restrictions.
  • Global Catalog (GC): A distributed data repository stored on domain controllers that provides fast searches for objects across the entire forest and helps in user logon by validating universal group membership.

2. ADDS Deployment (6 Marks)

  • Add Domain Controller: Installs an additional domain controller in an existing domain to increase redundancy, load balancing, and improve authentication performance.
  • Add New Domain: Creates a new domain within an existing forest. This is used when organizations need separate administrative boundaries or different security policies.
  • Add New Forest: Creates a completely new Active Directory forest. This is chosen when no existing AD infrastructure is available or when a fully independent directory structure is required.

3. Reasons for Server Promotion (5 Marks)

  • Central management: Promoting a server allows it to manage users, computers, and resources centrally within the domain, simplifying administration.
  • Authentication: The promoted server can authenticate users and devices, ensuring that only authorized accounts can access network resources.
  • Policies: The server can enforce Group Policy Objects (GPOs) to control user and computer settings across the domain.
  • Replication: Promoting a server ensures that directory data is replicated between domain controllers, maintaining consistency and redundancy.
  • Security: A promoted server can apply security settings, monitor access, and provide a secure environment for domain resources.

4. Server Manager vs Control Panel (4 Marks)

  • Server Manager: A Windows Server tool used to install, configure, and manage server roles and features, monitor server performance, and perform administrative tasks across multiple servers.
  • Control Panel: A Windows interface for configuring system settings on a single machine, such as network settings, hardware, user accounts, and basic system configurations. It is not used for domain-wide server management.

5. Windows Server Installation (8 Marks)

  • A. Types:
    • Full GUI: A Windows Server installation with a graphical user interface, allowing easy management using GUI-based tools and applications. Suitable for beginners or environments that require visual management.
    • Server Core: A minimal installation without a graphical interface, optimized for performance and security. Managed mainly through command-line tools, PowerShell, or remote administration.
  • B. Static IP: Assigning a fixed IP address to the server to ensure reliable network communication, access to services, and proper integration within the domain environment.
  • C. Ping Test: Using the ping command to check network connectivity between the server and other devices, ensuring the server can communicate properly on the network.

6. GPO Definition and Types (5 Marks)

GPO (Group Policy Object): A set of rules and configurations used in Active Directory to control the behavior and security of users and computers within a domain. GPOs help enforce consistent settings across multiple devices.

  • Local GPO: Applies only to the individual computer on which it is configured. Used for managing settings on a single machine without affecting other computers in the domain.
  • Domain GPO: Applied to users and computers within an Active Directory domain. It allows centralized management of security policies, software installation, and user environment settings across multiple machines.
  • Starter GPO: A template that contains predefined policy settings. It can be used to quickly create new GPOs with consistent configurations, reducing setup time and ensuring standardization.

7. Role vs Feature (2 Marks)

  • Role: A primary function or service provided by a Windows Server, such as DNS Server, DHCP Server, or Active Directory Domain Services. Roles define what the server does in the network.
  • Feature: An optional component that supports server roles or enhances functionality, such as .NET Framework, Failover Clustering, or Telnet Client. Features are not essential for the server’s main purpose but add additional capabilities.
SECTION B — Scenario-Based Questions (15 Marks)

Scenario 1: Physical Server Installation

  • Prepare ISO media: Obtain or download the Windows Server ISO file and create bootable media (USB/DVD) to start the installation.
  • Choose GUI/Core: Decide between a Full GUI installation for graphical management or Server Core for minimal, command-line-based setup.
  • Connect hardware: Ensure the server hardware (CPU, RAM, storage, NIC) is properly installed and connected to power and network.
  • Assign IP: Configure a static IP address to ensure stable network communication and domain connectivity.
  • Test connectivity: Use commands like ping to verify network communication with other devices or servers.

Scenario 2: Virtual Server Installation

  • Mount ISO: Attach the Windows Server ISO to the virtual machine’s virtual CD/DVD drive for installation.
  • Create VM: Configure the virtual machine with appropriate CPU cores, memory, and storage resources for optimal performance.
  • Setup vNIC: Configure the virtual network interface card to provide network connectivity to the VM.
  • Assign IP: Set a static or DHCP-assigned IP to ensure the VM can communicate with the network and domain.
  • Test ping: Use the ping command to verify connectivity between the VM and other devices or servers in the network.
