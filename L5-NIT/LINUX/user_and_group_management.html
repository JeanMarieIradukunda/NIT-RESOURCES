🧑💻 User and Group Management in Ubuntu Linux

Understand User Creation, Group Management, Permissions, and File Ownership

1. User Management in Ubuntu Linux

Ubuntu uses the Linux user management system based on configuration files like /etc/passwd, /etc/shadow, and /etc/group.

🧩 User Attributes

AttributeDescriptionConfiguration Location / Command
Login NameUnique username identifying the user.sudo adduser username
Or manually edit /etc/passwd.
Encrypted PasswordStored securely using hash algorithms.passwd username — updates password in /etc/shadow.
UID (User ID)Numeric identifier for each user.
UID 0 = root; UID ≥1000 = normal users.		Found in /etc/passwd.
GID (Group ID)Primary group ID for user.Stored in /etc/group and /etc/passwd.
User’s Default ShellAssigned command-line interface (e.g. /bin/bash).sudo useradd -s /bin/bash username
Home DirectoryUser’s personal workspace./home/username, created using -m flag.

2. Group Management in Ubuntu Linux

Groups help organize users and manage shared permissions. A user can belong to one primary and multiple secondary groups.

🧩 Group Attributes

AttributeDescriptionCommand / Location
Group NameUnique group identifier.sudo groupadd groupname
GID (Group ID)Numeric identifier for the group.Stored in /etc/group.
Group MembersUsers within the group.getent group groupname
Group PasswordOptional security layer.gpasswd groupname
Group OwnershipFile/directory group ownership.chown :groupname filename
Group PermissionsDefines what the group can do (rwx).Controlled by ls -l permissions.
Group ExpirationDefines when user/group expires (optional).chage -E for users.

4. Understanding File and Directory Permissions

Every file in Ubuntu is protected by permissions defining who can read, write, or execute it. These ensure security and controlled collaboration.

👥 Entities (Ownership Classes)

ClassDescription
Owner (User)Creator or file owner.
GroupUsers sharing access rights.
OthersEveryone else on the system.

🔐 Types of Permissions

SymbolMeaningValueDescription
rRead4View content / list directory.
wWrite2Modify or delete files.
xExecute1Run files or enter directories.
Example Output:
-rw-r--r-- 1 john developers 120 Nov 4 13:45 file1.txt

Breakdown:
• - → Regular file
• rw- → Owner read/write
• r-- → Group read only
• r-- → Others read only

5. Setting Permissions

🧩 Symbolic Notation

TaskCommandResult
Add execute to ownerchmod u+x script.shOwner can execute.
Remove read from groupchmod g-r file.txtGroup cannot read.
Give read/write to allchmod a+rw file.txtEveryone can modify.
Set exact permissionschmod u=rw,g=r,o= file.txtOwner rw, group r, others none.

🧮 Octal (Numeric) Notation

OctalSymbolicDescription
777rwxrwxrwxEveryone full access.
755rwxr-xr-xOwner full, others read/execute.
644rw-r--r--Owner read/write, others read only.
700rwx------Only owner has access.
Use chmod 755 /var/www for typical directory permissions.

6. Permission Testing & Directory Rules

🔧 Viewing and Testing

TaskCommand
View permissionsls -l
Change ownershipsudo chown username:group file
Switch usersu - otheruser
Detailed infostat filename

🧱 Directory Permissions

PermissionEffect
rList contents with ls.
wCreate, rename, or delete inside directory.
xEnter directory using cd.

🌐 Practical Scenario

sudo mkdir /projects
sudo groupadd devteam
sudo chown :devteam /projects
sudo chmod 2775 /projects

🧩 The 2 in 2775 sets the setgid bit so new files inherit the group.
