Disk Partitioning in Linux — Ubuntu Server

Clear, practical notes with examples and command breakdowns.

What is Disk Partitioning?

Disk Partitioning is the process of dividing a physical hard disk into logical sections, called partitions, so each partition can be managed separately. Think of it as slicing a cake — each slice holds a different purpose.

Why Partition a Disk?

  • Organize data — separate system files, user files, and backups.
  • Security and stability — one partition filling won't affect the others.
  • Multiple OS installations — each OS can live on its own partition.
  • Performance — reduces fragmentation and can improve efficiency.

Common Workflow (Ubuntu Server)

Ubuntu Server typically uses the command line for partitioning. Main steps shown below with commands and explanations.

1. lsblk — List Block Devices

Purpose: View disks and partitions connected to the server.

lsblk

# Output example:
# NAME   MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
# sda      8:0    0  100G  0 disk
# ├─sda1   8:1    0   50G  0 part /
# ├─sda2   8:2    0   30G  0 part /home
# └─sda3   8:3    0   20G  0 part [SWAP]

Breakdown:

  • NAME — Disk or partition name
  • SIZE — Size of disk/partition
  • TYPE — disk or part
  • MOUNTPOINT — Where the partition is mounted

2. fdisk — Partition a Disk

Purpose: Create, delete, or modify partitions on a disk.

sudo fdisk /dev/sdb

Interactive commands inside fdisk:

  • m — show help
  • p — print partition table
  • n — create new partition
  • d — delete partition
  • t — change partition type (e.g., 82 for swap)
  • w — write changes to disk and exit
  • q — quit without saving

Example steps:

  1. sudo fdisk /dev/sdb → enter fdisk.
  2. Press n → create new partition.
  3. Choose primary (p) or extended (e).
  4. Enter partition number, first sector, and last sector (or size).
  5. Use t to change type if needed (e.g., 82 for swap).
  6. Use w to save changes.

3. mkfs — Format the Partition

Purpose: Create a filesystem on the partition so it can store files.

sudo mkfs.ext4 /dev/sdb1

Breakdown:

  • mkfs — make filesystem
  • ext4 — filesystem type (default, fast, reliable)
  • /dev/sdb1 — the partition to format

Other filesystem options: ext3, xfs, vfat.

4. mkswap — Create Swap Partition

Purpose: Format a partition as swap space (virtual memory).

sudo mkswap /dev/sdb2
sudo swapon /dev/sdb2

Breakdown:

  • mkswap — initialize partition for swap
  • swapon — enable the swap immediately

What is /dev/sdb2?

It is the name of a partition on the system. Breakdown:

  • /dev/ — device directory (hardware represented as files).
  • sd — SCSI disk naming (used for SATA/SSD/USB too).
  • b — second disk detected (a = first, b = second).
  • 2 — second partition on that disk.
# So 
sudo mkswap /dev/sdb2   # make it a swap area
sudo swapon /dev/sdb2   # enable it so the system uses the swap

💡 Example scenario (100GB disk /dev/sdb):

  • /dev/sdb1 → 80GB for data (ext4)
  • /dev/sdb2 → 20GB for swap

Check Active Swap

sudo swapon --show
# or
free -h

5. mount — Mount a Partition

Purpose: Attach the partition to the filesystem so it becomes accessible.

sudo mount /dev/sdb1 /mnt

To make permanent, add to /etc/fstab:

/dev/sdb1   /mnt   ext4   defaults   0   2

6. parted — Advanced Partitioning

Purpose: Modern tool with GPT support; useful for large disks and scripting.

sudo parted /dev/sdb
(parted) mklabel gpt
(parted) mkpart primary ext4 1MiB 100%
(parted) quit

Breakdown:

  • mklabel gpt — create GPT partition table
  • mkpart — create a partition (start to end)

Summary — Quick Steps

  1. Check disks: lsblk
  2. Create partitions: fdisk or parted
  3. Format partitions: mkfs.ext4
  4. Setup swap (if needed): mkswap + swapon
  5. Mount partitions: mount /dev/sdX /mount_point
  6. Make mounts permanent: edit /etc/fstab

💡 Typical partitions for a new Ubuntu Server:

  • / — root filesystem
  • /home — user data
  • swap — virtual memory

Quick Commands

lsblk
sudo fdisk /dev/sdb
sudo mkfs.ext4 /dev/sdb1
sudo mkswap /dev/sdb2
sudo swapon /dev/sdb2
sudo mount /dev/sdb1 /mnt
sudo parted /dev/sdb

Pro tip: Use lsblk -f to show filesystem types and UUIDs for /etc/fstab entries.
