📁 File Server Services

1. Configuration of File Server Service

A File Server stores and manages files so multiple users on a network can access, share, and store data efficiently.

Analogy: Like a school library: students (users) come to read, borrow, or return books (files).

2. Description of File Sharing Protocols

SMB/CIFS

Definition: Protocol mainly in Windows to share files, printers, and resources. Linux uses it via Samba.

Analogy: Classroom printer shared among students.

Example in School: Teacher shares assignment templates on a central PC.

NFS (Network File System)

Definition: Linux/Unix systems access remote files as if local.

Analogy: Shared locker in a school corridor.

Example in College: Lab PCs mount server directory via NFS, files appear local.

FTP

Definition: Protocol for uploading/downloading files over TCP/IP.

Analogy: Mail service sending/receiving packages.

Example in School: Teachers upload lecture notes to a server; students download from anywhere.

3. Installation of File Server Software

SMB/CIFS (Samba on Ubuntu)
sudo apt update
sudo apt install samba

Usage: Share files with Windows/Linux clients. Example: Share textbooks with staff.

NFS
sudo apt update
sudo apt install nfs-kernel-server

Usage: Linux-to-Linux sharing. Example: Lab assignments access.

FTP
sudo apt update
sudo apt install vsftpd

Usage: Upload/download files. Example: Teachers upload lesson materials; students download.

4. Perform FTP Server Configuration

sudo nano /etc/vsftpd.conf

Enable local users and write permissions:

local_enable=YES
write_enable=YES

Restart FTP service:

sudo systemctl restart vsftpd

Students can now log in with username/password to access shared notes.

5. Set Permissions and Access Control

Permissions ensure users access only what they are allowed:

  • Read (r): View files
  • Write (w): Modify files
  • Execute (x): Run files or scripts
sudo chown teacher:staff /srv/samba
sudo chmod 770 /srv/samba

Analogy: Only students in a class can open their locker; others cannot.

Example: Teachers edit assignments; students only view.

6. Apply Backup and Recovery Techniques

Definition: Regularly saving copies of data to prevent loss due to hardware failure, mistakes, or attacks.

  • rsync for incremental backups
  • tar for archive backups
  • Automated cron jobs
rsync -av /srv/samba /mnt/backup/

Analogy: Weekly photocopies of library books.

Example: All student assignment submissions backed up daily to a separate server.

🔧 FTP Server Configuration Example: NIT-L5-RESOURCES (Teacher)

🔧 Step-by-Step FTP Setup for 1 Teacher + 90 Students

Scenario: Teacher uploads notes; students download (read-only). Shared folder: /srv/ftp/NIT-L5-RESOURCES

sudo apt update
sudo apt upgrade -y

Ensure system is up-to-date.

sudo apt install vsftpd -y

vsftpd: Very Secure FTP Daemon.

sudo adduser teacher

Assign password: teacher123. Teacher uploads and manages notes.

for i in $(seq 1 90); do
    sudo adduser --disabled-password --gecos "" student$i
    echo "student$i:password123" | sudo chpasswd
done

Each student gets a unique account to log in and download notes.

sudo groupadd nit_students

Single group to control all students’ permissions.

for i in $(seq 1 90); do
    sudo usermod -aG nit_students student$i
done

Append all students to nit_students group.

sudo mkdir -p /srv/ftp/NIT-L5-RESOURCES

Central folder for teacher and students.

sudo chown teacher:nit_students /srv/ftp/NIT-L5-RESOURCES
sudo chmod 750 /srv/ftp/NIT-L5-RESOURCES

Teacher full control; students read-only. Optional 770 if students upload files.

sudo nano /etc/vsftpd.conf
listen=YES
anonymous_enable=NO
local_enable=YES
write_enable=YES
chroot_local_user=YES

Restrict users, allow teacher to upload notes.

for i in $(seq 1 90); do
    sudo usermod -d /srv/ftp/NIT-L5-RESOURCES student$i
done

Students land directly in shared folder upon login.

sudo systemctl restart vsftpd
sudo systemctl enable vsftpd

Apply configuration changes; enable service on boot.

Teacher uploads a note:

ftp <server-ip>
Username: teacher
Password: teacher123
cd NIT-L5-RESOURCES
put lecture1.pdf

Student downloads:

ftp <server-ip>
Username: student1
Password: password123
cd NIT-L5-RESOURCES
get lecture1.pdf
