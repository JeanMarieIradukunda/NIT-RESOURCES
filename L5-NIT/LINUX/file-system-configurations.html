Linux System Configuration

Linux system configuration refers to the process of setting up and customizing a Linux operating system to meet user, application, or system requirements. In simpler terms, it’s like tuning your Linux computer so it works the way you want it to.

Key Aspects of Linux System Configuration

  • Network Configuration: Setting up IP addresses (static or dynamic), DNS, gateways, and network interfaces.
  • User and Group Management: Adding, deleting, or modifying users and groups, managing permissions and access rights.
  • Service and Daemon Management: Starting, stopping, enabling, or disabling system services (like SSH, Apache, MySQL).
  • System Files Configuration: Editing configuration files located in /etc/ (e.g., /etc/hosts, /etc/fstab).
  • Hardware Configuration: Setting up printers, disks, storage devices, and peripherals.
  • Software Installation and Management: Installing, updating, or removing software using package managers like apt, yum, or dnf.
  • Security Configuration: Configuring firewalls, SELinux, AppArmor, and user permissions.
  • Kernel and Boot Configuration: Adjusting kernel parameters, bootloader settings (GRUB), and startup processes.
    GRUB (GNU GRand Unified Bootloader): GRUB is the default bootloader used by most Linux distributions. It loads the OS kernel and presents a boot menu for selecting operating systems or kernel versions.
    Key Functions: Boot management (dual-boot selection), kernel loading, command line interface for advanced boot commands, configuration handling (/boot/grub/grub.cfg), and recovery options.
    Main Configuration Files: /boot/grub/grub.cfg (main menu), /etc/default/grub (user settings like timeout and default entry), and /etc/grub.d/ (scripts to generate grub.cfg).
    Common Commands: sudo update-grub, sudo grub-install /dev/sdX, sudo nano /etc/default/grub.
    Boot Process: BIOS/UEFI initializes → GRUB loads → kernel and initrd loaded → root filesystem mounted → init/systemd starts services → login screen appears.

Example: Basic Linux System Configuration on Ubuntu Server

Step 1: Set a Static IP Address

Open the netplan configuration file (Ubuntu 18.04+ uses Netplan):

sudo nano /etc/netplan/00-installer-config.yaml

Example configuration:

network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:
      dhcp4: no
      addresses:
        - 192.168.1.100/24
      gateway4: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]

Apply configuration:

sudo netplan apply

Step 2: Create a New User

sudo adduser john
sudo usermod -aG sudo john

Step 3: Enable SSH Access

sudo apt update
sudo apt install openssh-server -y
sudo systemctl enable ssh
sudo systemctl start ssh
sudo systemctl status ssh

Step 4: Configure Firewall (UFW)

sudo ufw allow OpenSSH
sudo ufw enable
sudo ufw status

Step 5: Update System Packages

sudo apt update
sudo apt upgrade -y
✅ Result of Configuration:
  • Your server has a static IP.
  • A new user with sudo rights can log in.
  • SSH is enabled for remote access.
  • Firewall is active and configured.
  • System packages are updated.

Server Login

Server login is the process of accessing a server using a username and password or other authentication methods to manage, run commands, or use its resources.

Key Points

  • Credentials: Username, password, sometimes SSH keys for secure login.
  • Protocols: SSH (Linux), RDP (Windows), Telnet (rare), web-based logins (cPanel).
  • Purpose: Manage server settings, upload/download files, run applications, monitor logs.

Real-world Example on Ubuntu Server

ssh username@192.168.1.10

Enter your password when prompted. Once authenticated, you can run commands on the server.

💡 Tip: Using SSH keys instead of passwords is more secure and widely recommended.
