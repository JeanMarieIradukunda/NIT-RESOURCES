1. Definition of a Web Server
A web server is software (and the hardware it runs on) that stores, processes, and delivers web content (HTML pages, images, APIs, etc.) to clients (browsers) over the internet or a local network using protocols such as HTTP and HTTPS.
Client
A client is the user or device (usually a web browser) that requests web content from a server.
Server
A server is a computer or system that stores web files and provides services to clients over a network.
Web Server
A web server is software (and sometimes hardware) that receives requests from clients and sends back web pages using HTTP or HTTPS.
Apache HTTP Server
Apache is a process-based web server that is highly configurable and widely used for hosting websites.
Nginx
Nginx is an event-driven web server designed for high performance and handling many users at the same time.
HTTP
HTTP (HyperText Transfer Protocol) is the communication rule used to transfer web pages between client and server.
HTTPS
HTTPS is the secure version of HTTP that encrypts data using SSL/TLS certificates.
Web Page
A web page is a document (HTML) that is displayed in a web browser.
Website
A website is a collection of related web pages hosted on a web server.
Port
A port is a number used by a web server to listen for client requests (e.g., port 80 for HTTP, 443 for HTTPS).

2. Types of Web Servers for Linux

2.1 Apache HTTP Server
Apache HTTP Server (often just called Apache) is a free, open-source web server software used to deliver websites and web applications over the internet.

Apache runs on the server and sits between the client (user’s browser) and the server’s resources.
Simple flow:
Apache HTTP Server is a program that runs on a server and responds to requests from web browsers by sending web pages using the HTTP/HTTPS protocols.

Key features:
•	🌍 Most widely used web server in the world
•	🔓 Open source and free
•	🧩 Modular design – features can be added or removed using modules
•	🖥️ Runs on Linux, Windows, and macOS
•	🔐 Supports SSL/TLS (HTTPS) for secure communication
How Apache works:
1.	A user types a website address in a browser
2.	The browser sends an HTTP request to the server
3.	Apache processes the request
4.	Apache sends back the requested web page
Common uses:
•	Hosting websites
•	Running web applications (PHP, Python, etc.)
•	Shared hosting environments
•	Internal company servers
Advantages:
•	Very stable and reliable
•	Easy to configure
•	Large community and documentation

2.2 Nginx (Engine-X)
A Nginx server (pronounced “engine-x”) is a high-performance web server software used to deliver websites and web applications to users over the internet.
Simple definition:
Nginx is a web server that handles client requests and serves web content using HTTP and HTTPS, optimized for speed and scalability.
What Nginx does:
•	Receives requests from clients (browsers)
•	Serves static files (HTML, CSS, images)
•	Passes dynamic requests to application servers
•	Sends responses back to clients
Key features:
•	⚡ Very fast and efficient
•	🔁 Handles many simultaneous connections
•	🧠 Uses an event-driven architecture (low memory usage)
•	🔐 Supports SSL/TLS (HTTPS)
•	🔄 Works as a reverse proxy, load balancer, and cache
Common uses:
•	Hosting high-traffic websites
•	Reverse proxy in front of Apache
•	Load balancing multiple servers
•	Serving static content efficiently
Where Nginx stands:
Client (Browser)
        ↓
Nginx Server
        ↓
Web files / Application servers
Comparison (quick):
•	Apache → flexible, module-based
•	Nginx → fast, scalable, lightweight
Main difference 
Apache uses a process/thread-based model, while Nginx uses an event-driven model.
👉 This means:
•	Apache creates a new process or thread for each request (uses more memory)
•	Nginx handles many requests with fewer resources (more efficient)
Which is better to use? (single line):
Nginx is better for high-traffic and performance, while Apache is better for flexibility and ease of configuration.

3. Web Server Configuration Files

Apache Configuration Files
•	Main configuration: /etc/apache2/apache2.conf
•	Virtual hosts: /etc/apache2/sites-available/
•	Enabled sites: /etc/apache2/sites-enabled/
•	Ports: /etc/apache2/ports.conf
/etc/apache2/apache2.conf
This is the main Apache configuration file.
It controls the overall behavior of the Apache server, such as security settings, default directories, and global options. Other configuration files are included from here.
________________________________________
/etc/apache2/sites-available/
This directory contains all virtual host configuration files.
Each file defines how Apache should serve a specific website (domain name, document root, ports, etc.).
Sites placed here are not active by default.
________________________________________
/etc/apache2/sites-enabled/
This directory contains enabled (active) websites.
Apache only reads and serves sites that appear here.
Files in this folder are usually symbolic links pointing to files in sites-available.
________________________________________
/etc/apache2/ports.conf
This file defines the ports Apache listens on, such as:
•	Port 80 for HTTP
•	Port 443 for HTTPS
It tells Apache where to accept incoming connections.

4. Installation of Apache HTTP Server

Step 1: Update the system
sudo apt update && sudo apt upgrade -y
Step 2: Install Apache
sudo apt install apache2 -y
Step 3: Start and enable Apache
sudo systemctl start apache2
sudo systemctl enable apache2
________________________________________

5. Installation of Nginx

Step 1: Install Nginx
sudo apt install nginx -y
Step 2: Start and enable Nginx
sudo systemctl start nginx
sudo systemctl enable nginx
________________________________________

6. Firewall Configuration (UFW)

Allow Apache or Nginx traffic
For Apache
sudo ufw allow 'Apache Full'
For Nginx
sudo ufw allow 'Nginx Full'
Enable firewall
sudo ufw enable
sudo ufw status

✅ Perform Security on Web Server

1. Permissions
Permissions control who can read, write, or execute web files.
For Apache web files, the recommended settings are:
sudo chown -R www-data:www-data /var/www/html
sudo chmod -R 755 /var/www/html
•	chown → sets ownership to the Apache user (www-data)
•	chmod 755 → allows:
o	Owner to read/write/execute
o	Others to read and execute only
This prevents unauthorized users from changing website files.
________________________________________
2. Encryption
Encryption ensures data between the client and server is protected.
Without encryption:
•	Data travels in plain text
•	Can be intercepted by hackers
With encryption (HTTPS):
•	Data is encrypted using SSL/TLS
•	Protects login details, forms, and sensitive information
________________________________________
3. SSL Certificate
An SSL certificate allows your website to use HTTPS.
SSL/TLS is the technology that makes HTTPS secure.
________________________________________
✅ Meaning:
•	SSL = Secure Sockets Layer
•	TLS = Transport Layer Security
👉 TLS is the newer and more secure version of SSL, but people often still call it SSL.
________________________________________
🔒 What SSL/TLS does in HTTPS:
1.	Encrypts data between the user and the web server
2.	Prevents hackers from reading sensitive information (like passwords or credit card details)
3.	Verifies the identity of the website using an SSL certificate
________________________________________
🔁 How it works (simple steps):
1.	Browser connects to website using HTTPS
2.	Server sends its SSL/TLS certificate
3.	Browser checks the certificate is valid
4.	A secure connection is created
5.	Data is encrypted and transferred safely
________________________________________
🚦 Result:
HTTPS = HTTP + SSL/TLS
This is why the website shows:
•	🔒 Padlock icon
•	https:// in the address bar

To install SSL using Let’s Encrypt:
For Apache:
sudo apt install certbot python3-certbot-apache -y
sudo certbot --apache
For Nginx:
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx
________________________________________
🔁 Auto-renew SSL
SSL certificates renew automatically every 90 days. Test renewal:
sudo certbot renew --dry-run
