vsftpd Configuration & FTP Modes

Clear, practical and student-friendly Linux FTP notes

MOST-NEEDED & COMMON vsftpd.conf LINES

# Listen for FTP connections on IPv4
listen=YES

# Disable IPv6 listening
listen_ipv6=NO

# Disable anonymous access
anonymous_enable=NO

# Allow local users
local_enable=YES

# Enable upload, delete, rename
write_enable=YES

# Default permissions (rw-rw-r--)
local_umask=002

# Restrict users to their home directory
chroot_local_user=YES

# Allow writable chroot
allow_writeable_chroot=YES

# Shared directory
local_root=/srv/ftp/shared

# Passive mode
pasv_enable=YES
pasv_min_port=30000
pasv_max_port=31000

# Logging & performance
xferlog_enable=YES
log_ftp_protocol=YES
use_localtime=YES
reverse_lookup_enable=NO

# Security & limits
hide_ids=YES
max_clients=50
max_per_ip=5

WHY THESE LINES ARE IMPORTANT

Feature Purpose
local_enableAllows Linux users to login
write_enableEnables upload & delete
chroot_local_userImproves security
local_rootShared directory
pasv_*Required for FileZilla

Required Directory Setup

sudo mkdir -p /srv/ftp/shared
sudo chown root:ftpusers /srv/ftp/shared
sudo chmod 2775 /srv/ftp/shared

Restart vsftpd Service

sudo systemctl restart vsftpd

Best Practice Tips

  • ✔ Use FTP groups for shared access
  • ✔ Always enable Passive Mode
  • ✔ Disable anonymous access in production
  • ✔ Use FTPS for secure environments

FTP Modes Explained

FTP modes define how data connections are created between the client and server.

1️⃣ Active FTP Mode
  • Server connects back to the client
  • Uses port 20
  • Fast but firewall-unfriendly
2️⃣ Passive FTP Mode (Recommended)
  • Client connects to server
  • Works behind NAT & firewalls
  • Used by FileZilla
