Dynamic Host Configuration Protocol

1. Definition, Purpose & Application

Definition

DHCP (Dynamic Host Configuration Protocol) is a network service that automatically assigns IP addresses and other network configuration parameters to client devices.

Purpose
  • IP address
  • Subnet mask
  • Default gateway
  • DNS servers
  • Lease time
When to Use DHCP
  • Medium to large networks
  • Frequent device connections
  • Centralized IP management
  • Avoiding IP conflicts
  • Simplifying administration
Examples
Corporate LANs Schools Home Networks Virtual Machines

2. DHCP Handshake (DORA)

  1. Discover – Client broadcasts DHCPDISCOVER
  2. Offer – Server responds with DHCPOFFER
  3. Request – Client sends DHCPREQUEST
  4. Acknowledge – Server confirms with DHCPACK
📌 Result: Client receives a valid IP configuration

3. Installing DHCP Server

sudo apt update
sudo apt install isc-dhcp-server -y
dhcpd --version

4. DHCP Configuration Files

File Purpose
/etc/dhcp/dhcpd.conf Main DHCP configuration
/etc/default/isc-dhcp-server Interface selection
/var/lib/dhcp/dhcpd.leases Stores active and expired DHCP leases. A DHCP lease is a temporary assignment of an IP address to a client for a specific lease time, after which the client must renew the address or receive a new one.
Interface Configuration
INTERFACESv4="eth0"

5. DHCP Server Parameters

default-lease-time 600;
max-lease-time 7200;

subnet 192.168.1.0 netmask 255.255.255.0 {
    range 192.168.1.100 192.168.1.200;
    option routers 192.168.1.1;
    option subnet-mask 255.255.255.0;
    option domain-name-servers 8.8.8.8, 8.8.4.4;
    option domain-name "example.local";
}
Parameter Description
default-lease-timeDefault lease duration
max-lease-timeMaximum lease duration
rangeIP pool for clients
option routersDefault gateway
option domain-name-serversDNS servers

cat /var/lib/dhcp/dhcpd.leases
