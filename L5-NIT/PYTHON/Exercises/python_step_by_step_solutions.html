1️⃣ Special-Purpose Functions (map, filter, reduce)

🔹 Exercise 1: Student Marks Processing

marks = [45, 67, 89, 32, 76, 90, 54]

✅ Step 1: Increase each mark by 5 (using map)

bonus_marks = list(map(lambda m: m + 5, marks))
print(bonus_marks)
map() applies a function to each element
lambda m: m + 5 adds bonus marks
list() converts result to a list
🧠 Real life: Schools automatically add bonus points.

✅ Step 2: Filter passing students (≥ 50)

passed = list(filter(lambda m: m >= 50, bonus_marks))
print(passed)
filter() keeps only values that return True
Students who failed are removed
🧠 Real life: Filtering successful candidates.

✅ Step 3: Total class score (using reduce)

from functools import reduce

total_score = reduce(lambda a, b: a + b, passed)
print(total_score)
reduce() combines values into one
Adds all marks together
🧠 Real life: Calculating total results.

2️⃣ Lambda Functions

🔹 Exercise 3: Electricity Bill Calculator

bill = lambda units: units * 120

units = [50, 120, 300]

for u in units:
    print(f"Units: {u}, Bill: {bill(u)} RWF")
Lambda = small one-line function
No need to define a full function
Used when logic is short and simple
🧠 Real life: Utility billing systems.

🔹 Exercise 4: Sorting Phone Numbers

numbers = ["+250788123456", "+256701234567", "+250722999888"]

sorted_numbers = sorted(numbers, key=lambda n: not n.startswith("+250"))
print(sorted_numbers)
sorted() sorts lists
key decides how sorting happens
Rwandan numbers come first
🧠 Real life: Telecom databases.

3️⃣ Python Generators

🔹 Exercise 5: Daily Sales Generator

def daily_sales(sales):
    for sale in sales:
        yield sale

sales_data = [12000, 15000, 18000, 20000]

total = 0
for sale in daily_sales(sales_data):
    total += sale

print("Total sales:", total)
yield produces values one at a time
Saves memory
Perfect for large data
🧠 Real life: Processing large business data.

🔹 Exercise 6: Student ID Generator

def student_id_generator(start=1000):
    while True:
        yield start
        start += 1

gen = student_id_generator()

print(next(gen))
print(next(gen))
print(next(gen))
IDs generated only when needed
Infinite generator
🧠 Real life: Automatic ID systems.

4️⃣ Python Closures

🔹 Exercise 7: Salary Calculator

def salary_calculator(base_salary):
    def calculate(bonus):
        return base_salary + bonus
    return calculate

teacher_salary = salary_calculator(300000)
print(teacher_salary(50000))
Inner function remembers base_salary
This memory is called a closure
🧠 Real life: Payroll systems.

🔹 Exercise 8: Login Counter

def login_counter():
    count = 0
    def login():
        nonlocal count
        count += 1
        return count
    return login

user = login_counter()
print(user())
print(user())
print(user())
nonlocal allows modifying outer variable
Keeps count even after function ends
🧠 Real life: Tracking user activity.

5️⃣ Python Decorators

🔹 Exercise 9: Execution Logger

def logger(func):
    def wrapper():
        print("Function started")
        func()
        print("Task completed")
    return wrapper

@logger
def process_transactions():
    print("Processing bank transactions...")

process_transactions()
Decorators add extra behavior
No change to original function code
🧠 Real life: System logging.

🔹 Exercise 10: Authentication Decorator

def auth_required(func):
    def wrapper(user_logged_in):
        if user_logged_in:
            func()
        else:
            print("Access denied")
    return wrapper

@auth_required
def view_account_balance():
    print("Your balance is 250,000 RWF")

view_account_balance(True)
Controls access
Used heavily in web apps
🧠 Real life: Security systems.

6️⃣ Recursive Functions

🔹 Exercise 11: Loan Repayment Countdown

def repay(balance, monthly_payment):
    if balance <= 0:
        return 0
    return 1 + repay(balance - monthly_payment, monthly_payment)

months = repay(500000, 50000)
print("Months needed:", months)
Function calls itself
Stops when balance ≤ 0
🧠 Real life: Financial calculations.
