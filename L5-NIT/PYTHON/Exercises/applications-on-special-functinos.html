Python Group Presentation Tasks

Group Task 1: Lambda Functions

a. Definition & Explanation

Explain what a lambda function is in Python. Describe its syntax and how it differs from normal functions created using def.

b. Uses and Applications (When to Use)

Discuss situations where lambda functions are most useful, such as short operations, anonymous functions, and functional programming tools like map, filter, and reduce.

c. Real-Life Examples

  • Sorting student records by marks
  • Calculating discounts in an online shop
  • Filtering even/odd numbers in financial data

Group Task 2: Python Generators

a. Definition & Explanation

Define generators in Python and explain how they work using the yield keyword. Compare generators with normal functions and lists.

b. Uses and Applications (When to Use)

Explain when generators are preferred, especially in memory-efficient programming and handling large datasets.

c. Real-Life Examples

  • Reading large log files line by line
  • Streaming data from sensors
  • Pagination in web applications (loading posts gradually)

Group Task 3: Python Closures

a. Definition & Explanation

Define a closure in Python. Explain how a function can remember values from its enclosing scope even after the outer function has finished executing.

b. Uses and Applications (When to Use)

Discuss when closures are useful, such as data hiding, maintaining state, and avoiding global variables.

c. Real-Life Examples

  • Creating a counter for website visits
  • Implementing user-specific settings
  • Tracking login attempts in a system

Group Task 4: Python Decorators

a. Definition & Explanation

Explain what decorators are in Python and how they modify the behavior of functions using the @decorator syntax.

b. Uses and Applications (When to Use)

Describe when decorators should be used, such as logging, authentication, authorization, and performance monitoring.

c. Real-Life Examples

  • Logging function execution in web applications
  • Checking user authentication before accessing a page
  • Measuring execution time of functions

Group Task 5: Recursive Functions

a. Definition & Explanation

Define a recursive function and explain how it calls itself. Describe the importance of a base case.

b. Uses and Applications (When to Use)

Explain when recursion is appropriate, especially in problems that can be broken into smaller sub-problems.

c. Real-Life Examples

  • Calculating factorials and Fibonacci numbers
  • Searching files and folders in an operating system
  • Solving maze or tree-structure problems

Group Task 6: Higher-Order Functions

a. Definition & Explanation

Define higher-order functions and explain how they can accept functions as arguments or return functions.

b. Uses and Applications (When to Use)

Discuss why higher-order functions are useful in clean, reusable, and functional programming.

c. Real-Life Examples

  • Applying discounts or tax calculations dynamically
  • Data processing using map, filter, and reduce
  • Custom validation functions in web forms
