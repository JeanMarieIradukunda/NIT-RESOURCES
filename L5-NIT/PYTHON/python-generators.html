🧠 Python Generators — A Special Type of Function

Clear, structured study notes with examples and comparisons

A. Definitions & Explanations

✅ What is a Generator?

  • Special type of function using yield instead of return
  • Produces values one at a time
  • Remembers where it stopped
  • Does not store all results in memory
Think of a generator as a smart data producer that gives results only when requested.

✅ What is an Iterator?

  • Produces values one at a time
  • Keeps track of position
  • Supports __next__() and __iter__()
students = ["Ana", "Ben", "Chris"]
it = iter(students)

print(next(it))
print(next(it))

✅ Generator = Iterator Creator

A generator function automatically creates an iterator object.

def gen():
    yield 1

✅ Generator Object

  • Iterable
  • Lazy value production
  • Works with for and next()
g = gen()
print(type(g))

B. Key Ideas Behind Generators

⭐ Core Principles

  • Uses yield → pauses function
  • Execution resumes after each yield
  • Lazy evaluation saves memory
  • State remembered between yields
def counter():
    x = 1
    yield x
    x += 1
    yield x

C. Return Behavior

✅ What Does a Generator Return?

Calling a generator function returns a generator object, not values.

def student_ids():
    yield 101
    yield 102

g = student_ids()
print(g)

✅ Output Behavior

  • Values produced one by one
  • Only when requested
  • Ends with StopIteration
print(next(g))
print(next(g))

Preferred Loop Usage

for sid in student_ids():
    print(sid)

Different Data Types

def student_data():
    yield {"name":"Ana","grade":85}
    yield {"name":"Ben","grade":90}

D. Comparison with Regular Functions

🎓 Regular Function

def get_grades():
    grades = []
    for g in [70,80,90]:
        grades.append(g)
    return grades

⚡ Generator Version

def get_grades():
    for g in [70,80,90]:
        yield g

def attendance():

E. Main Use Cases

  • ✅ Large datasets
  • ✅ File processing
  • ✅ Streaming data
  • ✅ Data pipelines
  • ✅ Infinite sequences
def read_lines(file):
    for line in file:
        yield line

passed = (s for s in students if s["grade"] >= 50)

def roll_numbers():
    n = 1
    while True:
        yield n
        n += 1

F. Common Generator Methods

✅ next()

g = attendance()
next(g)

✅ list()

Converts to list — loses memory advantage.
list(attendance())

✅ Generator Expressions

grades = (g for g in range(5))

✅ send()

def student():
    name = yield
    print(name)

✅ close() & throw()

g.close()
g.throw(ValueError)

🎯 Quick Summary

  • ✅ Use yield
  • ✅ Memory efficient
  • ✅ Lazy evaluation
  • ✅ Automatically iterators
  • ✅ Ideal for large/streaming data
  • ✅ Pause & resume execution
  • ✅ Return generator objects
