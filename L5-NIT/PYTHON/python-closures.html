🧠 Python Closures — Study Notes

Clear, practical notes for students learning closures in Python

A. Definitions & Core Meaning

✅ What is a Closure?

A closure in Python is a function that remembers variables from its outer (enclosing) function even after that outer function has finished executing.

  • Remembers variables from its outer function even after it finishes
  • Created when an inner function uses outer variables and the outer function returns it

In simple terms:

  • It’s like a function with memory.
  • It allows you to keep some data “private” and use it later whenever you call the inner function.

Key Points:

  • It must be a nested function (a function defined inside another function).
  • The inner function uses variables from the outer function.
  • The outer function returns the inner function, creating the closure.
Think of a closure as a function with memory — it remembers its environment.

✅ Key Terms

  • Outer Function: contains another function
  • Inner Function: defined inside outer function
  • Free Variable: variable used in inner function but defined outside
def outer():
    x = 10   # free variable
    def inner():
        print(x)

✅ Closures, Nested Functions & Scope

Closures extend nested functions by remembering outer variables permanently.

B. Basic Concepts Behind Closures

  • 🔹 Lexical Scoping — variables accessed based on definition location
  • 🔹 Variable Binding — inner function binds to outer variables
  • 🔹 Remembering State — keeps outer variables alive after function ends
  • 🔹 Function Returning Function — outer function returns inner
  • 🔹 Stateful Functions — store internal state between calls

✅ Example 1 — Simple Closure

def make_multiplier(n):
    def multiply(x):
        return x * n
    return multiply

times3 = make_multiplier(3)
print(times3(5))  # 15

✅ Example 2 — Greeting Function

def greeting(name):
    def say_hello():
        print("Hello", name)
    return say_hello

greet_john = greeting("John")
greet_john()

C. How Closures Work Internally

  • Outer function creates variables
  • Inner function references them
  • Inner function returned → stored in closure cell
  • Access outer variables even after function finishes
  • Use nonlocal to modify outer variables

✅ Example with nonlocal

def counter():
    count = 0
    def increment():
        nonlocal count
        count += 1
        return count
    return increment

c = counter()
print(c())
print(c())

E. Common Methods & Patterns

✅ Function Factory

def power_factory(exp):
    def power(x):
        return x ** exp
    return power

square = power_factory(2)
cube = power_factory(3)

✅ Counter Pattern

def make_counter():
    count = 0
    def inc():
        nonlocal count
        count += 1
        return count
    return inc

✅ Accumulator Pattern

def accumulator():
    total = 0
    def add(x):
        nonlocal total
        total += x
        return total
    return add

✅ Decorators Use Closures

def logger(func):
    def wrapper():
        print("Running function")
        func()
    return wrapper

H. Advantages & Limitations

  • ✅ Advantages: Keeps state without classes, cleaner than globals, functional programming, decorators & factories
  • ⚠️ Limitations: Harder for beginners, debugging tricky, complex state better with classes
  • ❌ Common Mistakes: Forgetting nonlocal, not returning inner, expecting auto-change of outer vars, confusing with nested functions

I. Summary

  • Closures remember values from outer function even after it ends
  • Store state safely, avoid globals, enable function factories, power decorators, create customizable behavior
  • ✅ Use closures for small stateful behavior, private variables, function customization
  • ✅ Use classes when logic is complex or many related methods needed
