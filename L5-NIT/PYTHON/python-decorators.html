A. Core Meaning

Definitions & Core Concepts

✅ What is a Python Decorator?

A decorator is a function that adds extra behavior to another function without modifying the original function’s code. It wraps the function and extends what it does.

✅ In Simple Words

  • Add-on feature for a function
  • Keeps original function clean
  • Adds logging, timing, validation, permissions automatically

🔹 Functions as Objects

def greet():
    print("Hello")

say_hi = greet
say_hi()

🔹 Higher-Order Functions

Functions that take or return other functions. Decorators are higher-order functions.

🔹 Closures Connection

  • Inner wrapper function
  • Remembers original function
  • Runs it later
B. Foundations

Basic Concepts Behind Decorators

  • Functions returning functions

✅ Example — Basic Decorator

def my_decorator(func):
    def wrapper():
        print("Before function runs")
        func()
        print("After function runs")
    return wrapper

@my_decorator
def say_hello():
    print("Hello")

say_hello()
Output:
Before function runs → Hello → After function runs

✅ Decorator with Return Value

def double_result(func):
    def wrapper():
        return func() * 2
    return wrapper
C. Internal Mechanics

How Decorators Work Internally

Decorator Translation

@decorator
def f():
    pass

Equals:

def f():
    pass

f = decorator(f)
  • Original function passed to decorator
  • Decorator returns wrapper
  • Name now points to wrapper

Passing Arguments Safely

def logger(func):
    def wrapper(*args, **kwargs):
        print("Calling function...")
        return func(*args, **kwargs)
    return wrapper

E. Common Patterns

Common Decorator Patterns

Logging

def log_calls(func):
    def wrapper(*a, **k):
        print("Running:", func.__name__)
        return func(*a, **k)
    return wrapper

Timing

import time
def timer(func):
    def wrapper(*a, **k):
        start = time.time()
        result = func(*a, **k)
        print("Time:", time.time()-start)
        return result
    return wrapper

Decorator with Parameters

def repeat(n):
    def decorator(func):
        def wrapper():
            for _ in range(n):
                func()
        return wrapper
    return decorator

H. Pros & Cons

Advantages & Limitations

Benefits
  • Reusable logic
  • Keeps code clean
  • Separates concerns
  • Avoids repetition
  • Framework power feature
Limitations
  • Harder for beginners
  • Behavior may be hidden
  • Debugging complexity
  • Too many decorators reduce clarity

❌ Common Beginner Mistakes

  • Forgetting *args, **kwargs
  • Not returning wrapper result
  • Losing metadata → use functools.wraps
  • Overusing decorators
