1️⃣ Class, Object, Method, and Attribute (Using Students and Modules)

🎯 Scenario:

We want to model Students who register for Modules (Courses).

✅ Example Code with Clear Comments

# CLASS
# A class is a blueprint/template used to create objects.
class Student:
    
    # CONSTRUCTOR
    # This runs automatically when we create an object.
    # It initializes the attributes.
    def __init__(self, name, student_id, module):
        
        # ATTRIBUTES (Variables that belong to the object)
        self.name = name              # Student name
        self.student_id = student_id  # Student ID
        self.module = module          # Module (Course) name
    
    # METHOD
    # A method is a function that belongs to a class.
    def display_info(self):
        print("Name:", self.name)
        print("Student ID:", self.student_id)
        print("Module:", self.module)


# OBJECT
# An object is an instance (real example) created from a class.
student1 = Student("Alice", "S101", "Mathematics")
student2 = Student("Bob", "S102", "Physics")

# Calling a METHOD using objects
student1.display_info()
print("-----------")
student2.display_info()

🔎 Clear Differentiation

Concept Meaning Example From Code
Class Blueprint/template class Student:
Object Real instance created from class student1, student2
Attribute Data/variables inside object name, student_id, module
Method Function inside class display_info() 
2️⃣ Differentiate: Constructor, Getter, and Setter
A. Constructor
✔ What it is:

A special method that runs automatically when an object is created.

✔ In Python:

It is written as:

def __init__(self):
✔ Purpose:

Initialize attributes

Set starting values

B. Getter
✔ What it is:

A method used to access (get/read) private attributes safely.

C. Setter
✔ What it is:

A method used to modify (set/update) private attributes safely.

✅ Example Showing Constructor, Getter, and Setter
class Student:
    
    def __init__(self, name, student_id):
        # Private attribute (Encapsulation)
        self.__name = name
        self.__student_id = student_id
    
    # GETTER (Read data)
    def get_name(self):
        return self.__name
    
    # SETTER (Update data)
    def set_name(self, new_name):
        self.__name = new_name


student1 = Student("Alice", "S101")

print(student1.get_name())  # Using getter

student1.set_name("Grace")  # Using setter
print(student1.get_name())
🔎 Differences Table
Feature	Constructor	Getter	Setter
When it runs	Automatically when object is created	When called	When called
Purpose	Initialize data	Read data	Modify data
Special name?	__init__()	No	No
3️⃣ Encapsulation, Inheritance, and Polymorphism in Python
🔐 A. ENCAPSULATION
✔ Meaning:

Hiding internal data and controlling access to it.

✔ Why?

Protect data

Prevent direct modification

Improve security

✔ Steps to Create Encapsulation

Make attributes private using __

Create getter to read data

Create setter to update data

✅ Example
class Student:
    
    def __init__(self, name, grade):
        self.__grade = grade   # Private attribute
        self.name = name
    
    def get_grade(self):       # Getter
        return self.__grade
    
    def set_grade(self, grade): # Setter
        if grade >= 0 and grade <= 100:
            self.__grade = grade
        else:
            print("Invalid grade")
✔ Guidelines for Encapsulation

Use __attribute for private variables

Always validate data inside setter

Never access private attributes directly outside class

Use getters for reading

👨‍👦 B. INHERITANCE
✔ Meaning:

One class (child) inherits properties and methods from another class (parent).

✔ Why?

Code reuse

Avoid repetition

Extend functionality

✔ Steps to Create Inheritance

Create a parent class

Create a child class using:

class ChildClass(ParentClass):

Use super() to call parent constructor

✅ Example
# Parent class
class Person:
    
    def __init__(self, name):
        self.name = name
    
    def display(self):
        print("Name:", self.name)


# Child class inheriting from Person
class Student(Person):
    
    def __init__(self, name, module):
        super().__init__(name)   # Call parent constructor
        self.module = module
    
    def show_module(self):
        print("Module:", self.module)


student1 = Student("Alice", "Mathematics")
student1.display()       # Inherited method
student1.show_module()   # Child method
✔ Guidelines for Inheritance

Use inheritance when classes have "is-a" relationship

Student is a Person

Use super() to reuse parent constructor

Do not duplicate code from parent class

🔄 C. POLYMORPHISM
✔ Meaning:

Same method name behaves differently in different classes.

✔ Why?

Flexibility

Clean code

Easier extension

✔ Types in Python

Method Overriding (most common)

Same method name in different classes

✅ Example (Method Overriding)
class Person:
    
    def role(self):
        print("I am a person")


class Student(Person):
    
    def role(self):   # Overriding parent method
        print("I am a student")


class Teacher(Person):
    
    def role(self):
        print("I am a teacher")


# Polymorphism in action
people = [Student(), Teacher(), Person()]

for p in people:
    p.role()
✔ Output:

Each object responds differently to the same method role().

✔ Guidelines for Polymorphism

Use same method name across related classes

Override parent method in child class

Keep method purpose similar

Use loops to apply polymorphism
