1. APPLYING OOP CONCEPTS

Object-Oriented Programming (OOP) is a programming style where software is designed using objects that represent real-world things.

Instead of writing a long list of instructions, you organize your program into classes and objects.

OOP is a way of writing programs by creating blueprints (classes) and building real things (objects) from them.

🔹 Key Ideas in OOP

1️⃣ Class

A class is a blueprint/Design/Drawing/Proposal or template used to create objects.

It defines:

  • The data (attributes) an object will have
  • The actions (methods) an object can perform

A class does not represent one real thing — it represents the design of many similar things.

If OOP was like building something:
• Class → the design plan
• Object → the real item built from that plan

You can create many objects from one class.

🎓 Real-Life Examples

Class: Student

It defines:

  • name
  • age
  • grade
  • study()
  • take_exam()

Now in real life:

  • John → Student object
  • Mary → Student object
  • Eric → Student object

All are created from the Student class, but each has different data.

2️⃣ Course Example

Class: Course

It defines:

  • course_name
  • course_code
  • duration
  • start_course()

Real-life objects:

  • Mathematics Course
  • ICT Course
  • Physics Course

Each course is an object created from the Course class.

🧠 Why Classes Are Important

  • Organize code clearly
  • Avoid repeating code
  • Create structured systems like school management systems
  • Build reusable designs

2️⃣ Object

An object is a real instance created from a class.

It represents a specific thing that contains:

  • Attributes (data) → what it has
  • Methods (actions) → what it can do

A class is the blueprint. An object is the actual thing built from that blueprint.

🔹 Real-Life Example

If a class is a Student template, then an object is a real student like John or Mary.

You can create many objects from one class.

Objects:

Inheritance

Inheritance is a feature in OOP that allows one class to reuse the properties and methods of another class.

It helps avoid repeating code and makes programs organized.

A child class can inherit (receive) features from a parent class.

Parent Class → gives common features
Child Class → adds its own special features

Real-Life Situation

In a school:

All people have:

  • name
  • age

So we create a Person class.

Then:

  • Student inherits from Person
  • Teacher inherits from Person

Both Student and Teacher automatically have name and age.

But:

  • Student can study()
  • Teacher can teach()

💻 Simple Python Example

class Person:
    def __init__(self, name):
        self.name = name

class Student(Person):   # Inheriting from Person
    def study(self):
        print(self.name, "is studying")

student1 = Student("John")
student1.study()

Student inherits name from Person.

Polymorphism

Polymorphism means one method name can behave differently in different classes.

“Poly” means many

“Morph” means forms

So it means many forms.

Same method → different results.

In school:

  • Student → attends class
  • Teacher → teaches lesson
  • Principal → manages school

Same method name, different behavior.

class Student:
    def role(self):
        print("I attend classes")

class Teacher:
    def role(self):
        print("I teach students")

def describe(person):
    person.role()

describe(Student())
describe(Teacher())

Same method name → different output.

Encapsulation

Encapsulation means protecting data inside a class and controlling how it is accessed or changed.

It prevents unauthorized modification of important data.

Hide sensitive information and allow access only through safe methods.

In a school system:

Student marks should:

  • Not be changed directly by everyone
  • Only be updated by authorized system

So marks are protected inside the class.

class Student:
    def __init__(self):
        self.__marks = 0   # Private variable

    def set_marks(self, marks):
        if marks >= 0:
            self.__marks = marks

    def get_marks(self):
        return self.__marks

student1 = Student()
student1.set_marks(85)
print(student1.get_marks())

📘 Syntax of Python OOP

(Classes, Objects, Attributes, and Methods)

✔ 1️⃣ Syntax of a Python Class

class ClassName:
    # attributes and methods go here

Explanation:

  • class → keyword used to define a class
  • ClassName → name of the class (usually starts with Capital letter)
  • Colon : → starts the class body
  • Indentation → defines what belongs inside the class
class Student:
    pass

pass means the class is empty for now.

✔ 2️⃣ Syntax of Attributes

Attributes are variables inside a class. They store data about the object.

class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

Explanation:

  • __init__ → special method (constructor)
  • Runs automatically when object is created
  • self → refers to the current object
  • self.name and self.age → attributes

✔ 3️⃣ Syntax of Methods

class Student:
    def study(self):
        print("Student is studying")

Methods must include self as first parameter.

✔ 4️⃣ Syntax of Creating Objects

student1 = Student("John")
print(student1.name)
student1.study()

✔ Complete Example

class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def introduce(self):
        print("My name is", self.name, "and I am", self.age, "years old.")

student1 = Student("John", 18)
student2 = Student("Mary", 17)

print(student1.name)
student1.introduce()
