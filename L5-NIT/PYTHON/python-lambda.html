🧠 Python Lambda Functions

Modern, student-friendly notes with examples and real-life use cases

A. Definitions & Core Meaning

✅ What is a Lambda Function?

  • A small one-line function written without def
  • Used for quick, short tasks
  • Often created where it is immediately needed

✅ Why is it called Anonymous?

  • No function name
  • Created inline
  • Used immediately
lambda x: x * 2

✅ Difference from Normal Function

def double(x):
    return x * 2

lambda x: x * 2

✅ General Syntax

lambda arguments : expression

lambda a, b: a + b

B. Basic Concepts of Lambda Functions

✅ Structure

lambda parameters : single_expression
  • Parameters → inputs
  • Expression → calculated & returned

✅ Rules & Limitations

  • ✅ Only one expression
  • ❌ No multiple statements
  • ❌ No assignments
  • ❌ No multi-line logic
  • ✅ Auto-return (no return keyword)

✅ Example

square = lambda x: x * x
print(square(5))
Output: 25

✅ Lambda vs Multi-Line Function

lambda x: x + 1

def add_one(x):
    y = x + 1
    return y
Use regular functions when logic becomes longer.

✅ Simple Examples

add = lambda a, b: a + b
is_even = lambda x: x % 2 == 0

Example — Conditional Logic

max_val = lambda a, b: a if a > b else b

def max_val(a, b):
    if a > b:
        return a
    else:
        return b
Performance difference is very small — choose readability first.

D. Most Common Use Cases

✅ Sorting

students = [("Ana",80), ("Bob",65)]
students.sort(key=lambda s: s[1])

✅ Filtering

evens = list(filter(lambda x: x % 2 == 0, nums))

✅ Mapping

squares = list(map(lambda x: x*x, nums))

✅ Quick Calculations

tax = lambda price: price * 0.18

✅ Callbacks

button_click_handler = lambda: print("Clicked")

E. Functions Commonly Used with Lambda

map()

result = list(map(lambda x: x+10, nums))

filter()

result = list(filter(lambda x: x > 10, nums))

reduce()

from functools import reduce
total = reduce(lambda a,b: a+b, [1,2,3,4])

sorted(), max(), min()

top = max(students, key=lambda s: s[1])

Lambda vs List Comprehension

list(map(lambda x: x*x, nums))
[x*x for x in nums]

F. Student Real-Life Examples

sorted_marks = sorted(marks, key=lambda m: m[1], reverse=True)
passing = list(filter(lambda s: s >= 50, scores))
weighted = lambda test, exam: test*0.4 + exam*0.6
top = max(students, key=lambda s: s["avg"])
present_days = sum(map(lambda d: d, attendance))

G. Advantages & Disadvantages

  • ✅ Short & quick
  • ✅ Good for temporary logic
  • ✅ Useful with map/filter/sorted
  • ⚠️ Hard to read when complex
  • ⚠️ Only one expression
  • ⚠️ Not for big logic
❌ Avoid lambda for multi-step logic, complex conditions, or highly reused code.

H. Summary

  • Lambda = small one-line anonymous function
  • Uses lambda keyword
  • Returns automatically
  • Best for short functional-style operations
Rule: Use lambda for small tasks — use def for larger logic.
