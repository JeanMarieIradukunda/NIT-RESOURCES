Python Object-Oriented Programming

Classes & Objects Rules / Guidelines

Complete modern notes for understanding Python OOP Classes, Objects, Methods, Constructors, Inheritance, Encapsulation and more.

1
Class Naming Rule

Use Capital letters for class names.

✅ Correct
class Student:
❌ Wrong
class student:
Class names usually use PascalCase in Python OOP. This improves readability and follows Python conventions.
2
Method Naming Rule

Use small letters for method names.

✅ Correct
def study(self):
❌ Wrong
def Study(self):
Python methods commonly use lowercase or snake_case naming style.
3
self Rule

self must be the first parameter in instance methods.

✅ Correct
def study(self):
❌ Wrong
def study():
self represents the current object itself and allows access to attributes and methods inside the class.
4
Object Creation Rule

To use a class, create an object.

object_name = ClassName()

student1 = Student()
Objects are real instances created from classes.
5
Constructor Rule (__init__())

__init__() runs automatically when object is created.

class Student:
    def __init__(self):
        print("Object created")
Constructors initialize object data automatically.
6
Argument Matching Rule

Arguments passed during object creation must match __init__() parameters.

✅ Correct
class Student:
    def __init__(self, name):
        self.name = name

student1 = Student("John")
❌ Wrong
student1 = Student()
TypeError
Missing required arguments causes errors during object creation.
7
Attribute Rule

Use self.attribute to create object variables.

✅ Correct
self.name = name
❌ Wrong
name = name
Without self, the variable becomes local and cannot be accessed outside the method.
8
Accessing Attributes Rule

Use dot notation (.)

✅ Correct
student1.name
Dot notation is used to access object attributes and methods.
9
Calling Methods Rule

Methods are called using objects.

✅ Correct
student1.study()
❌ Wrong
study()
Methods belong to objects, so they must be called through objects.
10
Indentation Rule

All class contents must be indented.

✅ Correct
class Student:
    def study(self):
        print("Study")
❌ Wrong
class Student:
def study(self):
print("Study")
Python uses indentation to define blocks of code.
11
One Object ≠ Another Object Rule
student1 = Student("John")
student2 = Student("Alice")

student1 and student2 are different objects.

Each object stores its own independent data.
12
Method Access Rule
class Student:
    def __init__(self, name):
        self.name = name

    def show(self):
        print(self.name)
Methods can access attributes through self.
13
Class Variables Rule
class Student:
    school = "RCA"
Student.school
Class variables are shared among all objects of the class.
14
Private Variable Rule
self.__password = "1234"
Double underscore (__) helps hide sensitive variables.
15
Inheritance Rule
class Person:
    def speak(self):
        print("Hello")

class Student(Person):
    pass
Child classes inherit methods and properties from parent classes.
16
Method Overriding Rule
class Animal:
    def sound(self):
        print("Animal sound")

class Dog(Animal):
    def sound(self):
        print("Bark")
Overriding allows child classes to provide their own implementation.
17
Encapsulation Rule
self.__balance
Encapsulation protects data from direct external modification.
18
Object-Oriented Principle Rule
Encapsulation
Inheritance
Polymorphism
Abstraction
These are the four major pillars of Object-Oriented Programming.
19
Constructor Optional Rule
class Student:
    pass

But you cannot pass arguments unless __init__() exists.

A constructor is optional, but useful for initializing data.
20
Error Prevention Rules
✔ Use self
✔ Match arguments correctly
✔ Indent properly
✔ Create object before calling methods
✔ Use self.attribute
✔ Use dot notation (.)
✔ Keep class names Capitalized
✔ Keep method names lowercase
Following these rules helps avoid common Python OOP errors.
MOST IMPORTANT

Golden Rule of OOP

CLASS → creates → OBJECT

OBJECT → uses → METHODS & ATTRIBUTES

class Car:
    def drive(self):
        print("Driving")

car1 = Car()

car1.drive()
