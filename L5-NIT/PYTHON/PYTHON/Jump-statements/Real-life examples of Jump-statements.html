🐍 Jump Statements in Python

Understanding when and where to use break, continue, and pass is what makes you a great programmer.

1. break — Real-Life Applications

🧾 a) Search Operations (Stop When Found)
students = ["Alice", "John", "Peter", "Maria", "David"]
target = "Maria"

for name in students:
    if name == target:
        print(f"{target} found in the list!")
        break  # Stop searching
✅ Use Case: Stop searching after finding a match — e.g., user IDs, barcodes, or names.
💳 b) ATM PIN Attempts
correct_pin = "1234"

for attempt in range(3):
    pin = input("Enter PIN: ")
    if pin == correct_pin:
        print("Access granted.")
        break
    else:
        print("Wrong PIN, try again.")
✅ Use Case: Used in login or security systems to exit once correct input is received.
🚗 c) Sensor or Monitoring Systems
temperatures = [80, 85, 89, 105, 110]

for t in temperatures:
    if t > 100:
        print("⚠ Warning: System overheated! Stopping check.")
        break
✅ Use Case: Used in IoT or industrial monitoring to stop when safety thresholds are reached.

2. continue — Real-Life Applications

📋 a) Data Cleaning / Skipping Invalid Entries
data = [45, None, 78, "", 23, None, 90]

for item in data:
    if item is None or item == "":
        continue
    print(f"Processing {item}")
✅ Use Case: Used in data preprocessing to ignore corrupt or missing entries.
🧍 b) Attendance / Presence Checking
students = {"Alice": "present", "John": "absent", "Peter": "present"}

for name, status in students.items():
    if status == "absent":
        continue
    print(f"{name} is attending class.")
✅ Use Case: Used in attendance or HR systems to process only active records.
💰 c) Payment System Validation
transactions = [
    {"id": 1, "status": "success"},
    {"id": 2, "status": "failed"},
    {"id": 3, "status": "success"},
]

for tx in transactions:
    if tx["status"] == "failed":
        continue
    print(f"Processing transaction {tx['id']}")
✅ Use Case: Used in finance apps to skip failed transactions safely.

3. pass — Real-Life Applications

🧩 a) Function or Class Prototyping
def calculate_salary():
    pass  # To be implemented later
✅ Use Case: Used in large projects to reserve space before adding full logic.
🏛 b) Exception Handling (Temporary Ignoring)
try:
    risky_operation()
except Exception:
    pass  # Handle later
✅ Use Case: Used in debugging or testing to continue program flow safely.
🔒 c) Security or Access Control (Future Logic)
def check_user_permission(user):
    pass  # Future: Add admin/lecturer/student roles
✅ Use Case: Used in API or app design to reserve logic for future development.

Summary — Real-World Use Cases

Statement Real-Life Example Common Application
break Stop PIN attempts after success Login systems, search operations
continue Skip missing data entries Data cleaning, analytics
pass Placeholder for future logic App development, OOP prototypes
