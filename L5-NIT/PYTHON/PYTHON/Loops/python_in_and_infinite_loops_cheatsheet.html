Python: `in` & Infinite Loops

`in` Keyword — Quick Facts

  • Membership test: x in y returns True/False.
  • Iteration: used in for item in iterable: loops.
  • Works on lists, tuples, sets, strings, ranges, dicts (dicts check keys).
  • Use not in for negation.
  • Prefer set for many membership checks (O(1) average).
Examples
# List membership
fruits = ["apple","banana","cherry"]
if "banana" in fruits:
    print("Found banana")

# String substring
if "py" in "python":
    print("Substring found")

# Dictionary (checks keys)
student = {"name":"Alice","age":20}
if "age" in student:
    print("Key 'age' exists")
Notes & Pitfalls
  • Dict: in checks keys. Use .values() or .items() if needed.
  • Large lists are O(n). Convert to set() for repeated checks.
  • Strings: "a" in "ab" works only for substrings, not list elements.
Quick tip

Readable code beats clever code — use in for clear membership tests instead of manual loops.

Real‑World Example: Search a Name

A very simple example for beginners: checking if a student's name exists in a list.

students = ["Alice", "Brian", "Clara"]
search_name = input("Enter a name to search: ")

if search_name in students:
    print(search_name, "is in the list!")
else:
    print(search_name, "is not in the list.")

Here we use in to see if the input matches any name in the list. Very beginner-friendly!

Infinite Loops — Quick Facts

  • Use cases: servers, listeners, background tasks, sensors.
  • Safety: always provide a graceful stop (signal, command, keyboard interrupt).
  • Avoid busy‑wait: use time.sleep() or blocking I/O.
Basic patterns
# Simple infinite loop (stop with Ctrl+C)
while True:
    print("Running...")

# Safer: allow user exit
while True:
    cmd = input("type 'exit' to stop: ")
    if cmd.strip().lower() == 'exit':
        break
Best practice: sleep + exception handling
import time
try:
    while True:
        do_work()
        time.sleep(1)   # avoid 100% CPU
except KeyboardInterrupt:
    print("shutting down gracefully")

Real‑World Examples

1) Simple HTTP server loop (concept)

A server accepts connections and processes them in a loop. Real servers use frameworks or async IO.

import socket
s = socket.socket()
s.bind(('0.0.0.0', 8000))
s.listen()
try:
    while True:
        client, addr = s.accept()   # blocking call
        handle(client)
except KeyboardInterrupt:
    s.close()
2) Sensor/IoT read loop

Poll a sensor every second and send readings to a server.

import time

while True:
    reading = read_sensor()
    send(reading)
    time.sleep(1)
3) Log-watching tool (stream)

Follow a file for new lines; blocking I/O avoids busy-wait.

with open('app.log') as f:
    f.seek(0, 2)  # go to end
    while True:
        line = f.readline()
        if not line:
            time.sleep(0.5)
            continue
        process(line)
