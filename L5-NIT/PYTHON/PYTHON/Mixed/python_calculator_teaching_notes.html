1. Basic calculator (step-by-step)
Records two numbers and an operator, then computes the answer.
Beginner

Notes: use int() to convert user input into integers. Division / returns a float. Always validate the operator.

# Record calculation Details
num1 = int(input("Enter First Number: "))
operator = input("Enter operator (+, -, *, /, //, %, **): ")
num2 = int(input("Enter Second Number: "))

# Perform recorded operation
if operator == "+":
    answer = num1 + num2
elif operator == "-":
    answer = num1 - num2
elif operator == "*":
    answer = num1 * num2
elif operator == "/":
    answer = num1 / num2
elif operator == "//":
    answer = num1 // num2
elif operator == "%":
    answer = num1 % num2
elif operator == "**":
    answer = num1 ** num2
else:
    answer = "Invalid operator!"

print("Result:", answer)

2. Continuous calculator (loop) — keep working until user exits
Prompt repeatedly. Handle division-by-zero errors gracefully.
Interactive 
print("🔢 Simple Calculator")
print("Type 'exit' as operator to quit.\n")

while True:
    # Get first number
    num1 = int(input("Enter First Number: "))

    # Get operator
    operator = input("Enter operator (+, -, *, /, //, %, ** or exit): ")
    if operator.lower() == "exit":
        print("Calculator closed. Goodbye! 👋")
        break  # stop the loop

    # Get second number
    num2 = int(input("Enter Second Number: "))

    # Perform operation
    if operator == "+":
        answer = num1 + num2
    elif operator == "-":
        answer = num1 - num2
    elif operator == "*":
        answer = num1 * num2
    elif operator == "/":
        if num2 != 0:
            answer = num1 / num2
        else:
            answer = "Error: Division by zero!"
    elif operator == "//":
        if num2 != 0:
            answer = num1 // num2
        else:
            answer = "Error: Division by zero!"
    elif operator == "%":
        if num2 != 0:
            answer = num1 % num2
        else:
            answer = "Error: Division by zero!"
    elif operator == "**":
        answer = num1 ** num2
    else:
        answer = "Invalid operator!"

    print("Result:", answer, "\n")
3. Using match / case (Python 3.10+)
A switch-like statement — cleaner branching for operators.
Python 3.10+ 
print("🔢 Simple Calculator (using switch/match)")
print("Type 'exit' as operator to quit.\n")

while True:
    # Get first number
    num1 = int(input("Enter First Number: "))

    # Get operator
    operator = input("Enter operator (+, -, *, /, //, %, ** or exit): ")
    if operator.lower() == "exit":
        print("Calculator closed. Goodbye! 👋")
        break

    # Get second number
    num2 = int(input("Enter Second Number: "))

    # Perform operation using match (switch-like)
    match operator:
        case "+":
            answer = num1 + num2
        case "-":
            answer = num1 - num2
        case "*":
            answer = num1 * num2
        case "/":
            answer = num1 / num2 if num2 != 0 else "Error: Division by zero!"
        case "//":
            answer = num1 // num2 if num2 != 0 else "Error: Division by zero!"
        case "%":
            answer = num1 % num2 if num2 != 0 else "Error: Division by zero!"
        case "**":
            answer = num1 ** num2
        case _:
            answer = "Invalid operator!"

    print("Result:", answer, "\n")

Tip: use python --version to confirm you have Python 3.10+ before using match.

Quick teaching tips
  • Always validate user input (numbers and operator). Consider using try/except to catch non-integer input.
  • For decimals, use float() instead of int().
  • Explain division behavior: / vs //.
  • Show how boolean arithmetic works (True = 1, False = 0) as a short exercise.
At-a-glance

Operators supported: + - * / // % **. Use exit to stop the loop versions.

Good exercises
  1. Allow decimal input and round the result to 2 decimal places.
  2. Accept full expression input like 10 * 5 and parse it.
  3. Add a history log of previous calculations.
  4. Build a simple GUI using tkinter (desktop) or a web front-end.
Safety & validation

Never use eval() on raw user input — it can run arbitrary code. Instead parse or validate inputs yourself.
