Python Higher-Order Functions Notes

A. Definition & Explanation

A higher-order function is a function that either:

  • Takes one or more functions as arguments, or
  • Returns a function as its result.

This allows for more flexible, reusable, and modular code.

Key Points:

  • Functions in Python are first-class objects—they can be passed around like integers, strings, or lists.
  • Higher-order functions enable functional programming patterns, like applying operations on other functions.

Example – Passing a Function as an Argument:

def greet(name):
    return f"Hello, {name}!"

def custom_greeting(func, name):
    return func(name)  # Calls the passed-in function

print(custom_greeting(greet, "Alice"))  # Output: Hello, Alice!

Example – Returning a Function:

def multiplier(n):
    def multiply(x):
        return x * n
    return multiply

double = multiplier(2)  # Returns a function that doubles a number
print(double(5))        # Output: 10

B. Uses and Applications (When to Use)

  • Write cleaner, reusable code.
  • Dynamically modify behavior without rewriting functions.
  • Functional programming: apply operations over sequences or compose functions.

Advantages:

  • Avoids code repetition.
  • Makes your program more flexible and extensible.
  • Often used in callbacks, event handling, and data processing pipelines.

C. Real-Life Examples

1. Applying Discounts or Tax Calculations Dynamically:

def apply_discount(price, discount_func):
    return price - discount_func(price)

def ten_percent(price):
    return price * 0.10

print(apply_discount(100, ten_percent))  # Output: 90.0

2. Data Processing Using map, filter, and reduce:

from functools import reduce

numbers = [1, 2, 3, 4, 5]

# map: double each number
doubled = list(map(lambda x: x*2, numbers))

# filter: keep only even numbers
evens = list(filter(lambda x: x % 2 == 0, numbers))

# reduce: sum all numbers
total = reduce(lambda x, y: x + y, numbers)

print(doubled)  # [2, 4, 6, 8, 10]
print(evens)    # [2, 4]
print(total)    # 15

3. Custom Validation Functions in Web Forms:

def validate_input(input_value, validation_func):
    if validation_func(input_value):
        return "Valid input"
    else:
        return "Invalid input"

def is_positive(number):
    return number > 0

print(validate_input(10, is_positive))  # Output: Valid input
print(validate_input(-5, is_positive))  # Output: Invalid input
✅ Summary:
- Higher-order functions accept functions as arguments or return functions.
- They make your code cleaner, reusable, and modular.
- Common use cases: data processing, dynamic behaviors, and functional programming.
