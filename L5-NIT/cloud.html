14. Hypervisor Types (3 Marks)
Type-1 Hypervisor: Runs directly on hardware Example: VMware ESXi, Microsoft Hyper-V

Type-2 Hypervisor: Runs on top of an operating system Example: VMware Workstation, Oracle VirtualBox
15. High Availability (HA) (3 Marks)
HA pools hosts and virtual machines into a cluster. If a host fails, HA automatically restarts the affected VMs on other hosts to minimize downtime.
16. Choosing a Cloud Service Provider (3 Marks)
Important factors include:
  • Security and compliance certifications
  • SLA uptime guarantees
  • Pricing models and hidden costs
  • Data sovereignty and location
17. Software-Defined Datacenter (SDDC) (4 Marks)
SDDC is an architecture where compute, storage, and networking are fully virtualized and managed by software. Traditional datacenters rely on hardware configuration, while SDDC uses automated policy-based management, making infrastructure more flexible and scalable.

SECTION B – Attempt Any 3 Questions (30 Marks)

Question 18
GlobalTech Corporation is planning to build a new enterprise datacenter to host its critical business applications, storage systems, and virtualization infrastructure. Before deploying IT equipment, the company must ensure that the physical environment is properly designed to support power, cooling, and security requirements.

As part of the infrastructure planning team, you have been assigned to evaluate and prepare the physical environment for the new datacenter facility.
a) Outline the key steps involved in conducting a site survey. (4 Marks)
1. Physical Location Analysis: Evaluate environmental risks such as flood plains, seismic zones, and accessibility.

2. Power Availability: Ensure the location supports dual power grids and capacity for backup generators.

3. Connectivity: Verify availability of multiple fiber links and ISP providers to ensure carrier neutrality.

4. Security Assessment: Evaluate neighborhood safety, perimeter security, and controlled access points.
b) Describe the significance of implementing Hot and Cold Aisle Containment in the infrastructure design. (3 Marks)
Hot/Cold aisle containment prevents the mixing of hot exhaust air and cold intake air inside the datacenter. This improves cooling efficiency, reduces energy consumption, and prevents hot spots that could damage server equipment.
c) List three essential physical security measures implemented in a datacenter. (3 Marks)
1. Biometric Access Control (fingerprint or retina scanning).
2. 24/7 CCTV surveillance systems.
3. Mantrap entry systems or dedicated security guards.
Question 19
TechSolutions Ltd currently hosts its web applications on local physical servers. Due to high maintenance costs, limited scalability, and increasing user demand, the company plans to migrate its web applications to a public cloud provider.
a) Compare the three major cloud service models: IaaS, PaaS, and SaaS. (6 Marks)
IaaS (Infrastructure as a Service): The provider supplies infrastructure such as servers, storage, and networking. The customer manages the operating system, applications, and data. Example: AWS EC2.

PaaS (Platform as a Service): The provider supplies infrastructure plus the operating system and runtime environment. The customer focuses on applications and data. Example: Google App Engine.

SaaS (Software as a Service): The provider manages the entire stack including infrastructure, software, and maintenance. The user simply accesses the application through the internet. Example: Gmail or Microsoft Office 365.
b) Give two situations where an organization should choose IaaS instead of SaaS. (2 Marks)
1. When full control over the operating system and system configurations is required.
2. When running legacy or specialized applications that are not available as SaaS.
c) Explain the role of Service Level Agreements (SLAs) when selecting a cloud provider. (2 Marks)
SLAs are formal contracts that define expected service levels such as uptime guarantees (e.g., 99.9%), support response times, and penalties for downtime. They ensure reliability and accountability from the cloud provider.
Question 20
Your organization is implementing VMware vSAN for its storage infrastructure.
a) Explain the architecture of a vSAN Disk Group including Cache Tier and Capacity Tier. (4 Marks)
A vSAN Disk Group consists of one caching device and multiple capacity devices.

Cache Tier: Handles high-speed I/O operations. In hybrid setups, about 30% is used for write buffering and 70% for read caching.

Capacity Tier: Stores the persistent data blocks and provides the main storage capacity for virtual machines.
b) Define "Failures to Tolerate (FTT)" and explain its impact on storage consumption. (3 Marks)
FTT defines how many host or disk failures the cluster can tolerate without losing data.

Higher FTT values require additional storage because data redundancy is increased. For example:
• FTT=1 using RAID-1 mirrors data and consumes approximately 2× storage.
• FTT=1 using RAID-5 requires about 1.33× storage overhead.
c) What is the difference between Hybrid vSAN and All-Flash vSAN? (3 Marks)
Hybrid vSAN: Uses SSD drives for cache and traditional HDDs for capacity. It is more affordable and suitable for general workloads.

All-Flash vSAN: Uses SSDs for both cache and capacity. It delivers higher performance and supports features such as deduplication and compression.
Question 21
XYZ Enterprise has recently virtualized its servers and deployed multiple ESXi hosts in its datacenter. The organization plans to implement vCenter Server Appliance (VCSA) for centralized management.
a) Describe the procedure for deploying the vCenter Server Appliance (VCSA). (4 Marks)
1. Download the vCenter Server Appliance ISO installer.

2. Mount the ISO and launch the installer from a client computer.

3. Stage 1: Deploy the OVA template to a target ESXi host and configure hostname, IP address, and root password.

4. Stage 2: Configure the appliance settings including SSO domain (vsphere.local), NTP configuration, and licensing.
b) Explain the function of Single Sign-On (SSO) in the vSphere environment. (3 Marks)
Single Sign-On (SSO) is an authentication service that allows users to log in once and gain access to multiple vSphere components without re-authenticating. It supports internal domains such as vsphere.local and external identity sources like Active Directory or LDAP.
c) What is a "Datacenter Object" in the vCenter inventory hierarchy? (3 Marks)
The Datacenter Object is the top-level logical container within vCenter. It groups clusters, hosts, virtual machines, networks, and datastores belonging to a specific organizational location.
Question 22
ABC Company operates an on-premises datacenter and plans to integrate public cloud services to improve scalability and business continuity.
a) Define Hybrid Cloud and explain one key benefit. (3 Marks)
A Hybrid Cloud combines private on-premises infrastructure with public cloud services.

Benefit: Flexibility and scalability. Organizations can keep sensitive data on-premises while using public cloud resources for temporary computing demands.
b) Compare VPN and Direct Connect for connecting on-premises datacenters to the cloud. (5 Marks)
VPN: Uses encrypted tunnels over the public internet.
Advantages: Low cost and quick setup.
Disadvantages: Higher latency and variable performance.

Direct Connect: Uses a dedicated physical fiber connection to the cloud provider.
Advantages: Low latency, consistent performance, and higher security.
Disadvantages: Expensive and longer setup time.
c) What is Cloud Bursting? (2 Marks)
Cloud bursting is a configuration where an application runs in a private cloud or datacenter but automatically uses additional public cloud resources when demand exceeds local capacity.

SECTION C – Attempt Only One Question (15 Marks)

Question 23
You have been hired as the Lead Technician to set up a new server room for a financial institution in Kigali. The room is currently empty.

Task: Create a detailed technical plan covering the following aspects.
a) What environmental controls (power, cooling, fire suppression) will you install and why? (5 Marks)
Power: Install a Dual-conversion Online UPS to provide clean and stable power, and a backup generator for long power outages. Use redundant PDUs to distribute power safely to rack equipment.

Cooling: Install CRAC units with N+1 redundancy to ensure continuous cooling even if one unit fails. Implement a Hot/Cold aisle layout to optimize airflow and cooling efficiency. Add monitoring sensors for temperature and humidity.

Fire Suppression: Install a Clean Agent Gas system such as FM-200 or Novec 1230. These systems suppress fire without damaging electronic equipment, unlike water sprinklers.
b) Describe the process of mounting servers and networking gear into racks, including cable management best practices. (5 Marks)
1. Planning: Place heavy equipment such as UPS units and storage systems at the bottom of the rack for stability. Network switches are typically placed at the Top-of-Rack (ToR) or Middle-of-Row.

2. Mounting: Install rail kits securely inside the rack. Carefully slide the servers into the rails and secure them with screws to ensure stability.

3. Cabling: Use Velcro straps instead of zip ties to avoid damaging cables. Separate power cables on one side of the rack and data cables on the other to prevent electromagnetic interference (EMI). Label all cables clearly at both ends for easy troubleshooting and maintenance.
c) Briefly describe how you will install ESXi on the bare-metal servers and verify connectivity before handing over to the network administrator. (5 Marks)
1. Access the server using remote management tools such as IPMI or iDRAC, or connect a monitor and keyboard directly.

2. Boot the server using the ESXi installer media (USB or ISO image).

3. Follow the installation prompts by selecting the installation disk, choosing the keyboard layout, and setting the root password.

4. After installation and reboot, press F2 in the Direct Console User Interface (DCUI) to configure the Management Network, including Static IP address, subnet mask, gateway, and VLAN settings.

5. Verify connectivity using the Test Management Network option to ping the gateway and DNS servers. Finally, open a web browser from a laptop and access the ESXi Host Client using the host IP address.
Question 24
A university currently runs all its student databases and e-learning portals on five local physical servers. They plan to modernize their infrastructure by virtualizing the environment and connecting to a public cloud for backup and peak-load management.

Task: Develop a migration plan addressing the following.
a) How will you convert physical servers to virtual machines (P2V) using vCenter? (5 Marks)
1. Implement the 3-2-1 backup strategy (3 copies of data, 2 different storage media, 1 offsite copy).

2. Use the VMware vCenter Converter Standalone tool.

3. Select "Powered-on machine" as the source and enter the IP address and administrator credentials of the physical server.

4. Select "VMware Infrastructure virtual machine" as the destination and provide vCenter Server credentials.

5. Adjust disk sizes if required and enable Thin Provisioning to save storage space.

6. Stop critical services (such as database services) before the final synchronization to ensure data consistency. Complete the conversion and power on the VM to verify functionality.
b) Propose a connectivity method suitable for a university budget and justify your choice. (5 Marks)
Proposed Solution: Site-to-Site VPN using IPsec.

Justification: Universities often operate with limited budgets. Dedicated connections such as AWS Direct Connect or Azure ExpressRoute are expensive. A VPN is cost-effective and still provides strong encryption through IPsec, ensuring secure data transmission over the public internet.

Since the requirement is primarily for backup and peak-load workloads rather than constant high-throughput operations, VPN bandwidth is generally sufficient.
c) Explain how you will configure a backup strategy ensuring critical student data is replicated to the cloud. Mention the Recovery Point Objective (RPO). (5 Marks)
1. Deploy a backup solution such as Veeam or Nakivo integrated with cloud storage services like Amazon S3 or Azure Blob Storage.

2. Configure automated backup and replication jobs that periodically copy VM data from the on-premises datacenter to the cloud.

3. Define the Recovery Point Objective (RPO), which represents the maximum acceptable data loss. For student databases, an RPO of 15 minutes to 1 hour is recommended.

4. Schedule replication during off-peak hours and use incremental backups to transfer only changed data blocks, reducing bandwidth usage and improving efficiency.
