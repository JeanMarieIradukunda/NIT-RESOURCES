📹 LEARNING OUTCOME 5: TEST CCTV SYSTEM

Interactive Learning & Field Practical Guide

This is the first step where you confirm that all CCTV parts are correctly installed and ready for testing.
📦 Material (What is used in installation)
Definition: Physical items used to build the CCTV system.

Examples:

CCTV cameras (IP or analog)
Cables (Ethernet / coaxial)
Connectors (RJ45, BNC)
Power supply (adapters, PoE switch)
NVR/DVR (recording device)
Monitors

👉 Purpose: To ensure all required parts are available and correctly installed.
🔧 Equipment (Tools used during installation)
Definition: Devices used to install or support the CCTV system.

Examples:

Drill machine (for mounting cameras)
Screwdrivers
Cable tester
Network tools (crimping tool)
Ladder

👉 Purpose: To confirm installation is properly done and safe.


These are tools used to check if the CCTV system is working correctly.
🖥️ Hardware Testing Tools
Definition: Physical tools used to test devices and connections.

Examples:

Cable tester (checks cable continuity)
Multimeter (checks power voltage)
Network tester (checks LAN cables)
Monitor screen (checks video output)

👉 Purpose: To verify that physical devices and connections are working properly.
💻 Software Testing Tools
Definition: Programs used to test system performance and camera settings.

Examples:

NVR/DVR software interface
CCTV mobile apps
IP camera configuration tools
Network scanning tools (e.g., IP finder)

👉 Purpose: To check camera configuration, recording, and network connection.

🔌 Physical Connectivity
Definition: Checking actual cables and hardware connections.

What to check:

Power cables are connected
Ethernet/coax cables are properly plugged
Cameras are firmly mounted
No loose wires

👉 Purpose: To ensure all devices are physically connected and powered.
🌐 Logical Connectivity
Definition: Checking if devices communicate correctly through the network.

What to check:

Camera IP address is detected
NVR/DVR detects all cameras
Live video is visible on monitor/app
Network connection is stable

👉 Purpose: To confirm the system is communicating correctly.

🔩 Hardware Functional Testing
Definition: Testing physical devices to confirm they work properly.

What to test:

Cameras power ON
Cameras rotate (if PTZ type)
LEDs or IR night vision works
NVR/DVR records video

👉 Purpose: To ensure all physical components are functioning.
💾 Software Functional Testing
Definition: Testing CCTV system software functions.

What to test:

Live view works
Recording starts/stops correctly
Playback videos work
Motion detection works
Remote access (mobile app/web)

👉 Purpose: To confirm system software works correctly.

🎥 Visual Quality
Definition: How clear the video image is.

Check:

Image sharpness
Color accuracy
Low/no blur
Night vision clarity

👉 Good quality = clear identification of people/objects.
⏱️ Latency
Definition: Delay between real action and camera display.

Check:

Live video delay time
Smooth real-time viewing

👉 Low latency = faster real-time monitoring.
🔊 Audio Quality
Definition: Quality of sound recorded (if camera has mic).

Check:

Clear sound recording
No noise or distortion

👉 Purpose: To ensure voice/audio is understandable.
📡 NVR/DVR Throughput
Definition: How much video data the recorder can handle.

Check:

Number of cameras supported
Recording without lag
Smooth multi-camera display

👉 Purpose: To ensure system can handle full workload.

📄 Generate Testing Report
Definition: Document that records all testing results.

Includes:

System setup details
Test results (pass/fail)
Issues found
Fixes applied

👉 Purpose: To officially record system performance.
📊 Interpret Testing Report
Definition: Understanding and analyzing test results.

What to check:

Which tests passed or failed
Performance quality level
Problems that need fixing
Final system readiness

👉 Purpose: To decide if CCTV system is ready for use or needs repair.

📌 SUMMARY

CCTV testing ensures:

  • Devices are correctly installed
  • System is properly connected
  • Cameras and software work well
  • Video quality is clear and reliable
  • Final report confirms system readiness
