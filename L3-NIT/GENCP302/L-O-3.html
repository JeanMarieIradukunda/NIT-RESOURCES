🧠 Program Debugging Notes

🧠 LEARNING OUTCOME 3: PERFORM PROGRAM DEBUGGING

Debugging is the process of finding and fixing errors in a program so it works correctly.

When writing C programs, we usually go through three main steps:

Identify errors
Compile the program
Test and debug the program


1. 🔍 IDENTIFICATION OF ERRORS
✅ What is an error?

An error is a mistake in a program that prevents it from working correctly or producing the expected output.

📌 Types of Errors in C Programming
1. Syntax Errors

These are mistakes in the grammar of C language.

👉 Example:

#include 

int main() {
    printf("Hello World")   // missing semicolon
    return 0;
}

❌ Problem: Missing ;
💡 Fix: Add semicolon

printf("Hello World");
2. Runtime Errors

These happen while the program is running.

👉 Example:

int a = 10, b = 0;
printf("%d", a / b);

❌ Problem: Division by zero
💡 Fix: Ensure denominator is not zero

if (b != 0)
    printf("%d", a / b);
3. Logical Errors

Program runs but gives wrong results.

👉 Example:

int a = 5, b = 3;
int sum = a - b;  // wrong logic
printf("%d", sum);

❌ Expected: 8
❌ Output: 2
💡 Fix:

int sum = a + b;
🛠 Techniques of Correcting Errors
Read error messages carefully
Check spelling and symbols
Use printf() to trace values
Break program into small parts
Review logic step by step
View Error Handling Examples 

2. ⚙️ COMPILATION OF THE C PROGRAM
📌 What is compilation?

Compilation is the process of turning C code into machine code (0s and 1s) that the computer understands.

🔄 Steps of Compilation
Write program (.c file)
Preprocessing (handles #include)
Compilation (checks syntax)
Linking (connects libraries)
Execution (runs program)
💻 Types of Compilers
1. GCC (GNU Compiler Collection)
Most popular C compiler
Used in Linux, Windows (via MinGW)

👉 Example:

gcc program.c -o program
2. Turbo C Compiler
Old compiler
Used in basic learning environments
3. Clang Compiler
Modern compiler
Fast and gives clear error messages
🧪 Compilation Techniques
1. Single File Compilation
gcc main.c -o main
2. Compile with Warnings
gcc -Wall main.c -o main

👉 Helps detect possible mistakes

3. Debug Mode Compilation
gcc -g main.c -o main

👉 Used for debugging with tools like GDB


3. 🧪 TEST OF THE C PROGRAM
📌 What is testing?

Testing means running the program with different inputs to make sure it works correctly.

🎯 Design of Effective Test Cases

A test case includes:

Input
Expected output
Actual output
📌 Example Program:
int add(int a, int b) {
    return a + b;
}
🧪 Test Cases:
Input (a, b)	Expected Output
2, 3	5
0, 0	0
-1, 1	0
🧠 Good Test Case Design Rules
Test normal values (e.g., 5 + 3)
Test boundary values (e.g., 0, -1)
Test extreme values (very large numbers)
Test wrong input (if possible)


🐞 DEBUGGING TECHNIQUES

Debugging means finding and fixing errors in a program.

🔧 Common Debugging Techniques
1. Using printf Debugging

Add print statements to check values.

printf("Value of a = %d\n", a);
2. Step-by-Step Execution

Run program line by line mentally or using debugger tools.

3. Using Debugger Tools (like GDB)

Allows you to:

Pause program
Check variables
Run step by step
4. Commenting Code

Disable parts of code to isolate error:

// printf("This line is disabled");
5. Code Review

Carefully re-read code or ask someone else to check it.

📌 SUMMARY
Errors in C: Syntax, Runtime, Logical
Compilation: Converts C code into machine code
Compilers: GCC, Clang, Turbo C
Testing: Using input/output cases to verify correctness
Debugging: Finding and fixing program errors using tools and techniques
