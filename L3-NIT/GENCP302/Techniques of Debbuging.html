🐞 TECHNIQUES OF DEBUGGING (WITH SIMPLE EXAMPLES)

Debugging means finding and fixing errors in a program.

1. 📖 Read Error Messages Carefully
💡 Idea:

When you compile a program, the compiler shows error messages. These messages help you know what is wrong.

❌ Example:
#include 

int main() {
    printf("Hello World")   // missing semicolon
    return 0;
}
⚠️ Compiler message:
error: expected ';' before 'return'
💡 Meaning:

The compiler is telling you that a semicolon is missing.

✅ Fix:
printf("Hello World");


2. 🔤 Check Spelling and Symbols
💡 Idea:

Small spelling mistakes or wrong symbols can break a program.

❌ Example:
#include 

int main() {
    prinf("Hello");  // wrong spelling
    return 0;
}
💡 Problem:

prinf is incorrect (should be printf)

⚠️ Error:
undefined reference to 'prinf'
✅ Fix:
printf("Hello");


3. 🧪 Use printf() to Trace Values
💡 Idea:

We insert printf() to check what values variables have.

❌ Example (wrong result):
#include 

int main() {
    int a = 5, b = 10;
    int sum = a - b;  // wrong logic

    printf("Sum = %d", sum);
    return 0;
}
🐞 Debugging using printf():
printf("a = %d\n", a);
printf("b = %d\n", b);
printf("sum = %d\n", sum);
💡 What we learn:

We see the values and realize subtraction is wrong.

✅ Fix:
int sum = a + b;


4. 🧩 Break Program into Small Parts
💡 Idea:

If a program is long, divide it into smaller parts and test each part.

❌ Example:
int main() {
    int a = 10, b = 20;
    int sum = a + b;
    int avg = sum / 2;
    printf("%d", avg);
}
🐞 Debug step-by-step:
Step 1: Test sum only
printf("Sum = %d", sum);
Step 2: Test average separately
printf("Average = %d", avg);
💡 Benefit:

You find exactly where the problem is.


5. 🧠 Review Logic Step by Step
💡 Idea:

Read your code slowly and check if the logic makes sense.

❌ Example:
int a = 10, b = 20;
int max;

if (a > b)
    max = b;
else
    max = a;
💡 Problem:

Logic is reversed.

🧠 Step-by-step thinking:
If a > b → max should be a ❌ (but code says b)
If a < b → max should be b ❌ (but code says a)
✅ Fix:
if (a > b)
    max = a;
else
    max = b;

Technique	What you do
