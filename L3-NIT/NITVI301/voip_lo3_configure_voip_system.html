LEARNING OUTCOME 3: CONFIGURE VOIP SYSTEM

Configuration of Router Based VoIP system

This is the process of setting up a router so that VoIP devices can connect and make calls.

  • The router manages both internet traffic and voice traffic.
  • Proper configuration ensures IP phones, computers, and VoIP servers communicate properly.
  • Includes assigning IP addresses, configuring gateways, setting passwords, and sometimes prioritizing voice traffic (QoS).

1. VoIP Essential Router-Based Configurations

These are the minimum settings a router must have to support VoIP:

1. Interface

  • This is the connection point on the router where devices connect.
  • Each interface is given an IP address and subnet so that devices on the network can communicate.
  • Example: LAN interface 192.168.1.1 connects phones and computers.
Steps:
  1. Enter configuration mode: enable → configure terminal
  2. Select interface: interface GigabitEthernet0/1
  3. Assign IP: ip address 192.168.1.1 255.255.255.0
  4. Activate interface: no shutdown

2. IP Default Gateway

  • This is the IP address the router uses to send traffic to other networks or the internet.
  • It allows VoIP calls to reach external SIP servers or other networks.
  • Example: WAN gateway 203.0.113.1 for internet connectivity.
Command: ip route 0.0.0.0 0.0.0.0 203.0.113.1

3. Passwords

  • Routers must have secure passwords to protect configuration and access.
  • Admin accounts should have strong passwords to prevent unauthorized access.
  • Optional: Secure access via SSH instead of Telnet for safer management.
Steps:
  1. Set admin user: username admin privilege 15 secret MyStrongPassword123
  2. Enable SSH: ip domain-name office.local → crypto key generate rsa → ip ssh version 2
  3. Disable Telnet: line vty 0 4 → transport input ssh → login local

2. Creating VoIP Pool for Voice System

A VoIP Pool is a range of IP addresses reserved for VoIP devices (like IP phones) in a network.

  • The router or DHCP server assigns IP addresses automatically to VoIP devices from this pool.
  • Ensures IP phones have proper network addresses for calling and registration.

Key Elements of a VoIP Pool

A. Pool Name

  • This is a label for the pool so you can identify it easily.
  • Example: VOIP_POOL_01
  • Purpose: Helps organize multiple pools if you have different office areas or departments.

B. Pool Address (Network Address and Subnet Mask)

  • Network Address: The first IP in the range used for the VoIP pool.
  • Subnet Mask: Defines how many IP addresses are available in the pool.
Parameter Example
Network Address 192.168.10.0
Subnet Mask 255.255.255.0
  • This provides addresses from 192.168.10.1 to 192.168.10.254 for phones.
  • Purpose: Defines the range of IP addresses available for VoIP devices.

C. Voice Option (Option 150)

  • DHCP Option 150 tells IP phones where the VoIP server (IP PBX) is located.
  • Phones use this IP to register with the PBX and make calls.
option 150 ip 192.168.10.100
  • Purpose: Automatically directs IP phones to the PBX server for registration, so you don’t have to configure each phone manually.

3. Sample VoIP Pool Configuration (Cisco-style)

ip dhcp pool VOIP_POOL_01
network 192.168.10.0 255.255.255.0
default-router 192.168.10.1
option 150 ip 192.168.10.100
  • ip dhcp pool VOIP_POOL_01 → Name of the pool
  • network 192.168.10.0 255.255.255.0 → Defines the address range
  • default-router 192.168.10.1 → Gateway for phones
  • option 150 ip 192.168.10.100 → Points phones to the VoIP server

Creating a VoIP pool automates IP assignment and ensures all phones connect to the VoIP server correctly, simplifying network management in offices or call centers.

4. Configuration of Phone Line Number

This is the process of assigning a telephone number (extension) to an IP phone in a VoIP system.

In VoIP networks, each phone must have a unique extension number so users can call each other inside the organization.

Example

  • Reception → Extension 100
  • Manager → Extension 101
  • Office 1 → Extension 102

Purpose

  • Allows users to dial and communicate internally
  • Helps the VoIP server (IP PBX) identify each phone
  • Organizes communication inside the network

Phone Directory for IP Phone

A phone directory is a list of users, extension numbers, and sometimes names stored in the VoIP system.

IP phones can access this directory to find and call other users easily.

Example Directory

Name Extension
Reception 100
Manager 101
Finance 102
IT Office 103

Purpose

  • Makes it easy to search for contacts
  • Reduces the need to remember extension numbers
  • Improves communication efficiency

Some systems allow the directory to be stored on the PBX server or downloaded automatically to IP phones.

ePhones

ePhones (Ethernet Phones) are IP phones registered to a VoIP router or VoIP server.

In many systems (especially Cisco VoIP systems), an ePhone configuration identifies a specific IP phone device.

Each ePhone usually includes:

  • MAC address of the phone
  • Extension number
  • Phone type or model
  • Button configuration

Example Configuration (Cisco-style)

ephone 1
mac-address 001B.D5A1.2345
type 7960
button 1:1

Explanation:

  • ephone 1 → Identifies the phone device
  • mac-address → Unique hardware address of the IP phone
  • type 7960 → Model of the phone
  • button 1:1 → Assigns extension 1 to button 1

Purpose

  • Registers the IP phone with the VoIP system
  • Connects the physical device to a phone number
  • Allows the phone to make and receive calls

5. Configuration of Phone Call Manager (CME)

1. Phone Call Manager (CME) Configuration

Call Manager Express (CME) is a Cisco router-based VoIP system. It manages IP phones, extensions, and internal calls without a separate server.

The configuration includes:

  • Activating telephony services
  • Registering phones (ephones)
  • Assigning directory numbers (DNs / extensions)
  • Auto-assigning line numbers
  • Testing the system

2. Telephone Services

The telephony-service command activates VoIP services on the router.

telephony-service
ip source-address 192.168.1.1 port 2000
  • ip source-address → The router IP that phones use to register
  • port 2000 → Default SIP signaling port for IP phones

Purpose:

  • Allows the router to handle IP phone registration
  • Enables internal VoIP calling features

3. Maximum Number of ePhones

ePhones are the actual IP phones connected to the router. You must configure the maximum number of phones the router can support.

max-ephones 10
  • The router can register up to 10 IP phones.

Purpose:

  • Prevents system overload
  • Helps plan capacity for office phones

4. Maximum Number of Directory Numbers (DN)

Directory Numbers (DNs) are the extension numbers assigned to phones. You can define the maximum number of extensions in the system.

max-dn 20
  • The system can create 20 extensions.

Purpose:

  • Organize the phone system
  • Assign extensions to phones automatically

5. Auto Assignment of Line Numbers

CME can automatically assign extension numbers to phones when they register.

auto assign 1 to 10
  • Phones registering will receive extensions 1–10 automatically.

Purpose:

  • Reduces manual configuration
  • Makes installation faster

6. Testing the System

After configuration, you need to verify everything works correctly.

Steps:
  1. Check registered phones: show ephone registered
  2. Check assigned directory numbers: show ephone-dn
  3. Test call between extensions (Example: Extension 101 → Extension 102)
  4. Test VoIP features: Call hold, transfer, voicemail, auto-assigned extensions

Purpose:

  • Ensure phones are connected
  • Confirm calls can be made internally
  • Verify extensions and features work correctly

7. Sample CME Configuration (Cisco CLI)

telephony-service
ip source-address 192.168.1.1 port 2000
max-ephones 10
max-dn 20
auto assign 1 to 10
  • telephony-service → Activates CME
  • ip source-address → Router IP for phone registration
  • max-ephones 10 → Max 10 phones
  • max-dn 20 → Max 20 extensions
  • auto assign 1 to 10 → Automatic assignment of extensions

6. Configuration of VoIP System Server Based (IP PBX / PABX / PBX)

A VoIP Server-Based System is a centralized voice system that manages calls for an organization.

IP PBX: Internet Protocol Private Branch Exchange (VoIP system using IP network)

PABX / PBX: Traditional private branch exchange for analog or digital lines

The server handles:

  • User registration
  • Call routing
  • Extension management
  • External calls
  • Advanced features like IVR, FollowMe, and CRM integration

1. VoIP Users

People or devices using the VoIP system.

  • Can be IP phones, softphones, or mobile devices.
  • Each user is assigned a unique extension (Directory Number).
  • Purpose: Identify callers and manage calls internally.

2. Frequency for Voice

Voice is analog by nature, but VoIP converts it to digital signals.

  • Human voice ranges roughly 300–3400 Hz, which is enough for conversation.
  • VoIP uses compression and sampling to send voice efficiently over networks.

3. Frame and Packet (Voice Packet)

  • Frame: Small unit of digital data in a network.
  • Packet: A chunk of data (including voice) sent over IP networks.
  • Voice is encoded, divided into packets, and sent over the network.
  • Purpose: Allows real-time voice communication over IP networks.

4. Layered Model

VoIP uses standard network protocols and layered models:

Layer Function
Application VoIP software / IP PBX services
Transport UDP (User Datagram Protocol)
Network Internet Protocol (IP)
Data Link / Physical Ethernet / Switch / Cabling

Key Protocols:

  • IP → Delivers packets between devices
  • UDP → Lightweight transport for real-time traffic
  • RTP → Real-Time Transport Protocol carries the actual voice data

Purpose: Ensures voice is sent reliably and quickly with minimal delay.

5. Inter PBX Circuits

Connections between two PBX servers (branches). Allows calls between offices without using PSTN (saves costs).

Example: PBX Office A ↔ PBX Office B

6. Switchboard Facilities

Central point for call management and operator assistance. Routes incoming calls to the correct extension.

Example: Reception answers incoming call → transfers to Sales / Support.

7. Extension Facilities

Internal numbers (DNs) for each user or phone.

Example: 100 = Reception, 101 = Manager, 102 = IT

Purpose: Enables internal communication without public lines.

8. Public Exchange Services

Access to external phone networks (PSTN / GSM). Enables national and international calling.

9. Alarms

System alerts to notify issues:

  • Fuse Alarm: Triggered when a protective fuse blows.
  • Mains Fail: Occurs when main power supply fails. The system may switch to backup batteries.
  • Purpose: Ensure system reliability and maintenance.

10. Tones

Tone Meaning
Dial tone Phone ready to dial
Ring tone Destination phone ringing
Busy tone Line is busy

11. Numbering

Assigns internal extensions logically.

Department Extensions
Reception 100–109
Sales 110–119
IT 120–129

12. Local Calls

Calls made within the same PBX system. These are internal calls between extensions without using external phone lines.

Example: Extension 101 → Extension 102

Purpose: Allows seamless internal communication between employees or departments, reducing the need for public lines and minimizing call costs.

13. Outgoing Exchange Line / Inter PBX Calls

Calls that go outside the local PBX or to another branch PBX. These calls are routed through exchange lines or inter-PBX circuits.

Example: Extension 101 → PSTN → Client Phone / Extension at another branch PBX

Purpose: Enables external communication with clients, suppliers, or other offices, while integrating multiple PBX systems.

14. Exchange Line Relay Sets

Hardware relays that connect internal PBX extensions to external telephone lines. Can operate manually or automatically to switch calls between internal and external lines.

Purpose: Ensures calls can flow seamlessly from internal extensions to external networks, providing reliability and flexibility in call management.

15. Interactive Voice Response (IVR)

An automated phone menu that guides callers using voice prompts or keypad input, helping route calls efficiently without human intervention.

Example Menu:

  • Press 1 for Sales
  • Press 2 for Support
  • Press 3 for Accounts

Purpose: Automates call distribution, reduces workload for receptionists, ensures calls reach the correct department, and improves customer service efficiency.

16. Call Routing Methods

Method Description Explanation / Example
Round Robin Calls distributed one by one sequentially Calls are assigned to agents in order.
Example: Call 1 → Agent A, Call 2 → Agent B, Call 3 → Agent C, Call 4 → Agent A (repeat).
Purpose: Ensures equal distribution among all agents.
Regular Calls go to a fixed first destination Calls are sent to a primary agent first, then forwarded if busy.
Example: All incoming sales calls → Sales Agent 1 → if busy → Sales Agent 2.
Purpose: Ensures calls reach the main contact first.
Uniform Calls go to the agent idle the longest Calls are routed to the agent who has been free for the longest time.
Example: Agent A idle 10 min, Agent B idle 5 min → next call goes to Agent A.
Purpose: Balances workload fairly among agents.
Simultaneous Rings multiple phones at the same time All designated agents’ phones ring simultaneously.
Example: Call → Agent A + Agent B + Agent C simultaneously → first to answer takes the call.
Purpose: Reduces waiting time for callers.
Weighted Calls distributed based on agent priority Calls are assigned based on predefined agent priorities or skill levels.
Example: Senior Agent → 70% of calls, Junior Agent → 30% of calls.
Purpose: Ensures experienced agents handle more calls or critical tasks.

17. FollowMe

Forwards calls to multiple devices until answered.

Example: Office Phone → Mobile → Home Phone → Voicemail

Purpose: Never miss important calls.

18. Customer Relationship Management (CRM) Software

Integration with VoIP allows automatic display of customer information when a call arrives.

Example: Salesforce, HubSpot, Zoho CRM

Purpose:

  • Improves customer service
  • Tracks all interactions

7. Configuration of Calls Routing

Call Routing is the process of defining how incoming calls are directed inside a VoIP or PBX system.

It ensures that calls reach the right person, department, or device efficiently.

Interactive Voice Response (IVR)

IVR is an automated phone menu that interacts with callers using voice prompts or keypad input.

Example:

  • Welcome to ABC Company
  • Press 1 for Sales
  • Press 2 for Support
  • Press 3 for Accounts
  • Press 0 for Reception

Purpose:

  • Automates call handling
  • Reduces workload on receptionists
  • Ensures calls go to the correct department

Call Routing Methods

Different methods define how incoming calls are distributed among agents or extensions:

a) Round Robin

Calls are distributed in order, one after another.

Example:

  • Call 1 → Agent A
  • Call 2 → Agent B
  • Call 3 → Agent C
  • Call 4 → Agent A (repeat)

Purpose: Equal distribution among agents.

b) Regular Routing

Calls are sent to a fixed destination first, then forwarded if busy.

Example:

  • All incoming sales calls → Sales Agent 1 → If busy → Sales Agent 2

Purpose: Ensure calls reach the main contact first.

c) Uniform

Calls go to the agent who has been idle the longest.

Example:

  • Agent A idle 10 min, Agent B idle 5 min → next call goes to Agent A

Purpose: Balance workload fairly.

d) Simultaneous

Rings multiple phones at the same time. First agent to answer takes the call.

Example:

  • Call → Agent A + Agent B + Agent C simultaneously

Purpose: Reduce waiting time.

e) Weighted

Calls are distributed based on priority or skill level.

Example:

  • Senior Agent → 70% of calls
  • Junior Agent → 30% of calls

Purpose: Ensure experienced agents handle most calls.

FollowMe

FollowMe is a feature that forwards calls to multiple numbers until answered.

Example Sequence:

  • Call → Office Phone
  • If not answered → Mobile Phone
  • If not answered → Home Phone
  • If not answered → Voicemail

Purpose:

  • Never miss important calls
  • Allows users to answer calls on multiple devices

Customer Relationship Management (CRM) Integration

CRM software tracks customer information and integrates with the VoIP system.

How it works:

  • Caller dials company → VoIP system identifies the caller → CRM displays customer info on agent’s screen.

Purpose:

  • Improve customer support
  • Track communication history
  • Allow agents to respond quickly and efficiently

Examples:

  • Salesforce
  • HubSpot
  • Zoho CRM

8. Testing of VoIP System

VoIP system testing is the process of checking if the voice network works correctly and delivers clear voice communication. It measures network performance and verifies that calls work correctly.

Main parameters tested:

  • Speed (Bandwidth)
  • Latency
  • Jitter
  • Packet Loss
  • Quality of Service (QoS)

Speed (Bandwidth)

Speed or bandwidth is the amount of data that can travel through the network per second. VoIP requires enough bandwidth to carry voice packets.

Example:

  • One VoIP call may require 80–100 kbps.

Purpose:

  • Ensure the network can support multiple voice calls.
  • Example test tools: Speed test tools, Network monitoring software

Latency

Latency is the time it takes for voice packets to travel from the caller to the receiver. In simple terms: Delay between speaking and hearing the voice.

Good VoIP latency should be:

QualityLatency
Excellent< 150 ms
Acceptable150–300 ms
Poor> 300 ms

High latency causes echo and delay during conversation.

Jitter

Jitter is the variation in packet arrival time. Voice packets should arrive at regular intervals. Irregular arrival causes distorted or broken voice.

Example:

  • Packet 1 → arrives on time
  • Packet 2 → arrives late
  • Packet 3 → arrives early

Good VoIP jitter value:

QualityJitter
Good< 30 ms

Packet Loss

Packet loss occurs when some voice packets are lost during transmission.

Example:

  • Sent packets → 1 2 3 4 5
  • Received → 1 2 4 5
  • Packet 3 lost

This causes:

  • Missing audio
  • Choppy voice
  • Call interruptions

Good VoIP packet loss should be:

QualityPacket Loss
Good< 1%

Quality of Service (QoS)

QoS is a network technique that prioritizes voice traffic over other data. This ensures voice packets are sent faster than normal data traffic.

Example:

  • Network Traffic ↓
  • Router Priority:
    1. Voice packets
    2. Video
    3. Normal data

Purpose:

  • Improve voice clarity
  • Reduce delay and packet loss

VoIP Router-Based Testing

This testing is done when the VoIP system is managed by a router with Call Manager Express (CME).

Local Call Testing

Example test: Extension 101 → Extension 102

Check:

  • Call connection
  • Voice clarity
  • No delay
  • No packet loss

Commands that can be used in routers:

  • ping
  • show voice call summary
  • show ephone registered

Purpose: Verify the router handles VoIP calls correctly.

VoIP Server-Based Testing

In this setup, a VoIP server (IP PBX) manages calls.

Examples of VoIP servers:

  • IP PBX systems
  • VoIP call servers

Testing includes:

Local Calls

Test calls between internal extensions.

Example: Extension 201 → Extension 202

Verify:

  • Phone rings correctly
  • Clear voice communication
Outgoing / External Calls

Test calls from the VoIP system to external phone networks.

Example: Office Extension → PSTN → Mobile Phone

Verify:

  • Connection success
  • Voice quality
  • Correct routing
Call Routing Testing

Check if calls are sent to the correct destination.

Example: Customer Call → IVR → Support Department

Verify:

  • Correct department receives the call
  • Routing rules work properly
FollowMe Testing

Test the FollowMe feature.

Example:

  • Call → Office Phone
  • No answer → Mobile Phone
  • No answer → Voicemail

Verify:

  • Call forwarding works correctly
  • No missed calls
