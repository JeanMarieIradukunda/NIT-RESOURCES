Operating Systems (OS) Notes

Types of Operating Systems (OS)

You already know operating systems like Windows, Linux, and macOS. These are names of operating systems.

The following are types (categories) of operating systems, classified based on how they work and how they manage tasks.

Think of it like this:
  • Windows, Linux → Names of operating systems
  • Batch, Real-time, Time-sharing → Ways an OS operates

1. Batch Operating System

  • Processes many jobs together without user interaction.
  • Jobs are submitted and stored in a queue.
  • Executed one after another automatically.
Examples:
  • Payroll systems
  • Old bank transaction processing
  • Mainframe computers
👉 Not interactive | 👉 Slow response

2. Distributed Operating System

  • Manages multiple computers working together as one system.
  • Tasks and resources are shared across machines.
Examples:
  • Cloud computing
  • Google servers
  • University networks
👉 Many computers, one system

3. Real-Time Operating System (RTOS)

  • Responds to inputs within a fixed time limit.
  • Correctness depends on timing.

a) Hard Real-Time Operating System

  • Deadlines must never be missed.
  • Failure can be dangerous.
  • Heart pacemaker
  • Aircraft navigation system
  • Nuclear reactor control
👉 Very strict timing

b) Soft Real-Time Operating System

  • Deadlines are important but flexible.
  • Performance degrades if delayed.
  • Video streaming
  • Online games
  • Multimedia systems
👉 Timing is important, but flexible

4. Network Operating System (NOS)

  • Allows computers to share resources over a network.
  • Each computer runs its own OS.
  • Windows Server
  • Linux Server
  • Novell NetWare
👉 Each computer is independent

5. Time-Sharing Operating System

  • Many users use the computer at the same time.
  • CPU time is divided into small time slices.
  • Unix
  • Linux
  • Modern Windows
👉 Many users, fair CPU sharing

6. Mobile Operating System

  • Designed for smartphones and tablets.
  • Manages touch input, apps, battery, and wireless communication.

Key Characteristics:

  • Touch-based interface
  • Battery efficient
  • Supports app stores
  • Handles sensors (camera, GPS, fingerprint)
Examples:
  • Android
  • iOS
👉 Used on phones and tablets

7. Embedded Operating System

  • Built into a device to perform one specific function.
  • Not designed for general-purpose use.
  • Often works in the background.

Key Characteristics:

  • Small size
  • Fast and reliable
  • Limited user interaction
  • Designed for a specific task
Examples:
  • Washing machines
  • Microwave ovens
  • Digital cameras
  • Traffic lights
  • Smart TVs
👉 Built inside devices

IMPORTANT FINAL CLARIFICATION 🚀

  • Windows, Linux, macOS are NOT types.
  • They are operating systems that can belong to many types.
  • Linux → Time-sharing, Network OS, Distributed OS
  • Windows Server → Network OS
  • Unix → Time-sharing OS

Operating System File System

1. File System Role

Description:

  • A file system is a way an operating system organizes and stores data on storage devices (hard drives, SSDs, USB drives).
  • It manages files and directories so users and applications can read, write, and access data efficiently.

Key Roles:

  • Organize Data: Creates a hierarchy (folders, subfolders) to keep files manageable.
  • Store and Retrieve: Tracks where files are physically stored on the disk.
  • Access Control: Ensures security, giving permissions (read/write/execute) to users.
  • Data Integrity: Prevents data loss and corruption.
Real-life example:
  • Your Documents folder on Windows, where you store Word, Excel, or PDF files.
  • Photos on a USB drive or memory card are stored in folders by a file system.

2. Major Types of File Systems in Microsoft Windows

a) File Allocation Table (FAT)

Description: One of the oldest file systems. Uses a table to track where files are stored on the disk.

Variants:

  • FAT12 – Used in very small storage devices like floppy disks.
  • FAT16 – Supports larger disks, used in early PCs.
  • FAT32 – Supports drives up to 2 TB; still used in USB drives and SD cards for compatibility.
Real-life example:
  • FAT32: USB drives used to transfer files between Windows, Mac, and cameras.
  • FAT12: Old floppy disks (rarely used now).

b) New Technology File System (NTFS)

Description: Modern Windows file system, designed for large storage and security. Supports permissions, encryption, compression, and large files.

Advantages:

  • Handles very large drives and files.
  • File permissions and security for different users.
  • Reliable due to journaling (records changes to prevent corruption).
Real-life example:
  • Windows system (C:) drive.
  • Software installations and system files.

3. Major Types of File Systems in Linux / Unix

a) Extended File System (ext4)

Description: The most widely used Linux file system. Supports large files, journaling, and high performance.

Real-life example:
  • Ubuntu and other Linux distributions use ext4 by default.

b) XFS: X File System

Description: High-performance file system optimized for large files and large disks.

Real-life example:
  • Used on enterprise Linux servers and databases.

c) Btrfs (B-tree File System)

Description: Modern Linux file system with advanced features such as snapshots, compression, and self-healing.

Real-life example:
  • Used in modern Linux systems and cloud environments.

4. Major Types of File Systems in macOS

a) Apple File System (APFS)

Description: Modern macOS file system optimized for SSDs. Supports encryption, snapshots, and fast performance.

Real-life example:
  • Default file system on macOS Big Sur and later.

b) HFS+: Hierarchical File System

Description: Older macOS file system used before APFS.

Real-life example:
  • Older Mac computers and backup drives.

5. Cross-Platform and Removable Media File Systems

a) exFAT: Extended File Allocation Table

Description: Designed for flash storage. Removes FAT32 file size limits and works across operating systems.

Real-life example:
  • Large USB flash drives and SD cards.

b) ISO 9660

Description: File system standard for optical media.

Real-life example:
  • CDs, DVDs, and ISO installation files.

6. Network and Advanced File Systems

a) Network File System (NFS)

Description: Allows files to be shared over a network, mainly in Linux and Unix environments.

Real-life example:
  • Shared folders on Linux servers.

b) Server Message Block (SMB / CIFS)

Description: Network file system used mainly by Windows.

Real-life example:
  • Windows file sharing on local networks.

c) ZFS

Description: Advanced file system with strong data integrity, snapshots, and storage pooling.

Real-life example:
  • Enterprise servers and data centers.

Key Points to Remember

  • Different operating systems use different file systems.
  • Some file systems focus on compatibility, others on security or performance.
  • Choosing the right file system improves reliability and efficiency.
