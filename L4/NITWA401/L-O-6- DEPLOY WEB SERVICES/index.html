Introduction to Web, Website, Webpage, and Web Server

Web

The web (World Wide Web) is a system that allows people to access information through the internet using browsers like Google Chrome or Mozilla Firefox.

Example: Searching information on Google or opening YouTube.
Website

A website is a collection of webpages connected together under one domain name.

Example:
  • YouTube
  • Wikipedia
Webpage

A webpage is a single page on a website.

Example: The “About Us” page on a school website.
Web Server

A web server is a computer or software that stores websites and delivers webpages to users through the internet or local network.

Example: Internet Information Services running on Windows Server 2016.

5.1 Installation of Web Server

1. Apache HTTP Server
  • Free and open-source web server
  • Commonly used on Linux and Windows
  • Supports PHP websites
Example: Hosting a school website using Apache.
2. Internet Information Services (IIS)
  • Web server developed by Microsoft
  • Works with Windows Server
  • Supports ASP.NET applications
Example: Hosting company websites on Windows Server 2016.
3. Nginx
  • Fast and lightweight web server
  • Good for high-traffic websites
Example: Used by streaming and social media platforms.
4. LiteSpeed Web Server
  • High-performance web server
  • Faster than Apache in some cases
Example: Used by hosting companies.
5. Apache Tomcat
  • Used for Java web applications
  • Supports JSP and Servlets
Example: Hosting Java-based student management systems.
Hardware Requirements

Before installing a web server on a Windows Server 2016 Domain Controller, ensure the computer meets the minimum hardware requirements.

Minimum Requirements
  • Processor: 1.4 GHz 64-bit CPU
  • RAM: 2 GB (Server with Desktop Experience)
  • Storage: 32 GB minimum (40–50 GB recommended)
  • Network: 1 Gbps Ethernet adapter recommended
  • Display: 1024 × 768 resolution or higher
Recommended for a Small Web Server
  • Dual-core 2.0 GHz processor or higher
  • 4 GB RAM or more
  • 50 GB free storage or more
  • Stable LAN/Internet connection
Example Server Specifications:
  • Dual-core processor
  • 4 GB RAM
  • 50 GB free storage
  • Static IP address
  • Windows Server 2016 Domain Controller
Software Requirements

The following software and network configurations are required before installing and configuring IIS.

  • Windows Server 2016 installed and updated
  • Administrator privileges
  • Internet Information Services (IIS) role enabled
  • TCP/IP configured with a static IP address
  • DNS configured correctly
  • Updated drivers and system patches
Firewall Configuration
  • Allow HTTP (Port 80)
  • Allow HTTPS (Port 443)
IIS Role Services
  • Web Server
  • Common HTTP Features
    • Default Document
    • Static Content
    • HTTP Errors
  • Application Development (if needed)
    • ASP.NET
    • .NET Extensibility
  • Security
    • Request Filtering
    • Windows Authentication
  • Management Tools
    • IIS Management Console
Perform Installation Process (IIS)
Steps to Install IIS on Windows Server 2016
Open Server Manager
Click Add Roles and Features
Select Web Server (IIS)
Click Next
Click Install
Wait for installation to finish
Result: IIS web server becomes ready for hosting websites.

5.2 Configure Web Server (IIS)

Real Host

A real host is a website that uses a real physical folder on a server to store and display website files directly.

It means the website content is stored in an actual directory on the computer hosting the web server.

Example:
C:\inetpub\wwwroot\schoolsite
on a Windows Server running Internet Information Services.
Virtual Host

A virtual host is a method used by a web server to host multiple websites on one server using different domain names, IP addresses, or port numbers.

It allows one physical server to serve many websites independently.

Example:
  • school.local
  • library.local
  • company.local
even though all websites are running on the same server.
Host Website
Add Website in IIS
Open IIS Manager
Right-click Sites
Click Add Website
Site Name

The name used to identify the website in IIS.

Example: School Website
Physical Path

Location of website files on the server.

Example:
C:\inetpub\wwwroot\school
Test Connectivity

Open a browser and type:

  • http://localhost
  • Server IP address
Example:
http://192.168.1.10
If the webpage opens successfully, the website is working.

5.3 Implement Security Access Control

SSL (Secure Sockets Layer)

SSL is a security technology that encrypts data sent between a user’s web browser and a web server to protect information from unauthorized access.

SSL helps keep sensitive information such as passwords, usernames, and payment details secure during communication.

Websites using SSL normally start with:

https://

instead of:

http://
Example: Online banking and e-commerce websites use SSL certificates to secure user data.
Antimalware

Antimalware is software designed to detect, prevent, and remove harmful programs such as viruses, worms, spyware, ransomware, and other malicious software from a computer or server.

It helps protect systems, files, and networks from cyber threats.

Example: Microsoft Defender scans and protects Windows Server from malware attacks.
Make Password Uncrackable

Use strong passwords containing:

  • Uppercase letters
  • Lowercase letters
  • Numbers
  • Symbols
Example:
Scho0l@2026
Avoid weak passwords like:
123456
Update Website

Regular updates fix errors and improve security.

Example: Updating plugins and web applications.
Do Regular Backup

Backup helps recover files after failure or attack.

Types of backup:

  • Full backup
  • Incremental backup
Example: Saving website files to external storage every week.

5.4 Deploying Web Application

Deployment

Deployment is the process of making a web application or website available for users by installing, configuring, and publishing it on a server.

Deployment includes transferring application files, configuring settings, and testing the application so users can access it through a browser.

Example: Uploading a school management system to a Windows web server so students and teachers can use it online.
1. Manual Deployment

Files are copied manually to the server.

Example: Copying website files using USB or Remote Desktop.
2. Version Control

Antimalware is software designed to detect, prevent, and remove harmful programs such as viruses, worms, spyware, ransomware, and other malicious software from a computer or server.

It helps protect systems, files, and networks from cyber threats.

Tracks changes in application files.

Common tool:
Git

Example: Developers upload code to GitHub before deployment.
3. Containerization

Applications run inside containers for easier deployment.

Common tool:
Docker

Example: Running a web application inside a Docker container.
Test the Deployment

After deployment:

  • Open the application in browser
  • Check links and pages
  • Verify database connection
Example: Testing login page and dashboard.

5.5 Test Web Application

Monitoring Tools

Tools used to observe server and application performance.

Examples:

  • Task Manager
  • Performance Monitor
  • IIS Logs

They help detect:

  • Errors
  • High CPU usage
  • Network problems
Security

Protect the application from attacks and unauthorized access.

Example: Firewall and antivirus software.
Performance

Ensure the website works fast and efficiently.

Methods:

  • Remove unused files
  • Optimize images
  • Upgrade RAM
Scalability

Ability of the application to support more users.

Example: Adding another server when website traffic increases.
Documentation

Written information about installation, configuration, and maintenance.

Example: Administrator guide showing how to restart IIS or restore backup.
