Group Discussion Task

Understanding DNS in Windows Server 2016
Course: Windows Server Administration
Learning Outcome 4: Deploy DNS Service
Estimated Time: 45 – 60 Minutes

Discussion Task

Your team is responsible for deploying and managing the Domain Name System (DNS) service in a Windows Server 2016 network.

Discuss how DNS works and how it is deployed, configured, and tested in a Windows Server environment. Focus on understanding the concepts and how they are applied during real system administration.

Discussion Points

1. DNS Service Installation

  • Explain the purpose of DNS in a network.
  • Describe the process of installing the DNS role in Windows Server 2016.
  • Why is DNS considered a critical service in enterprise networks?

2. DNS Records

Discuss the role of the following DNS records and provide practical examples:

  • A Record
  • CNAME Record
  • MX Record
  • PTR Record
  • NS Record
  • SOA Record

3. DNS Query Types

Explain how DNS resolves domain names using:

  • Recursive Query
  • Iterative Query
  • Non-Recursive Query

Also discuss the role of:

  • DNS Resolver
  • DNS Root Server
  • Authoritative DNS Server

4. DNS Zones and Zone Files

  • Forward Lookup Zone
  • Reverse Lookup Zone
  • Standard Primary Zone
  • Standard Secondary Zone
  • Active Directory–Integrated Zone

Explain why organizations might choose one type instead of another.

5. DNS Configuration Settings

  • Choosing a DNS hosting provider
  • Using Public DNS IPv4 addresses
  • Using Public DNS IPv6 addresses

Provide examples of commonly used public DNS servers.

6. Testing DNS Configuration

  • Verify DNS settings
  • Use Ping to test a domain name
  • Resolve a domain name using nslookup

Guiding Question

If DNS is not properly configured on a server or network, what problems could users experience when trying to access websites, email services, or internal company systems?

Expected Outcome of the Discussion

  • Explain the role and structure of DNS in a network.
  • Identify common DNS records and their functions.
  • Describe how DNS queries work during name resolution.
  • Differentiated between DNS zone types.
  • Understand how DNS is configured and tested in Windows Server 2016.

Additional Scenario Question (Independent from DNS Topic)

Website Performance and Load Management Scenario

Your company’s website has been receiving more visitors than usual, causing slow response times. The IT team wants to make sure the website can handle many users at the same time without crashing.

You have been asked to set up a system that shares the workload across multiple servers.

Explain all the steps you would follow to set up, configure, and manage this system so that multiple servers work together efficiently.

Include how you would:

  • Prepare the servers and network environment
  • Configure the load sharing system
  • Start and deploy the system
  • Test the system with multiple users
  • Monitor performance and fix possible issues
  • Maintain and update the system to ensure long-term reliability
