🖥️ What is Server Administration?

Server administration means managing and maintaining computer servers — the powerful computers that store, process, and deliver data or services to other computers on a network.

The person doing this is called a Server Administrator or System Administrator. Their main job is to ensure servers:

  • Run smoothly
  • Stay secure
  • Are properly backed up
  • Serve users or clients reliably
Example: PVC TSS School computer lab network with teachers' laptops, students' computers, printers, and a main server.

⚙️ Role of Server Administration at PVC TSS

  • Install and configure the server
    • Set up Windows Server or Linux OS
    • Create student and teacher accounts
  • Manage users and permissions
    • Allow teachers to edit marks
    • Allow students to access only their folders
  • Maintain security
    • Protect server from viruses
    • Manage firewalls and network security
  • Backup and recovery
    • Regularly save important school data
    • Restore data if the server crashes
  • Update and monitor performance
    • Keep the system up to date
    • Check for errors or slow performance

💾 Data Management Before & After Server Administration

Stage Style Challenges/Solutions
Before Server Administration Manual, local storage Disorganized, unsafe, no backups
After Server Administration Centralized, secure, managed by admin Organized, safe, easy access and recovery
Centralized = One main place → secure, fast, organized.
Non-centralized = Many separate places → risky, slow, messy.

🧠 Active Directory & Related Concepts

  • AD (Active Directory): Microsoft service for managing users, computers, and resources.
  • Domain: Network group managed by AD. Example: pvctss.local
  • DC (Domain Controller): Server that runs AD, verifies users, controls access.
  • ADDS: Windows Server role to make a server a DC.
Installing ADDS → gives tools to manage AD
Promoting to DC → server becomes main controller of the network.

🎯 Why Promote AD to DC?

  • Centralized user authentication
  • Centralized resource management (files, printers)
  • Security policies enforcement
  • Replication and redundancy
  • Simplified administration
Promoting AD to DC turns your server from a “capable tool” into the main controller of the school network.
