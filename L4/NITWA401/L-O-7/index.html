// Overview

What is FTP?

FTP (File Transfer Protocol) is a standard network method used to transfer files between computers over a LAN or the Internet. It enables uploading files to a server, downloading files from a server, and managing files on a remote system.

Simple Example: You use FTP when you upload a website from your computer to a web server so that people can see it online.

What is an FTP Server?

An FTP Server is a computer or software system that stores files, waits for FTP connections from users, and allows users to upload, download, or manage files.

Upload Files

Send files from your local computer to the remote FTP server's storage directory.

Download Files

Retrieve files stored on the FTP server to your local machine.

Manage Files

Create, rename, move, or delete files and folders on the remote server.

Multi-User Access

Multiple users can connect simultaneously with different permission levels.

// Purpose

Why Use FTP / FTP Server?

FTP is used because it makes file sharing easy, fast, and remote.

1. File Sharing over Network

Allows users in different locations to share files without USB or physical transfer.

Example: A teacher uploads notes to FTP server; students download them.
2. Central Storage

All files are stored in one place (FTP server), making it easier to manage and control data.

Example: A company stores all project files on one server.
3. Easy Upload and Download

Users can transfer large files easily — faster than email attachments.

4. Remote Access

Access files from anywhere (internet required). Useful for organizations with multiple branches.

5. Controlled Access (Security)

Admin can set permissions: Read only, Upload allowed, or Full control.

Example: Students can download files but cannot delete them.
// Section 1.1

Definition of Key Terms

FTP — File Transfer Protocol

A network protocol used to upload files to a server, download files from a server, and share files between computers.

Example: A company uploads website files from a computer to a web server using FTP.
FileZilla

A popular FTP client software used to connect to FTP servers. It helps users upload files, download files, and manage remote folders.

Example: A student uses FileZilla to upload HTML files to a school website server.
Hosting Account

A service that provides storage space on a server along with FTP access credentials (username + password).

Example: A web hosting company gives you an FTP server address, username, password, and storage space for your website.

Install FTP Services (Windows Server 2016)

FTP service is installed through IIS (Internet Information Services) in Windows Server 2016.

// FTP Installation Path in Windows Server 2016 Server Manager └── Add Roles and Features └── Role-based installation └── Select your server └── Web Server (IIS) └── Role Services ├── FTP Server │ ├── FTP Service ✓ │ └── FTP Extensibility ✓ └── Click Install

Result: After clicking Install, the FTP server is now available and ready to configure on your Windows Server.

// Section 2

Configure the FTP Server

After installation, configuration is required to make the FTP server functional.

2.1 Root Directory

The root directory is the main folder shared via FTP. It acts as the "home folder" for FTP users — all uploaded and downloaded files are stored here.

// Example Root Directory Path C:\FTP\SharedFiles

2.3 Passive FTP Mode (Optional)

Passive mode helps FTP work through firewalls — allowing smoother connections for users outside the company network.

Why it matters: Firewalls typically block normal (active) FTP connections. Enabling passive mode resolves this issue for remote users.

2.4 Enable Logging (Optional)

Logging records all FTP activities including login attempts, file uploads/downloads, and errors.

  • Login attempts (successful and failed)
  • File uploads and downloads
  • System errors and warnings
Example: Administrator checks logs to see who deleted a file.
// Section 2.2

Access Permissions

Permissions control what users can do in the FTP folder.

READ Read Permission

Users can view and download files from the server.

→ A student downloads lecture notes.
WRITE Write Permission

Users can upload files and modify existing files on the server.

→ A user uploads a report to the server.
EXECUTE Execute Permission

Users can run executable files or open scripts/programs on the server.

Note: Execute permission is rarely enabled for security reasons.

Setting Permissions for Shared Directory

  1. 1
    Go to FTP Folder

    Navigate to your FTP directory, e.g. C:\FTP\SharedFiles

  2. 2
    Open Properties

    Right-click the folder → select Properties.

  3. 3
    Security Tab

    Navigate to the Security tab in the Properties window.

  4. 4
    Add Users or Groups

    Click Edit → Add → type the username or group name.

  5. 5
    Assign Permissions

    Select Read, Write, or Modify based on the user's role.

Teachers

Full access — can upload, download, modify, and delete files.

Students

Read-only access — can only download and view files.

// Section 2.5

Test the FTP Server

Testing ensures the server is working correctly before use.

  1. 1
    Open FTP Client

    Launch FileZilla (or any FTP client of your choice).

  2. 2
    Enter Connection Details

    Input the Server IP address, Username, and Password.

  3. 3
    Click Connect

    Establish the connection to the FTP server.

  4. 4
    Test Transfers

    Try uploading a test file and downloading it back. Verify both actions succeed.

Expected Result: Connection is successful and files transfer properly in both directions.

// Section 3

Implement FTP Server File Sharing

3.1 FTP Server Software Options

  • IIS FTP Server — Built-in to Windows Server 2016 (most commonly used)
  • FileZilla Server — Third-party, free, open-source option
  • Serv-U FTP Server — Commercial-grade FTP solution

3.3 Test FTP File Sharing

  1. 1
    Connect via FTP Client

    Open FileZilla and connect to the server using your credentials.

  2. 2
    Login with User Account

    Use the account that has been configured with appropriate permissions.

  3. 3
    Test Download

    Attempt to download a file from the shared directory.

  4. 4
    Test Upload (if allowed)

    Attempt to upload a file and check if it appears on the server.

  5. 5
    Verify Changes

    Confirm that uploaded files are visible and permissions are correctly applied.

Success means: File sharing is working correctly and permissions are correctly applied for each user role.
