Learning Outcome 3: Deploy DHCP Services (10 Hours)

1. Introduction to DHCP

What is DHCP?

DHCP (Dynamic Host Configuration Protocol) is a network service that automatically assigns IP addresses and other network settings to client computers.

Instead of manually configuring each device, DHCP:

  • Assigns IP addresses automatically
  • Reduces configuration errors
  • Prevents IP conflicts
  • Saves time in large networks
  • Provides centralized IP management

2. DHCP Operations (DORA Process)

When a client connects to the network, DHCP uses the DORA process:

1. Discover

  • The client sends a broadcast message.
  • It searches for a DHCP server on the network.
  • This is the first step in obtaining an IP address.

2. Offer

  • The DHCP server responds with an IP address offer.
  • The server proposes network configuration details.
  • This includes:
  • IP Address
  • Subnet Mask
  • Default Gateway
  • DNS Server

3. Request

  • The client accepts the offered IP address.
  • It sends a request message to the DHCP server.
  • This confirms the selected IP address.

4. Acknowledgement (ACK)

  • The server confirms the assignment.
  • The IP address is officially leased to the client.
  • The client can now use the network.

What is a Lease?

A lease is the time period a DHCP server allows a client to use an IP address.

When the lease expires, the client must renew it.

This allows efficient IP address management.

3. Installation of DHCP Role (Using GUI)

Prerequisites

  • Windows Server must be installed
  • Server must have a static IP address
  • Server must be joined to a domain (Active Directory)
  • Administrator privileges required

Step-by-Step DHCP Installation (GUI Method)

Step 1: Open Server Manager

  • Click Start
  • Open Server Manager

Step 2: Add Roles and Features

  • Click Manage
  • Select Add Roles and Features

Step 3: Installation Type

  • Choose Role-based or feature-based installation
  • Click Next

Step 4: Select Server

  • Choose the local server
  • Click Next

Step 5: Select DHCP Server Role

  • Check DHCP Server
  • Click Add Features
  • Click Next

Step 6: Install

  • Click Install
  • Wait for completion

Step 7: Post-Installation Configuration

  • Click Complete DHCP Configuration
  • Authorize the DHCP server in Active Directory

4. DHCP Authorization

Why Authorization is Required

  • Prevents unauthorized (rogue) DHCP servers
  • Ensures only trusted servers assign IP addresses
  • Required in Active Directory environments

If not authorized, the DHCP service will not work properly.

5. DHCP Configuration

A. Enable the DHCP Server

  • Open Server Manager
  • Go to Tools
  • Open DHCP
  • Ensure the service is started

B. Create and Configure a Scope

What is a Scope?

A Scope is a range of IP addresses that the DHCP server can assign to clients.

Without a scope, DHCP cannot distribute IP addresses.

Scope Configuration Steps (GUI)

  • Step 1: Open DHCP Console (Server Manager → Tools → DHCP)
  • Step 2: Create New Scope (Right-click IPv4 → New Scope)
  • Step 3: Set Scope Name (example: Office Network)
  • Step 4: Set IP Address Range (Start IP: 192.168.1.10 / End IP: 192.168.1.100)
  • Step 5: Set Subnet Mask (Example: 255.255.255.0)
  • Step 6: Configure Exclusions (Optional)
  • Step 7: Lease Duration
  • Step 8: Activate Scope (Select Yes, I want to activate this scope)

6. Configure Scope Options

  • Default Gateway
  • DNS Server
  • Domain Name
  • Optional: Reservations (using MAC Address)

These settings are automatically given to clients.

7. Starting the DHCP Service

  • Open Services
  • Ensure DHCP Server is running
  • If not, click Start

The service must be running for clients to receive IP addresses.

8. Testing DHCP Configuration

A. On the Client Computer

Check IP Address

ipconfig /all

This shows:

  • Assigned IP address
  • Subnet mask
  • Default gateway
  • DNS server

Renew IP Address

ipconfig /renew

Release IP Address

ipconfig /release

B. On the DHCP Server

Check Active Leases

  • Open DHCP Console
  • Go to IPv4 → Scope → Address Leases
  • Verify client appears in the list

If the client appears: ✔ DHCP is working correctly.
