LEARNING OUTCOME 4: DEPLOY DNS SERVICE

1. INTRODUCTION TO DNS (DOMAIN NAME SYSTEM)

The Domain Name System (DNS) is a network service that translates domain names (human-readable names) into IP addresses (numbers used by computers to identify devices on a network).

Computers communicate using IP addresses, but humans prefer easy names. DNS acts like the phonebook of the internet.

Example
When a user types: www.google.com
DNS converts it into an IP address such as: 142.250.190.4
This allows the computer to connect to the correct web server.

DNS helps to:

  • Locate websites on the internet
  • Identify servers in a network
  • Support email delivery
  • Simplify network administration

In Windows Server 2016 and later, DNS can be installed as a server role. It allows administrators to manage domain name resolution within a local network or organization.

2. INSTALLATION OF DNS SERVICE

DNS service can be installed through Server Manager in Windows Server.

Steps to Install DNS Server

1. Open Server Manager
2. Click Manage
3. Select Add Roles and Features
4. Choose Role-based or feature-based installation
5. Select the Server
6. Check DNS Server
7. Click Add Features
8. Click Next
9. Click Install

After installation, DNS management tools are available in:

Tools → DNS

3. DESCRIPTION OF DNS RECORDS

A DNS record is an entry in the Domain Name System (DNS) database that stores information about a domain and how it should be handled on the network or internet.

DNS records define how a domain name is connected to resources such as IP addresses, mail servers, or other domain names. They allow DNS servers to respond correctly when a user requests a domain.

They are stored inside DNS zone files.

A DNS zone is a portion of the DNS namespace that is managed and controlled by a specific DNS server or group of DNS servers.

It contains a collection of DNS records for a particular domain or part of a domain. DNS zones allow administrators to organize and manage DNS information efficiently.

A. A Record (Address Record)

Definition
An A record maps a domain name to an IPv4 address.

Domain Name: www.school.com
IP Address: 192.168.1.20

When users type www.school.com, DNS returns 192.168.1.20.

Use

  • Website hosting
  • Server identification

B. CNAME Record (Canonical Name)

Definition
A CNAME record creates an alias for another domain name.

Instead of pointing to an IP address directly, it points to another hostname.

mail.school.com → server.school.com

This means mail.school.com is an alias of server.school.com.

Use

  • Redirecting multiple names to one server
  • Simplifying domain management

C. MX Record (Mail Exchange)

Definition
An MX record specifies the mail server responsible for receiving email for a domain.

school.com → mail.school.com

Emails sent to:

info@school.com

are delivered to mail.school.com.

Use

  • Email routing
  • Email server configuration

D. PTR Record (Pointer Record)

Definition
A PTR record maps an IP address to a domain name.

This is called reverse DNS lookup.

192.168.1.20 → server.school.com

Use

  • Network troubleshooting
  • Email server verification

E. NS Record (Name Server)

Definition
An NS record identifies the DNS servers responsible for a domain.

school.com → ns1.school.com
school.com → ns2.school.com

Use

  • Delegating DNS authority
  • Identifying DNS servers

F. SOA Record (Start of Authority)

Definition
The SOA record contains important administrative information about a DNS zone.

It includes:

  • Primary DNS server
  • Administrator email
  • Zone version number
  • Refresh intervals
Primary Server: ns1.school.com
Admin Email: admin@school.com

Use

  • DNS zone management
  • Synchronizing secondary servers

4. DESCRIPTION OF DNS QUERIES

DNS queries are requests sent by computers to find the IP address of a domain name.

Recursive Query: requires the DNS server to return the final answer to the client.

If the server does not know the answer, it will query other DNS servers until it finds the correct response.

Example: A user requests: www.microsoft.com

The DNS server continues searching until it returns the correct IP address.

This Recursive Query is used by client computers.

Iterative Query: In an iterative query, the DNS server returns the best possible answer it has.

If it does not know the answer, it refers the requester to another DNS server.

It is Used between DNS servers.

Non-Recursive Query: occurs when the DNS server already has the answer in its cache or zone file.

It returns the result immediately without contacting other servers.

Example
If the DNS server already knows the IP for school.com, it sends the answer directly.

5. DNS RESOLVER

A DNS Resolver is a system component that initiates DNS queries to find the IP address of a domain.

It is usually located on:

  • Client computers
  • Internet Service Provider (ISP) servers
Example
When a user types:
www.facebook.com
The DNS resolver starts the search process.

6. DNS Root Server

Definition

A DNS Root Server is the top level of the DNS hierarchy.

It directs queries to the correct Top-Level Domain (TLD) servers.

Example
When searching for:
www.example.com
The root server directs the request to the .com DNS server.

Importance

  • Starting point for DNS queries
  • Supports global internet name resolution

7. Authoritative DNS Server

Definition

An Authoritative DNS Server contains the official DNS records for a domain.

It provides the final answer for DNS queries.

Example
The DNS server responsible for:
school.com

contains all records such as:

  • A
  • MX
  • CNAME

Types

  • Primary DNS server
  • Secondary DNS server

8. Description of DNS Zones and Zone Files

What is a DNS Zone?

A DNS zone is a portion of the DNS namespace that is managed by a specific DNS server.

What is a Zone File?

A zone file stores DNS records such as:

  • A
  • MX
  • NS
  • PTR

It contains all configuration information for a domain.

9. Types of DNS Zones

Forward Lookup Zone

Definition

A Forward Lookup Zone maps domain names to IP addresses.

www.school.com → 192.168.1.20

Use

Normal DNS name resolution.

Reverse Lookup Zone

Definition

A Reverse Lookup Zone maps IP addresses to domain names.

192.168.1.20 → server.school.com

Use

  • Troubleshooting
  • Email verification

Standard Primary Zone

Definition

A Standard Primary Zone stores the main writable copy of the DNS database.

Characteristics

  • Allows DNS record modifications
  • Stored locally on the DNS server

AD-Integrated Zone

Definition

An Active Directory Integrated Zone stores DNS information in Active Directory.

Advantages

  • Automatic replication
  • Improved security
  • Fault tolerance

Example

Used in Active Directory Domain environments.

Standard Secondary Zone

Definition

A Standard Secondary Zone is a read-only copy of the primary DNS zone.

Purpose

  • Backup DNS server
  • Load balancing
  • Fault tolerance

10. Perform DNS Installation Process

DNS Configuration After Installation

1. Open Server Manager
2. Click Tools
3. Select DNS
4. Right-click Forward Lookup Zones
5. Select New Zone
6. Choose Primary Zone
7. Enter the zone name (example: school.com)
8. Create the zone file
9. Add DNS records such as:
• A Record
• MX Record
• CNAME Record

14. Testing DNS Configuration

After configuring DNS, testing ensures it works correctly.

Verify DNS Settings

Check DNS configuration in the network adapter.

1. Open Control Panel
2. Open Network and Sharing Center
3. Select Change adapter settings
4. Open Internet Protocol (IPv4)
5. Verify DNS server addresses

Ping a Domain Name

Ping can test whether DNS resolves a domain name.

ping www.google.com

Result

If DNS works, the system returns an IP address.

Example:
Pinging www.google.com [142.250.190.4]

Resolve a Domain Name

The nslookup command is used to resolve domain names.

nslookup www.microsoft.com

Output

Displays:

  • DNS server used
  • IP address of the domain
