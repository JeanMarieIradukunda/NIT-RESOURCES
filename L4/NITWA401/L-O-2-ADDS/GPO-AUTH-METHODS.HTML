GROUP POLICY MANAGEMENT

1. Management of Group Policy Objects (GPO)

1.1 Description of GPO

Group Policy Object (GPO) is a feature in Windows Server that allows administrators to control and manage:

  • User settings
  • Computer settings

Using GPO, an administrator can:

  • Enforce security rules
  • Control software installation
  • Set desktop restrictions
  • Configure system behavior

GPOs work in an Active Directory (AD) environment.

1.2 Types of GPO

Local GPO

  • Applies to one computer only
  • Does not require Active Directory
  • Settings affect all users on that computer
  • Used mostly on standalone computers
Example: Disable Control Panel on one PC

Non-Local GPO (Domain GPO)

  • Created in Active Directory
  • Can be applied to Site, Domain, or Organizational Unit (OU)
  • Used in enterprise networks
Example: Password policy for all users in a domain

Starter GPO

  • Used as a template
  • Helps create new GPOs with predefined settings
  • Saves time and ensures consistency
Example: Starter GPO for security settings

1.3 GPO Hierarchy Policies (Order of Application)

GPOs are applied in the LSDOU order: Local → Site → Domain → OU. This order determines which policy settings take precedence.

Local Policy

  • Definition: GPO stored on the computer itself
  • Priority: Lowest
  • Where Applied: Only affects the local computer
Example: Lab PC restricts all users from changing wallpaper

Site Policy

  • Definition: GPO linked to an Active Directory site
  • Priority: Higher than Local, lower than Domain
  • Where Applied: All computers in a specific physical location
Example: All PCs in Lab A have a shared desktop background

Domain Policy

  • Definition: GPO linked to the entire domain
  • Priority: Higher than Site, lower than OU
  • Where Applied: All users and computers in the domain
Example: Every student must use the same password format

OU (Organizational Unit) Policy

  • Definition: GPO linked to a specific OU
  • Priority: Highest
  • Where Applied: Specific users or computers inside an OU
Example: All computers in IT OU have software X installed automatically
Important Notes:
  • When multiple GPOs configure the same setting, the last applied GPO takes precedence (OU > Domain > Site > Local)
  • Administrators can use Block Inheritance or Enforce Policy to control the application of GPOs

2. Creation of Accounts (Server Administration)

In Server Administration, an Account refers to an identity that is allowed to access a server and its resources. It defines who can log in and what actions they are allowed to perform.

An account consists of:

  • Username
  • Authentication method (password, SSH key, certificate, etc.)
  • Permissions and privileges

The server uses this information to determine access rights and security policies.

2.1 User Accounts

A User Account represents a person who logs into the network to access resources. User accounts are centrally managed using Active Directory (AD) in Windows Server environments.

Examples of Users

  • Students
  • Teachers
  • Employees
  • Administrators

Types of User Accounts

  • Domain User Accounts – Managed centrally by Active Directory
  • Local User Accounts – Exist on a single computer only (not recommended in domains)

User Account Characteristics

  • Allows users to log in to the domain
  • Provides access to files, printers, applications, and network resources
  • Contains username, password, and group membership

Example

  • Username: jsmith
  • Domain: school.local

When a user logs in, Windows verifies the user's identity, permissions, and applicable Group Policy Objects (GPOs).

2.2 Computer Accounts

A Computer Account represents a device (PC, laptop, or server) that is joined to a domain. Each computer has its own identity within Active Directory.

Computer Account Characteristics

  • Represents a device, not a person
  • Authenticates to the domain
  • Receives security and configuration policies
  • Account names end with a $ symbol

Examples

  • PC01$
  • SERVER02$

Why Computer Accounts Exist

  • Enable secure communication with the domain
  • Allow authentication before users log in
  • Support device-based security policies

When a computer is joined to a domain (e.g., PC01.school.local), Active Directory automatically creates a corresponding computer account (PC01$).

2.3 User Accounts in Group Policy (GPO)

User Configuration GPOs apply to users regardless of which computer they log into.

Examples of User GPO Settings

  • Desktop wallpaper
  • Start Menu restrictions
  • Control Panel access control
  • Folder redirection
  • Logon scripts

Applied when: The user logs in

2.4 Computer Accounts in Group Policy (GPO)

Computer Configuration GPOs apply to computers regardless of which user logs in.

Examples of Computer GPO Settings

  • Windows Updates configuration
  • Firewall settings
  • USB device restrictions
  • Automatic software installation
  • Security policies

Applied when: The computer starts up

2.5 Key Differences (Exam Important)

Feature User Account Computer Account
Represents Person Device
Stored in Active Directory Yes Yes
GPO Section User Configuration Computer Configuration
Applied When User logs in Computer boots
Affects User behavior Machine behavior

2.6 How GPOs Are Applied

  1. Computer authenticates using its Computer Account
  2. User authenticates using their User Account
  3. Computer GPOs are applied first
  4. User GPOs are applied next

GPO Processing Order: Local → Site → Domain → OU (LSDOU)

2.7 Real-Life Example (School Network)

User Account (Student)

  • Can log in on any school computer
  • Receives restricted desktop
  • No Control Panel access
  • Mapped network drives

Computer Account (School Lab PC)

  • USB ports disabled
  • Automatic updates enabled
  • Software installed automatically

3. Sharing Networked Resources

Network sharing allows multiple users to access the same resources.

  • Common shared resources: Files, Folders, Printers, Applications
  • Steps to share a folder:
    1. Right-click the folder
    2. Select Properties
    3. Go to Sharing tab
    4. Set permissions (Read, Write, Modify, Full Control)

1️⃣ Sharing Networked Resources (Windows Server & GPO Management)

What are Networked Resources?

Networked resources are system resources stored on a server and accessed by users or computers over a network.

  • Shared folders (files, documents)
  • Printers
  • Applications
  • Network drives

In Windows Server, these resources are typically hosted on File Servers or Print Servers and controlled using Active Directory + GPOs.

How Windows Server Shares Resources

A. File Sharing

Windows Server uses SMB (Server Message Block) protocol to share files.

A shared folder has two layers of control:

  • Share Permissions (network-level): These permissions control access to the folder over the network. They determine what remote users can do with the folder, such as read, write, or modify files, when connecting from another computer.
  • NTFS Permissions (file-system-level): These permissions control access to the folder and its files at the local file system level. They provide more granular control than share permissions, allowing you to set specific rights like read, write, modify, or full control for individual users or groups, regardless of how they access the folder.

🔐 Effective permission = Most restrictive combination

B. Role of GPO in Sharing Resources

GPO does not store files, but controls how users access shared resources:

  • Map network drives automatically
  • Hide or restrict access to shared folders
  • Control offline file access
  • Enforce security on shared resources

Example (Very Important)

In a school or company:

  • A shared folder exists: \\Server01\Students
  • A GPO maps it as drive S: for students
  • Only users in Students OU can see and access it

This ensures:

  • ✔ Centralized access
  • ✔ Security
  • ✔ Easy management

Why GPO is Important for Resource Sharing

  • Without GPO: Admin must manually configure each PC
  • With GPO: Access is automatic, permissions are consistent, changes apply instantly across the network

4. Administration of Objects

In Active Directory, objects include Users, Computers, Groups, and Printers.

  • Tasks: Create, Delete, Modify, Move between OUs
  • Tools: ADUC (Active Directory Users and Computers), GPMC (Group Policy Management Console)

2️⃣ Administration of Objects (Active Directory & GPO Context)

What Are Objects?

An object is any item stored in the directory.

  • User accounts
  • Computer accounts
  • Groups
  • Printers
  • Shared folders
  • Organizational Units (OUs)

Administration of Objects Means:

  • Creating
  • Modifying
  • Deleting
  • Securing
  • Applying policies

Role of GPO in Object Administration

GPO controls how objects behave:

A. User Objects

  • Restrict Control Panel
  • Set desktop environment
  • Enforce password policies
  • Redirect folders
  • Apply login scripts

📌 Applied when: User logs in

B. Computer Objects

  • Configure firewall
  • Control Windows updates
  • Disable USB ports
  • Install software
  • Apply security baselines

📌 Applied when: Computer starts

Delegation of Administration (Very Important)

Windows Server allows delegation:

  • Junior admins can manage specific objects without full admin rights
  • Example: Teacher IT staff can reset student passwords but cannot modify servers

This improves:

  • ✔ Security
  • ✔ Accountability
  • ✔ Scalability

5. User Authentication Methods

Authentication is the process of verifying identity before granting access to a computer, network, or resource.

3️⃣ User Authentication Methods (Windows Server Environment)

What Is User Authentication?

Authentication verifies: “Are you really who you claim to be?”

Windows Server uses centralized authentication via Active Directory Domain Services (AD DS).

Common Authentication Factors:

  • Something you know (password)
  • Something you have (smart card, token)
  • Something you are (biometrics)

Windows Server mainly uses:

  • Username + password
  • Smart cards
  • Certificates
  • MFA (with extensions)

Authentication Flow (Domain Login):

  • User enters credentials
  • Computer contacts Domain Controller
  • Credentials are verified
  • Access is granted
  • GPOs are applied

5.1 LDAP (Lightweight Directory Access Protocol)

  • Definition: LDAP is a protocol used to access and manage directory services, such as Active Directory (AD). It defines a standard method for querying and modifying directory information over a network.
  • LDAP is commonly used to:
    • Check user login credentials during authentication
    • Search for information about users, computers, or other resources
    • Manage groups, permissions, and organizational structures in the directory
Analogy: A librarian checking a student’s ID to allow entry to the library.
Example: User logs in → system queries Active Directory via LDAP → access granted or denied

Role of LDAP in Windows Server:

  • Locate user accounts
  • Authenticate users
  • Retrieve group membership
  • Apply GPOs
  • Search AD objects

When a user logs in: Windows uses LDAP to find the user object in AD.

5.2 RADIUS (Remote Authentication Dial-In User Service)

  • Definition: RADIUS is a network protocol that provides centralized authentication, authorization, and accounting (AAA) for users who connect to a network service, such as Wi-Fi, VPN, or remote access.
  • RADIUS is commonly used to:
    • Authenticate users attempting to access network resources
    • Authorize user access based on roles or permissions
    • Maintain accounting logs of user activity for auditing purposes
Analogy: A security guard checks your ID at the school gate before entry.
Example: Student connects to school Wi-Fi → RADIUS server verifies login → access allowed or denied

RADIUS in Windows Server:

  • Windows Server provides RADIUS via Network Policy Server (NPS)
  • NPS integrates with Active Directory, VPN servers, Wi-Fi access points, switches

RADIUS performs the AAA Model:

  • Authentication – Who are you?
  • Authorization – What are you allowed to access?
  • Accounting – What did you do?

Relationship Between LDAP & RADIUS

  • LDAP → Finds and verifies user objects
  • RADIUS → Controls network access using AD accounts
  • They work together, not against each other

🔚 Final Concept Map (Mental Picture)

  • Active Directory → Stores objects
  • LDAP → Finds and manages objects
  • Authentication → Verifies users
  • RADIUS → Controls network access
  • GPO → Controls behavior of users and computers
  • Shared Resources → Access controlled by AD + GPO
