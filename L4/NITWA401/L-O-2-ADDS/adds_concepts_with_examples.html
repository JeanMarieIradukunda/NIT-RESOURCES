Active Directory Concepts

1. Domain
  • A domain is the basic unit of Active Directory.
  • It is a logical group of objects (users, computers, printers, groups) that share the same database and security policies.
  • Each domain has a unique name, like school.local.
✅ School analogy: Think of a domain as one school campus (e.g., NIT). Everyone in that campus follows the same rules.
2. Domain Controller (DC)
  • A Domain Controller is a server that stores the AD database and handles authentication and authorization.
  • It ensures that users and computers prove their identity before accessing resources.
✅ School analogy: The head office of the school that checks ID cards and decides permissions.
3. Forest
  • A forest is the topmost container in AD.
  • It is a collection of one or more domains sharing the same schema and global catalog.
  • The first domain created is called the forest root domain.
✅ School analogy: A school district (PADRI VJEKO CENTRE) containing multiple campuses (NIT, FAD, BDC).
4. Tree
  • A tree is a hierarchy of domains within a forest.
  • Domains in a tree share a contiguous namespace (e.g., nit.school.local, fad.school.local).
✅ School analogy: A branch of schools under one district that share a common name.
5. Organizational Units (OUs)
  • OUs are containers within a domain used to organize users, groups, and computers.
  • Policies and control can be delegated at the OU level.
✅ School analogy: Departments or classes inside NIT, e.g., ICT Department, Finance Department.
6. Global Catalog (GC)
  • Stores a partial replica of all objects in the forest.
  • Allows users to search and log in quickly across domains.
✅ School analogy: A central library index across all campuses.
7. Trust Relationship
  • A trust is a link between domains/forests allowing cross-domain access.
  • Trusts can be one-way or two-way.
✅ School analogy: Agreement between schools allowing students to use other campuses' resources.
8. Group Policy (GPO)
  • A set of rules applied to users and computers in a domain/OU.
  • Controls password rules, desktop settings, software installation.
✅ School analogy: School rules applied to classes or departments.
9. User Accounts
  • Represents a person in AD.
  • Stores username, password, and permissions.
✅ School analogy: A student/teacher ID for accessing facilities.
10. Computer Accounts
  • Represents computers joined to a domain.
  • Allows authentication and centralized management.
✅ School analogy: Registered school computers with unique IDs.
11. Security Groups & Distribution Groups
  • Security Groups: Assign permissions to resources.
  • Distribution Groups: Used for email communication only.
✅ School analogy: Security Group = Science Club (access labs), Distribution Group = Debate Team mailing list.
12. Schema
  • The blueprint of Active Directory.
  • Defines the structure and attributes of objects.
✅ School analogy: Standard student record format for all schools.
Real-World Example
Forest: PADRI VJEKO CENTRE — contains all domains (NIT, FAD, BDC) sharing the schema and global catalog.
Tree: NIT, FAD, BDC — separate campuses under the same forest.
Domain: Each campus (NIT, FAD, BDC) has its own rules and staff/student records.
DC: Administration office of each campus managing logins.
GC: Central library index for all campuses.
OUs: Departments or classes inside a campus.
User Accounts: Student/teacher IDs.
Computer Accounts: Registered lab computers.
Trust Relationship: Cross-campus resource access.
GPO: School rules like uniforms and exam policies.
Security & Distribution Groups: Clubs and communication groups.
Schema: Standard record format for all campuses.

Active Directory Hierarchy – Real-World Example

🔹 FOREST (PADRI VJEKO CENTRE)

Represents the entire Active Directory environment. All domains (NIT, FAD, BDC) belong to this forest. They share a common schema (rules for objects) and global catalog.

👉 Analogy: The whole school district, PADRI VJEKO CENTRE.
🔹 TREES → NIT, FAD, BDC

Each tree can contain one or more domains. They are separate campuses but still under the same forest.

👉 Analogy: NIT campus, FAD campus, and BDC campus under one school district.
🔹 DOMAIN

A logical boundary where rules, policies, and authentication are managed. Each campus (like NIT) is a domain with its own rules but still part of the bigger forest.

👉 Analogy: Each campus (NIT, FAD, BDC) operates with its own staff and student records.
🔹 DOMAIN CONTROLLER (DC)

The server that manages login, authentication, and security inside the domain. It enforces policies and verifies users.

👉 Analogy: The administration office of each campus that checks ID cards and manages students.
🔹 GLOBAL CATALOG (GC)

A central searchable index that stores key information about all objects in the forest. Allows users in one domain to quickly find resources in another domain.

👉 Analogy: A central library index for all campuses so a student from NIT can look up a teacher in FAD.
🔹 ORGANIZATIONAL UNITS (OUs)

Sub-divisions inside a domain to organize users, groups, and computers. Example: ICT Department, Finance Department, Engineering Class.

👉 Analogy: Inside NIT campus, there are departments and classes to organize staff and students.
🔹 USER ACCOUNTS

Represent individual people in AD (students, teachers, staff). Provide username and password to access resources.

👉 Analogy: Each student/teacher has a school ID card.
🔹 COMPUTER ACCOUNTS

Represent each device (PC, laptop, server) joined to the domain. Enables central management (updates, policies, security).

👉 Analogy: Registered lab computers in the school that are officially recognized.
🔹 TRUST RELATIONSHIP

Links between different domains (NIT ↔ FAD ↔ BDC). Allows cross-campus access to resources.

👉 Analogy: An agreement where NIT students can attend a FAD library session.
🔹 GROUP POLICY (GPO)

Set of rules controlling what users and computers can/can’t do. Examples: password rules, desktop backgrounds, software restrictions.

👉 Analogy: School rules like uniforms, exam policies, or lab regulations applied across students.
🔹 SECURITY & DISTRIBUTION GROUPS

Security Groups: Assign permissions (who can use lab, printer, etc.). Distribution Groups: Email lists or communication-only groups.

👉 Analogy:
Security Group = Science Club (with access to labs).
Distribution Group = Debate Team mailing list (just for communication).
🔹 SCHEMA

The blueprint of the Active Directory database. Defines what objects exist (user, computer, printer) and what attributes they have (username, password, ID, etc.).

👉 Analogy: The standard student record form that ensures all schools record names, IDs, and subjects the same way.
