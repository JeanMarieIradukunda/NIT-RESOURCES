🔹 PART 1: Definition of Key Terms

1. Active Directory (AD)
Centralized directory & management system

Definition: Active Directory is a centralized database system in Windows Server that stores information about users, computers, printers, and other resources in a network. It helps administrators manage and organize these resources easily.

Example (School Analogy): Think of Active Directory as the school registry office at PADRI VJEKO CENTRE TSS. It keeps all the records — teachers, students, departments, labs, and rules — in one place so the system manager can easily know who is who and who has access to what.

2. Domain Controller (DC)
Authentication & authorization server

Definition: A Domain Controller is a server that runs Active Directory and is responsible for authenticating and authorizing users and computers on the network.

Example: Imagine the System Manager’s main server in LAB 5 that checks who logs in. When a student from the NIT department logs into a lab computer, the Domain Controller verifies their username and password before granting access — just like a security gate checking student IDs before letting them enter the lab.

3. Organizational Unit (OU)
Grouping container for management

Definition: An Organizational Unit is a container within Active Directory used to group users, computers, and other objects for easy management and policy application.

Example: At PADRI VJEKO CENTRE TSS, each department (NIT, FAD, WOT, BDC) can be an OU. This allows the System Manager to manage user accounts, computers, and rules separately — for instance, applying different internet access rules to each department.

4. Global Catalog (GC)
Forest-wide searchable index

Definition: The Global Catalog is a distributed data repository that contains information about every object in an Active Directory forest, making it easier to locate resources quickly.

Example: The GC is like the school’s master student & staff list that shows all students and teachers in every department. If someone searches for “John in WOT,” the GC can quickly locate him even if he’s in a different department or lab.

5. Group Policy Object (GPO)
Configuration & enforcement rules

Definition: A Group Policy Object is a collection of settings that control the working environment of user accounts and computer accounts.

Example: At PADRI VJEKO CENTRE TSS, the System Manager creates a GPO to prevent students from installing new software or changing desktop backgrounds in LAB 6. It’s like a school rulebook that defines what students and teachers can or can’t do on the network.

6. Active Directory Domain Services (AD DS)
Core directory service engine

Definition: AD DS is the core service of Active Directory that stores directory data and manages communication between users and domains, including login authentication and directory searches.

Example: AD DS is like the main school database server that contains every department, class, and computer record. It allows the System Manager to authenticate users (staff and students) and control what they can access.

🔹 PART 2: Active Directory Concepts

Definition: A domain is a logical group of network objects (users, computers, printers) that share the same Active Directory database.

Example: PADRI VJEKO CENTRE TSS might have a domain named pvctss.local that includes all computers and users from LAB 5, LAB 6, and all departments — managed centrally by the System Manager.

(Reinforced) A DC is the server that stores the Active Directory database and manages logins and permissions for the domain. Example: The main server in LAB 5 acts as a DC, verifying every login from LAB 6 computers too — ensuring all users follow the same domain rules.

Definition: A forest is the highest-level container in Active Directory that holds one or more domains with a shared schema and trust relationships.

Example: If PADRI VJEKO CENTRE TSS expands and opens another branch called PADRI VJEKO COLLEGE, both schools’ domains (pvctss.local and pvcoll.local) can be part of one forest, allowing them to share resources securely.

Definition: A tree is a collection of one or more domains in a contiguous namespace within a forest.

Example: In the PADRI VJEKO forest, you can have nit.pvctss.local and fad.pvctss.local. They form a tree structure under the main domain pvctss.local — just like departments under one school.

Used to organize users, computers, and groups within a domain to apply policies efficiently. Example: Each department (NIT, FAD, WOT, BDC) is an OU managed by their own supervisor under the system manager.

Allows quick search across the entire forest for any object. Example: A teacher from FAD can easily find a student in WOT using the GC, without asking the WOT admin.

Definition: A trust is a link between two domains that allows users in one domain to access resources in another.

Example: If pvctss.local trusts pvcoll.local, then a teacher from PADRI VJEKO COLLEGE can log in to computers in PADRI VJEKO CENTRE’s LAB 6 — just like mutual trust between two school branches.

Allows the system manager to enforce rules such as password strength, software installation rights, or logon hours for students and staff.

Example:

  • GPO for LAB 5: Disable USB ports.
  • GPO for LAB 6: Set wallpaper with school logo.

Accounts created for individuals to log in and access resources within the domain. Example: Each student (e.g., john.nit@pvctss.local) has a user account to log in to any lab computer, store files, and use applications securely.

These are accounts that represent each computer within the domain. Example: Each computer in LAB 5 and LAB 6 has a computer account (like LAB5-PC01, LAB6-PC02) registered in AD, just as each lab desk has its own unique inventory tag.

Security Groups: Used to assign permissions to shared resources (e.g., file access).
Distribution Groups: Used mainly for email communication.

Example:

  • WOT Students (Security) — can access the shared WOT department folder.
  • All Staff (Distribution) — used to send announcements via email.

Definition: The schema defines the structure of objects and attributes in the Active Directory database.

Example: In the school analogy, the schema is like the rules for the student and staff database — defining what fields exist for each (e.g., Student ID, Name, Department, Email). If you want to add a new attribute like “RFID_Card_Number,” you must modify the schema.
