Windows Server – Group Policy Object (GPO)

Complete Structured Notes (Student Friendly)

1. What is GPO in Windows Server?

GPO (Group Policy Object) is a set of rules and settings used by administrators to control how users and computers behave in an Active Directory environment.

  • Enforce password policies
  • Restrict access to Control Panel
  • Install software automatically
  • Set desktop wallpapers
  • Control Windows Updates
  • Configure security settings
In short: GPO = Centralized management of users and computers.

2. User, Domain, and OU in GPO

a) User (GPO Context)

A User is an Active Directory account used to log in.

  • User Configuration applies when the user logs in
  • Desktop settings
  • Start Menu
  • Folder Redirection
  • Logon Scripts
Example: Disable Control Panel for students only

b) Domain

The Domain is the main Active Directory structure (e.g. school.local).

  • Applies to all users
  • Applies to all computers
Example: Enforce password policy for everyone

c) Organizational Unit (OU)

An OU is a container used to organize users and computers.

  • Students
  • Teachers
  • IT
  • Servers
GPOs linked to an OU apply only to objects inside that OU.

3. Default Policies

a) Default Domain Policy

  • Created automatically
  • Linked at Domain level
  • Password policy
  • Account lockout
  • Kerberos settings
Best Practice: Do NOT add many custom settings here.

b) Default Domain Controllers Policy

  • Linked to Domain Controllers OU
  • Applies only to DCs
  • Audit and security settings

4. User Configuration vs Computer Configuration

User Configuration

User Configuration in Group Policy defines settings that apply to a user account, regardless of which computer the user logs into.

  • Applies to users (user accounts)
  • Triggered when the user logs in
  • Controls the user environment and experience
  • Examples: Desktop settings, Start Menu, Control Panel restrictions

Computer Configuration

Computer Configuration in Group Policy defines settings that apply to a computer, regardless of which user logs into the machine.

  • Applies to computers (computer accounts)
  • Triggered when the computer starts or refreshes policy
  • Controls system-wide behavior and security
  • Examples: Security settings, Windows Updates, Firewall rules

5. Linked GPO & Processing Order

A Linked GPO is attached to a Domain, OU, or Site.

A GPO does nothing until it is linked.

GPO Processing Order (LSDOU)

  • Local
  • Site
  • Domain
  • OU (highest priority)

6. Real-Life Example

School Network

  • Domain: school.local
  • OUs: Students, Teachers
  • Password policy at Domain
  • Block games for Students
  • Allow USB for Teachers

7. Administrative Tasks (GUI)

A. Create an OU

  1. Open Active Directory Users and Computers
  2. Right-click Domain → New → Organizational Unit
  3. Enter name → OK

B. Create a User

  1. Right-click OU → New → User
  2. Enter user details
  3. Set password → Finish

C. Create a Group

  1. Right-click OU → New → Group
  2. Select Global & Security
  3. Click OK

D. Add User to Group

  1. Open Group → Members → Add
  2. Select user → OK

E. Create a GPO

  1. Open Group Policy Management
  2. Right-click Group Policy Objects → New
  3. Name GPO → OK

F. Link a GPO

  1. Right-click Domain or OU
  2. Link an Existing GPO
  3. Select GPO → OK

8. Verification Command

Force Group Policy update:

gpupdate /force
