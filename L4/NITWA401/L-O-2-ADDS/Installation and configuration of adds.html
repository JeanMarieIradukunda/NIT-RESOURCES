Interactive Active Directory Installation & Configuration

  • Set a Static IP:
    • Control Panel → Network and Sharing Center → Change adapter settings.
    • Right-click network adapter → Properties → IPv4 → Properties.
    • Set static IP, subnet mask, default gateway, and preferred DNS.
  • Rename the Server (Optional but Recommended):
    • Right-click This PC → Properties → Rename this PC → Restart.
Always set a static IP before promoting the server to DC.

  1. Open Server Manager.
  2. Manage → Add Roles and Features.
  3. Role-based or feature-based installation → Select server → Next.
  4. Check Active Directory Domain Services → Add Features → Next → Install.
Do not close Server Manager during installation.

  1. Click the flag icon in Server Manager → Promote this server to a domain controller.
  2. Choose deployment operation:
    • Add a new forest (first DC)
    • Add a domain controller to an existing domain
    • Add a new domain to an existing forest
  3. Enter domain name (e.g., school.local) → Next.
  4. Select Forest & Domain functional levels → Windows Server 2016.
  5. Set DSRM password → Next.
  6. Keep DNS server checked → Next.
  7. Review NetBIOS name → Next.
  8. AD database, log files, SYSVOL paths → Next.
  9. Review → Install → Server restarts automatically.
Example: Creating a new forest for a school domain: school.local

  • Log in after restart.
  • Server Manager → Tools → Active Directory Users and Computers → Verify domain appears.
  • Test DNS: nslookup school.local → Resolves to DC IP.
Tips:
  • DSRM password is critical for recovery.
  • First DC auto-configures DNS.
  • For lab setups, you may skip advanced GPOs.

1. Description of AD

AD stores information about network objects and centralizes authentication/authorization.

  • Forest: Top-level container of domains.
  • Domain: Logical group of objects.
  • OUs: Organizational Units for object organization.
Analogy: Forest → School district, Domain → School (NIT, FAD, BDC), OUs → Classrooms.
2. Domain Deployment Operations
  • Add DC to existing domain: Redundancy & fault tolerance.
  • Add new domain to existing forest: Create subdomain (e.g., FAD.school.local).
  • Add new forest: New AD environment (first DC in forest).
3. Promote Server to DC
  1. Install AD DS role.
  2. Promote server → Choose deployment → Enter domain → Set DSRM → Configure DNS/paths → Install.
  3. Verify AD installation → Open ADUC → Test DNS → Optionally create OUs, users, groups.
